# v0.5 / 引入 Agent X 循环迭代文档基线

## 版本分配依据

本轮是协作制度和文档体系扩展，不新增 Swift 功能、不改变 App 行为，归入 v0 协作自动化阶段。人工指定版本号为 `v0.5`。

## 背景

当前项目已有 Agent A / Agent B / Agent C 三角色流程：

- Agent A 分析目标并写版本化提示词。
- Agent B 在 `main` 上实现、轻量检查、提交并 push 到 `origin/main`。
- Agent C 下载 GitHub Actions artifact，核对 manifest、JUnit/摘要、日志和关键结果包。

后续需要支持 Agent X 作为主控调度角色，围绕人工总目标 X 循环调用 A -> B -> C，并基于 Agent C 的云端 artifact 验收结果决定继续、退回、暂停或完成。

## 目标

- 在入口规则中新增 Agent X 召唤词、职责、循环边界、停止条件和最终回复要求。
- 在 flow 和 flowchart 中描述 Agent X 主控循环。
- 在 test 规范中明确 Agent X 不得跳过 Agent C artifact 验收，失败时不得进入下一轮伪装成功。
- 在 prompt README 中明确 Agent X 可以要求 Agent A 为每轮生成版本化提示词。
- 在 README 和 update_log 中记录 v0.5 文档基线。

## 非目标

- 不启动真实 Agent X 自动循环。
- 不修改 Swift 源码、Xcode project、Gateway 或 workflow。
- 不新增测试 target。
- 不创建 PR、不切换分支；本轮按人工要求在 `main` 提交并 push 到 `origin/main`。

## 当前架构依据

- `AGENTS.md` 是 Agent 工作入口和主规则。
- `md/flow/flow.md` 记录当前真实逻辑和协作闭环。
- `md/flow/flowchart.md` 用 Mermaid 表达核心流程。
- `md/test/test.md` 规定本地轻量检查、云端结果包和 Agent C 复判。
- `md/prompt/README.md` 管理 Agent A 的版本化提示词。

## 实现步骤

1. 阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`README.md` 和 `md/prompt/README.md`。
2. 运行 `git status --short --branch`，必要时查看提交、分支和远端。
3. 修改文档，新增 Agent X 主控循环规则。
4. 明确 Agent X 停止条件和禁止项。
5. 记录 v0.5 更新日志。
6. 运行 `git diff --check`，复核 Markdown 代码块闭合和 diff 范围。

## 测试要求

- 必跑：`git diff --check`
- 必跑：`ruby -e 'require "yaml"; YAML.load_file(".github/workflows/ci-results.yml"); puts "yaml ok"'`
- Mermaid 图只需检查 Markdown 代码块闭合，不要求渲染。
- 本轮未改 Swift、Xcode project、Gateway 或 workflow，不需要运行 Swift smoke、Xcode build 或 Gateway smoke。

## CI、artifact 和 Agent C 验收要求

本轮按人工要求由 Agent B 提交并 push 到 `origin/main`，触发 GitHub Actions；Agent B 不下载或验收 artifact，后续由 Agent C 负责下载最新结果包并复判。未来 Agent X 真正主控循环时，每轮仍必须要求：

- Agent B 在 `main` 上提交并 push 到 `origin/main`。
- GitHub Actions 生成最新 run 的未加密 `ci-results` artifact。
- Agent C 下载最新 artifact 到 `/private/tmp/claw-c-review-<run_id>/`，核对 manifest、JUnit/摘要、日志和关键结果文件。
- Agent X 不得跳过 Agent C 复判，也不得把旧 run、旧 artifact 或本地输出冒充最新云端结果。

## 文档更新要求

- 协作制度变化必须同步 `README.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和 `md/prompt/README.md`。
- 正式版本记录写入 `update_log.md`。
- 入口规则写入 `AGENTS.md`，历史细节不要塞进入口文档。

## 验收标准

- 所有指定文档均说明 Agent X 的召唤方式和主控调度边界。
- Mermaid 图表达人工 -> Agent X -> Agent A -> Agent B -> GitHub Actions -> Agent C -> Agent X 判断的闭环。
- 测试规范明确 Agent X 不能跳过 Agent C artifact 验收，失败时不能继续下一轮伪装成功。
- `git diff --check` 通过。
- 本轮没有 Swift、Xcode project、Gateway 或 workflow 改动。

## 风险和禁止项

- 禁止把 Agent X 写成无限自动循环。
- 禁止 Agent X 替代 Agent C 的云端 artifact 验收。
- 禁止在总目标未完成时宣布完成。
- 禁止为了推进循环扩大无关改动范围。
- 禁止默认下载大体积测试数据、模型、历史 artifact 或无关产物。
