# v0.10（手机端Gateway能力复核摘要）Agent B 实现提示词

## 版本号

- 版本：v0.10
- 阶段：v0（核心智能能力）
- 任务名：手机端Gateway能力复核摘要
- 提交信息格式：`v0.10: 手机端Gateway能力复核摘要`

## 版本分配依据

v0.6 已增强 Gateway `agentTrace` 证据策略，v0.7 已把 `agentTrace` 安全 metadata 回传到手机端复核，v0.8 已完成 iPad/regular size class 多栏复核工作台，v0.9 已让桌面 Gateway 在每个 session 开始时写入 `gateway-capability-snapshot.json` `auditLog` artifact。

当前缺口是：v0.9 产生的是 session 级 `artifactStored`，没有 `actionID/actionKind/actionTitle`。Swift reducer 现在会在 `upsertResult(from:in:fallbackStatus:finished:)` 中直接忽略无 action 绑定的 event，导致手机端 Mission Run / Gateway session / iPad 工作台看不到这份能力快照。v0.10 继续归入 `v0（核心智能能力）`，目标是让手机端保存并展示 Gateway 能力安全摘要，不新增协议能力、不读取 Gateway `file://` 内容、不扩大真实电脑权限。

## 背景

Claw 当前是 SwiftUI iPhone/iPad 控制台 + 桌面 Claw Gateway 原型：

- 手机端负责输入任务、生成计划、展示风险、审批、发送 envelope 和查看事件/artifact 引用。
- 真实电脑动作只发生在用户授权的桌面/自托管 Gateway。
- Gateway v0.9 已在 `gatewayConnected` 后、任何 `actionStarted` 前写入 session 级 `auditLog` artifact `gateway-capability-snapshot.json`，payload 中包含 workspace、platform、token 指纹、allowedActionKinds、策略 allowlist 和 real/dry-run/disabled/unavailable/workspace-only 能力状态。
- Swift 端当前只显示 action-bound artifacts。`ClawGatewaySession.artifactCount` 只统计 `results.flatMap(\.artifacts)`，`ClawAgentTraceReviewSummary.latest(from:)` 只扫描 action result artifacts，`ClawGatewayEventStream.apply` 对无 action 绑定的 `artifactStored` 会被 `guard let actionID... else { return }` 丢弃。

v0.10 要补齐手机端对 session-level artifact 的复核体验：Gateway 给 capability snapshot artifact 附带安全 metadata，Swift reducer 保存 session-level artifacts，Mission Run 和 Gateway 会话卡片展示一个紧凑、安全、动态类型友好的 Gateway 能力复核摘要。iOS 端只使用 artifact event metadata，不读取 `file://` payload。

## 目标

- Gateway 为 `gateway-capability-snapshot.json` artifact 附带小型安全 `metadata` 字符串字典，供手机端直接展示。
- Swift 模型增加 session-level artifact 存储，例如 `ClawGatewaySession.sessionArtifacts: [ClawGatewayArtifact]`，并保持旧 JSON decode 兼容。
- `ClawGatewaySession.artifactCount` 和 artifact kind summary 必须包含 session-level artifacts。
- `ClawGatewayEventStream.apply` 对没有 `actionID/actionKind/actionTitle` 的 session-level `artifactStored` 不再丢弃，而是合并到 `sessionArtifacts` 并写入 `auditTrail`。
- action-bound artifacts 保持当前 result merge 逻辑，不改变 action result 行为。
- 新增 `ClawGatewayCapabilityReviewSummary` 或等价模型，从 `auditLog` artifact metadata 派生紧凑安全状态。
- `ClawMissionRunSummary` 增加可选 Gateway capability review，并由 `ClawStore.missionRunSummary` 派生。
- UI 在 Mission Run panel 和 Gateway session card 展示该摘要；iPad regular 布局复用同一个 panel/card，自然继承展示。
- Simulator 生成一个 session-level capability artifact metadata，或至少用 XCTest / Swift logic smoke 手工覆盖 reducer 处理 session-level event。
- 补齐 Swift logic smoke、XCTest、Gateway direct/WebSocket smoke metadata 断言，以及 README、协议、flow、flowchart、update_log。

## 非目标

- 不新增 `claw.computer.control.v1` schema 版本。
- 不新增 action kind、artifact kind、Gateway event kind。
- 不修改 `ClawMobileEnvelope` 字段，不把 capability review 写入 envelope。
- 不让 iOS 读取 Gateway `file://` artifact 内容；手机端只能使用 event 上的 `ClawGatewayArtifact.metadata` 和 artifact 基本字段。
- 不新增真实 Shell、浏览器、屏幕、窗口、Accessibility、AppleScript、桌面 App 或文件权限。
- 不改变 Gateway action allowlist、workspace 限制、token 校验或审批闸门。
- 不让 capability snapshot 或 metadata 成为执行计划来源。
- 不把 raw token、Authorization header、用户自然语言命令、action `instruction`、完整 `toolArguments`、网页正文、命令输出、截图内容、草稿正文、联系人或完整 workspace path 放进 metadata。
- 不做完整 artifact JSON viewer、live Gateway 心跳/重连、真实多轮 agent loop 或 UI 大重构。
- 不引入第三方依赖，不下载模型、大数据、历史 artifact 或完整 build cache。

## 实现步骤

1. 开始前同步和检查
   - 执行 `git fetch origin`。
   - 确认当前分支是 `main`，目标远端是 `origin/main`。
   - 执行 `git status --short --branch`。如果存在无关改动，不要覆盖、回滚或混入提交；无法判断归属时暂停说明。

2. 给 Gateway capability snapshot artifact 增加安全 metadata
   - 在 `Tools/claw-gateway-server.mjs` 中新增 helper，例如 `gatewayCapabilitySnapshotMetadata(snapshot)` 或 `gatewayCapabilitySnapshotMetadata(envelope, config, snapshot)`。
   - 建议先生成现有 `gatewayCapabilitySnapshot(...)` payload，再从 payload 派生 metadata，避免 payload 和 metadata 不一致。
   - `metadata` 必须是小型字符串字典，建议仅包含这些键：
     - `snapshotKind`: `gatewayCapability`
     - `tokenConfigured`: `true` / `false`
     - `tokenRequired`: `true` / `false`
     - `tokenFingerprint`: 短 SHA-256 指纹或空字符串
     - `allowedActionKinds`: 逗号分隔的 action kind 列表
     - `workspaceState`: 例如 `workspace-only`
     - `shellState`
     - `browserControlState`
     - `browserNetworkState`
     - `screenCaptureState`
     - `windowMetadataState`
     - `desktopControlState`
     - `safetyFlags`: 逗号分隔，例如 `allowlists-enforced,workspace-only,raw-token-omitted,final-submit-gated`
     - `platform`: `process.platform`
   - 可保留少量等价字段，但必须稳定、短小、安全，便于 Swift smoke/XCTest 断言。
   - 禁止 metadata 包含 raw token、Authorization header、用户自然语言命令、`instruction`、`toolArguments`、网页正文、命令输出、截图内容、草稿正文、联系人、完整 workspace root 或 session workspace path。
   - 写 capability snapshot artifact 时把 metadata 传给 `writeArtifact(...)`：

```js
const snapshot = gatewayCapabilitySnapshot(envelope, config, sessionID, sessionWorkspace);
const capabilitySnapshotArtifact = await writeArtifact(
  "auditLog",
  "gateway-capability-snapshot.json",
  snapshot,
  true,
  sessionConfig,
  gatewayCapabilitySnapshotMetadata(snapshot),
);
```

3. 更新 Gateway direct/WebSocket smoke 的 metadata 断言
   - 在 `Tools/claw-gateway-direct-smoke.mjs` 和 `Tools/claw-gateway-smoke.mjs` 的 `assertCapabilitySnapshot` 中同时检查 `snapshotArtifact.metadata`。
   - 断言 metadata：
     - 存在且是 object。
     - keys 正好是预期 allowlist，或至少不含任何禁止 key；推荐严格 key allowlist。
     - `snapshotKind === "gatewayCapability"`。
     - `tokenConfigured`、`tokenRequired`、`tokenFingerprint` 与 payload 中 `snapshot.token` 一致。
     - `allowedActionKinds` 与 payload/envelope 的 allowlist 一致。
     - `workspaceState` 与 payload `capabilities.workspace.state` 一致。
     - shell/browser/screen/window/desktop state 与 payload `capabilities.*.state` 一致。
     - `platform` 与 payload `gateway.platform` 一致。
     - JSON string 不包含 `smoke-token`，也不包含 `toolArguments`、`Authorization` 等敏感词。
   - direct smoke 覆盖 dry-run、Shell allowlist、desktop policy、browser policy 多组事件；至少对 dry-run 和 allowlist 场景断言 state 变化。
   - WebSocket smoke 覆盖 live path 的 metadata 存在、顺序和安全字段。

4. 扩展 Swift 模型保存 session-level artifacts
   - 在 `ClawGatewaySession` 增加向后兼容字段，例如：

```swift
var sessionArtifacts: [ClawGatewayArtifact]
```

   - `init` 默认 `sessionArtifacts: []`，旧调用点无需全部改动也能编译，或者按需显式传入。
   - 如果 Swift 自动 `Codable` 因新增非默认字段无法兼容旧 JSON，则实现自定义 `init(from:)` / `encode(to:)`，让旧 JSON 缺 `sessionArtifacts` 时默认为 `[]`。
   - `artifactCount` 必须统计 `sessionArtifacts.count + results.map(\.artifacts.count).reduce(0,+)`。
   - 建议增加只读 helper，例如 `allArtifacts` 或 `artifactKinds`，统一返回 `sessionArtifacts + results.flatMap(\.artifacts)`，避免 Store/UI 重复拼接。

5. 新增 capability review 派生模型
   - 在 `Claw/Core/ClawModels.swift` 新增 `ClawGatewayCapabilityReviewSummary` 或等价类型，参考 `ClawAgentTraceReviewSummary`。
   - 建议字段：
     - `snapshotCount: Int`
     - `latestTitle: String`
     - `hasMetadata: Bool`
     - `tokenConfigured: Bool?`
     - `tokenRequired: Bool?`
     - `tokenFingerprint: String?`
     - `allowedActionKinds: [String]`
     - `workspaceState: String?`
     - `shellState: String?`
     - `browserControlState: String?`
     - `browserNetworkState: String?`
     - `screenCaptureState: String?`
     - `windowMetadataState: String?`
     - `desktopControlState: String?`
     - `safetyFlags: [String]`
     - `platform: String?`
     - `isRedacted: Bool`
   - 增加 `compactStatus`，例如：
     - metadata 存在：`Gateway macOS · token sha256:abc123 · shell dry-run · desktop dry-run`
     - metadata 缺失：`已收到 Gateway 能力快照，metadata 待同步。`
   - 增加静态方法：
     - `static func latest(from session: ClawGatewaySession?) -> ClawGatewayCapabilityReviewSummary?`
     - `static func latest(from artifacts: [ClawGatewayArtifact]) -> ClawGatewayCapabilityReviewSummary?`
   - 只选择 `artifact.kind == .auditLog` 且 `artifact.title == "gateway-capability-snapshot.json"` 或 metadata `snapshotKind == "gatewayCapability"` 的 artifact。
   - 解析函数可复用 AgentTrace summary 的 `cleanValue`、`boolValue`、`listValue` 思路；如有重复，可提取小型私有 helper，但不要做无关重构。
   - 不读取 `artifact.reference` 指向的文件内容，不解析 `file://`。

6. 更新 Store reducer 和 Mission Run summary
   - `ClawGatewayEventStream.apply` 的 `.artifactStored` 分支需要区分：
     - 有 `actionID/actionKind/actionTitle`：保持现有 `upsertResult` 行为。
     - 无 action 绑定且 `event.artifacts` 非空：合并到 `session.sessionArtifacts`，并写入 `auditTrail`，例如 `event.<sequence> artifactStored session-level <summary>`。
     - 无 action 绑定且无 artifacts：只写 auditTrail，不创建 result。
   - 合并 session artifacts 时应按 `reference` 去重，最好复用现有 `mergedArtifacts`。
   - `sessionCompleted` 的最终状态仍只基于 action `results`；session-level audit artifact 不应让空 action session 被误判成功/失败。
   - `missionRunArtifactKinds(from:)` 改为扫描 `session.allArtifacts` 或 `session.sessionArtifacts + result artifacts`。
   - `missionRunStatusLine` 完成文案中的 artifact 数自然使用新的 `artifactCount`。
   - `ClawMissionRunSummary` 增加 `gatewayCapabilityReview: ClawGatewayCapabilityReviewSummary?`，并在 `ClawStore.missionRunSummary` 中派生。

7. 更新 simulator 或 reducer fixture
   - 推荐在 `ClawGatewayEventStream.simulatedEvents(...)` 的 `gatewayConnected` 后插入一个 session-level `artifactStored`，模拟 Gateway capability metadata，事件顺序与 live Gateway 一致。
   - 如果担心改变模拟事件计数对旧测试有影响，也可以只在 XCTest/LogicSmoke 中手工创建 session-level event 覆盖 reducer。但本轮目标是手机端正常体验，推荐 simulator 也生成 capability artifact，这样 Mission Run 默认能展示能力复核。
   - Simulator metadata 需与 Gateway metadata 字段同形，但使用安全占位值，例如 `tokenFingerprint` 来自 profile，`allowedActionKinds` 来自 profile allowlist，state 可为 `dry-run` / `workspace-only`。
   - 不要把 raw token 或完整 workspace path 放入 simulator metadata。

8. 更新 SwiftUI 展示
   - 在 `ClawMissionRunPanel` 中，在 AgentTrace review 附近展示 `summary.gatewayCapabilityReview`。
   - 在 `ClawGatewaySessionCard` 中展示 `ClawGatewayCapabilityReviewSummary.latest(from: session)`。
   - 建议新增小型 `ClawGatewayCapabilityReviewRow: View`，复用 `PhoneAgentTag` 和现有 panel/card风格。
   - row 建议展示：
     - 标题 `Gateway 能力`
     - snapshot 数量
     - redacted 状态
     - token configured/required 或 token 指纹短标识
     - shell/browser/desktop/screen/window state chips
     - `compactStatus`
     - safety flags 横向 chips
   - UI 禁止展示 raw path、raw token、Authorization header、`toolArguments`、命令输出、网页正文、截图内容或草稿内容。
   - 文本要 Dynamic Type 友好：使用系统语义字号、`fixedSize(horizontal:false, vertical:true)`、必要时横向 chip `ScrollView`；不要依赖固定字号压缩长文本。
   - iPad regular 工作台无需单独新布局，因为它复用 Mission Run panel 和 Gateway session card。

9. 更新 XCTest
   - 在 `ClawTests/ClawTests.swift` 增加或扩展测试：
     - 旧 `ClawGatewaySession` JSON 缺 `sessionArtifacts` 仍可 decode，`sessionArtifacts == []`。
     - session-level `artifactStored` 无 action 绑定时，`ClawGatewayEventStream.apply` 会保存到 `session.sessionArtifacts`，`artifactCount` 增加，`auditTrail` 有记录，不新增伪 action result。
     - action-bound `artifactStored` 仍进入对应 result，旧逻辑不回归。
     - `ClawGatewayCapabilityReviewSummary.latest(from:)` 能从 metadata 解析 token、allowedActionKinds、workspace/shell/browser/desktop/safety/platform 状态。
     - metadata 缺失时能降级显示，不崩溃。
     - `ClawMissionRunSummary.gatewayCapabilityReview` 在模拟发送或手工 session 中存在，`artifactKinds` 包含 `.auditLog`，`artifactCount` 包含 session-level artifact。
   - 不删除旧 AgentTrace 和 Mission Run 断言。

10. 更新 Swift logic smoke

## 风险

- 给 `ClawGatewaySession` 增加字段可能破坏旧 JSON decode；必须显式覆盖 legacy decode 测试。
- session-level artifacts 如果没有纳入 `artifactCount`，Mission Run 会继续漏掉 Gateway 能力快照。
- reducer 如果给无 action 绑定的 artifact 创建伪 result，会污染 action 成功/失败计数和最终 status。
- metadata key 不稳定会导致 Swift/Gateway smoke 脆弱；要用稳定 allowlist。
- metadata 如果包含完整 workspace path、`toolArguments` 或 payload 摘要，会违反安全边界。
- SwiftUI 如果直接解析 metadata，会把展示逻辑散落在 view 中；要集中在模型/Store 派生层。
- WebSocket smoke 在本机沙箱可能无法监听端口；需要按规则申请升级权限重跑，不能伪装通过。
- 文档遗漏会导致 Agent C 判定不通过，因为本轮改变了手机端 reducer 和展示行为。
