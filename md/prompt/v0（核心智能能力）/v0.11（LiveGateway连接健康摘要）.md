# v0.11（Live Gateway 连接健康摘要）Agent B 实现提示词

## 版本号

- 版本：v0.11
- 阶段：v0（核心智能能力）
- 任务名：Live Gateway 连接健康摘要
- 提交信息格式：`v0.11: Live Gateway连接健康摘要`

## 版本分配依据

v0.9 已让桌面 Gateway 在 session 开始时写入 `gateway-capability-snapshot.json` `auditLog` artifact，v0.10 已让手机端保存 session-level artifacts，并在 Mission Run / Gateway session / iPad regular 工作台展示 Gateway 能力复核摘要。

当前 `update_log.md` 和 `md/flow/flow.md` 的主要遗留项仍包含 live Gateway 心跳和重连。v0.11 不直接做自动重连或心跳协议，而是先补第一步可观测性：从已有 `ClawGatewayLiveRequest`、`gatewayConnectionState`、`gatewayEvents` 和最新 `ClawGatewaySession` 派生一个安全的 live 连接健康摘要，让 iPhone/iPad 能看清 preflight、是否可尝试 live、连接状态、事件进度、最新事件、session 状态和 fallback/error/completed 结果。

## 背景

Claw 当前是 SwiftUI iPhone/iPad 控制台 + 用户自托管桌面 Gateway：

- 手机端负责输入任务、生成计划、展示风险、审批、发送 envelope 和查看事件。
- 桌面 Gateway 负责真实或 dry-run 的浏览器、文件、Shell、屏幕、桌面 App 和 agent loop 动作。
- `ClawGatewayLiveRequest` 已记录 endpoint、transport、requestPath、tokenFingerprint、headers、bodyBytes、taskID、command、actionCount、canAttemptLive 和 preflightMessage；它不保存 raw token 到 envelope。
- `sendLatestClawMobileTaskOverLiveGateway()` 已通过 `URLSessionClawGatewayTransport` 发送 WebSocket envelope，并按 `ClawGatewayEvent` stream 更新 session。
- `ClawGatewaySessionPanel` 已展示 `lastGatewayEvent`、`ClawGatewayLiveRequestCard`、`ClawGatewaySessionCard` 和事件时间线；iPad regular 布局复用该 panel。

当前缺口：

- 手机端没有一个稳定的 live health 摘要，用户只能从散落的 request card、connection tag、lastGatewayEvent 和事件列表推断连接状态。
- `ingestGatewayEvents(_:)` 在 live 模式收到非终态事件时会把 `gatewayConnectionState` 设回 `.awaitingGateway`，会覆盖 `sendLatestClawMobileTaskOverLiveGateway()` 中进入事件流后的 `.streaming`。
- `URLSessionClawGatewayTransport` 当前本地先生成一个 `gatewayConnected`，之后桌面 Gateway 也可能推送 `gatewayConnected`；v0.11 需要让这种兼容路径在摘要中清晰可解释，或改成更清晰的现有事件语义，但不能新增 event kind。

## 目标

- 新增 `ClawGatewayLiveHealthSummary` 或等价派生模型，用于安全展示 live Gateway 连接健康状态。
- `ClawStore` 提供 computed summary，例如 `gatewayLiveHealthSummary`，只从现有 state 派生，不新增持久 source of truth。
- 摘要至少包含：
  - endpoint、transport、requestPath 和短 token fingerprint。
  - preflight 文案。
  - `canAttemptLive`。
  - 当前 `ClawGatewayConnectionState` 和标题。
  - 当前 session status。
  - 最新 session 相关事件数量。
  - 最新事件的 `sequence`、`kind`、`createdAt` 和安全摘要。
  - 是否 fallback、是否 error/failed、是否 completed。
  - 一个适合 UI 的 `compactStatus` / `detailLine` / chips 派生文案。
- 修正 live 状态语义：live 模式收到非终态 Gateway 事件后应保持或设置 `.streaming`，不能降回 `.awaitingGateway`；`.sessionCompleted`、`.fallbackUsed`、transport error / failed 仍按现有终态路径处理。
- UI 在 Gateway 会话面板、Live request 附近展示连接健康摘要；iPad regular 工作台通过复用 `ClawGatewaySessionPanel` 自然覆盖。
- 补 XCTest 和 Swift logic smoke 覆盖 health summary、状态语义、fallback/error/completed 和旧路径不回归。
- 同步 README、协议、flow、flowchart、update_log。

## 非目标

- 不新增 Gateway event kind。
- 不新增 action kind 或 artifact kind。
- 不改变 `claw.computer.control.v1` schema。
- 不做真实自动重连、心跳协议、后台保活或配对流程。
- 不增加桌面 Gateway 的 Shell、浏览器、屏幕、窗口、Accessibility、AppleScript、桌面 App 或文件权限。
- 不修改 action allowlist、workspace 限制、token 校验或审批闸门。
- 不把 live health 写入 envelope。
- 不让 health summary 成为执行计划来源。
- 不读取 Gateway `file://` artifact payload。
- 不显示 raw token、Authorization header、完整请求 header、完整 workspace path、`toolArguments`、命令输出、网页正文、截图内容、草稿正文或联系人。
- 不引入第三方依赖，不下载模型、大数据、历史 artifact 或完整 build cache。

## 实现步骤

1. 开始前同步和检查
   - 执行 `git fetch origin`。
   - 确认当前分支是 `main`，目标远端是 `origin/main`。
   - 执行 `git status --short --branch`。如存在无关改动，不要覆盖、回滚或混入提交；无法判断归属时暂停说明。

2. 新增 live health 派生模型
   - 在 `Claw/Core/ClawModels.swift` 新增 `ClawGatewayLiveHealthSummary` 或等价类型，建议 `Equatable`、`Codable`、`Sendable`。
   - 建议字段：
     - `endpoint: String`
     - `transport: String`
     - `requestPath: String`
     - `tokenFingerprint: String?`
     - `preflightMessage: String`
     - `canAttemptLive: Bool`
     - `connectionState: ClawGatewayConnectionState`
     - `sessionStatus: ClawGatewaySessionStatus?`
     - `eventCount: Int`
     - `latestEventSequence: Int?`
     - `latestEventKind: ClawGatewayEventKind?`
     - `latestEventAt: Date?`
     - `latestEventSummary: String?`
     - `hasFallback: Bool`
     - `hasError: Bool`
     - `isCompleted: Bool`
     - `compactStatus: String`
   - 可根据实现需要增加 `statusChips` 等安全派生字段，但不要保存 raw token/header。
   - `latestEventSummary` 可以使用已有 `event.summary` 的短安全摘要；不要拼接 raw token、Authorization header 或 artifact payload。
   - 缺少 `lastGatewayLiveRequest` 时要能降级为“尚未准备 Live Gateway 请求”，而不是返回 nil 后让 UI 空白。

3. 在 Store 派生 summary
   - 在 `ClawStore` 增加 computed property，例如：
     - `var gatewayLiveHealthSummary: ClawGatewayLiveHealthSummary { ... }`
   - 输入只使用：
     - `lastGatewayLiveRequest`
     - `gatewayConnectionState`
     - `gatewayEvents`
     - `clawGatewaySessions.first`
   - 事件计数建议限定到最新 session：`gatewayEvents.filter { $0.sessionID == latestSession.id }`。如果没有 session，可使用空数组或最近 request 对应 task 的事件。
   - latest event 选择应以 `createdAt` 为主、`sequence` 为辅，避免 `URLSessionClawGatewayTransport` 本地 `gatewayConnected` 和桌面 Gateway `gatewayConnected` sequence 不同导致摘要倒退。
   - `hasFallback` 可来自 latest-session events 中是否包含 `.fallbackUsed` 或当前 state 是否 `.fallbackSimulated`。
   - `hasError` 可来自 `gatewayConnectionState == .failed`、session status `.blocked`、或 latest-session events 中 `.actionFailed`。
   - `isCompleted` 可来自 `gatewayConnectionState == .completed` 或 session status `.completed`。

4. 修正 live connection state 语义
   - 更新 `ingestGatewayEvents(_:)` 中对 `gatewayConnectionState` 的处理。
   - 预期行为：
     - `.sessionCompleted`：保持现有逻辑，`latest.status == .blocked ? .failed : .completed`。
     - `.fallbackUsed`：保持 `.fallbackSimulated`。
     - live request 刚准备、尚未有 Gateway 非终态事件时，可为 `.awaitingGateway`。
     - live 模式收到 `.gatewayConnected`、`.actionStarted`、`.artifactStored`、`.actionCompleted`、`.actionFailed`、`.approvalRequested`、`.actionSkipped` 等非终态事件时，应保持/设置 `.streaming`，不能降回 `.awaitingGateway`。
   - 注意 fallback-to-simulation 路径：缺 token 时仍应安全回退并最终让现有 fallback/completed 行为和测试清晰表达；health summary 要能显示 `hasFallback == true`。
   - 不要为了状态修正改变 `ClawGatewaySessionStatus` 的 reducer final status 语义。

5. 处理 `URLSessionClawGatewayTransport` 的本地 `gatewayConnected` 语义
   - Agent B 需要明确判断并实现其中一种方案：
     - 保留当前兼容行为：本地 `.gatewayConnected` 代表“客户端 WebSocket 已打开并已发送 envelope”，桌面 Gateway 随后可能再推 `.gatewayConnected`；更新 summary 文案和测试，确保 health summary latest/eventCount 能正确解释重复 gatewayConnected。
     - 或改为更清晰的 client-open audit：仍只能使用现有 `ClawGatewayEventKind`，不能新增 event kind；如改动 yield 行为，必须更新 live transport tests 和文档。
   - 推荐保守路径是保留本地事件但把 summary 文案改清楚，例如“Live Gateway WebSocket 已打开并发送 envelope，等待桌面端事件。”；不要声称桌面端已完成认证，除非真实 Gateway 事件已经到达。
   - 不要新增 schema/event/action/artifact kind。

6. 更新 SwiftUI 展示
   - 在 `ClawGatewaySessionPanel` 中，在 `ClawGatewayLiveRequestCard` 附近展示 live health summary。

- 本轮只做手机端/iPad 可观测性和 live 状态语义修正。
- Health summary 从 `ClawGatewayLiveRequest`、`ClawGatewayConnectionState`、`gatewayEvents` 和最新 `ClawGatewaySession` 派生。
- Health summary 展示 preflight、是否可尝试 live、连接状态、事件数量、latest sequence/kind/time/summary、session status、fallback/error/completed。
- UI 只显示短 token fingerprint、endpoint、状态和事件健康摘要。
- 不显示 raw token、Authorization header、完整 headers、完整 workspace path 或 Gateway `file://` payload。
- 不新增 Gateway event kind，不新增 action/artifact kind，不改变 `claw.computer.control.v1`。
- 不做真实自动重连、心跳协议或新增桌面权限。
- live Gateway 心跳、重连和配对仍是后续扩展点。

## 验收标准

Agent C 应按以下标准验收：

- `origin/main` 最新 commit 包含 v0.11 实现和文档更新，提交范围只含本轮相关文件。
- `ClawGatewayLiveHealthSummary` 或等价模型存在，并从 Store 现有状态派生，不成为新的 source of truth。
- Summary 覆盖 preflight、canAttemptLive、connection state、event count、latest event sequence/kind/time/summary、session status、fallback/error/completed。
- `ingestGatewayEvents(_:)` 在 live 模式收到非终态事件后保持/设置 `.streaming`，不降回 `.awaitingGateway`。
- `.sessionCompleted`、`.fallbackUsed` 和 transport error / failed 仍按现有终态路径处理。
- `URLSessionClawGatewayTransport` 的本地 `gatewayConnected` 与桌面 Gateway `gatewayConnected` 兼容路径清晰，测试覆盖其事件计数/latest event 行为；如改动 yield 语义，未新增 event kind 且文档同步。
- Gateway 会话面板/Live request 附近展示 health summary；iPad regular 工作台自然可见。
- UI 不显示 raw token、Authorization header、完整 headers、`file://` payload、`toolArguments`、命令输出、网页正文、截图内容、草稿正文或联系人。
- XCTest 覆盖 idle/preflight、缺 token fallback、非终态 streaming、completed、error/failure 和重复/兼容 gatewayConnected 行为。
- Swift logic smoke 覆盖 live health summary 关键路径。
- README、协议、flow、flowchart、update_log 已同步。
- 本地轻量检查按触发条件运行并记录结果。
- GitHub Actions 最新 run 通过；Agent C 下载 `/private/tmp/claw-c-review-<run_id>/` 下最新未加密 artifact，核对 manifest/JUnit/日志/关键结果文件后通过。

## 风险

- 如果只新增 UI 而不修正 `ingestGatewayEvents`，live 流式事件仍会把状态降回“等待网关”，健康摘要会误导用户。
- 如果 health summary 直接扫描全局 `gatewayEvents` 而不限定最新 session，旧 session 事件可能污染当前连接摘要。
- `URLSessionClawGatewayTransport` 本地 `gatewayConnected` 和桌面 `gatewayConnected` 可能 sequence 不连续；latest event 选择要避免倒退。
- 如果 UI 展示 `request.headers`，可能泄露 Authorization header；只能展示短 token fingerprint 和安全字段。
- 如果把 health summary 写入 envelope 或协议字段，会扩大 schema 范围，违反本轮非目标。
- 如果把 fallback simulation 的 terminal completed 与 real live completed 混在一起，用户会误以为真实 Gateway 已执行；summary 应通过 `hasFallback` 清楚标记回退。
- SwiftUI 若把状态派生逻辑塞入 `body`，会增加维护成本并削弱测试覆盖；解析和状态文案应集中在模型/Store。
- 文档遗漏会导致 Agent C 判定不通过，因为本轮改变了 live 状态语义和手机端展示行为。
