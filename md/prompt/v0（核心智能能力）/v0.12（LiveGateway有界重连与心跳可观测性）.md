# v0.12（Live Gateway 有界重连与心跳可观测性）Agent B 实现提示词

## 版本号

- 版本：v0.12
- 阶段：v0（核心智能能力）
- 任务名：Live Gateway 有界重连与心跳可观测性
- 提交信息格式：`v0.12: Live Gateway有界重连与心跳可观测性`

## 版本分配依据

v0.11 已新增 `ClawGatewayLiveHealthSummary`，让 iPhone/iPad 能看到 Live Gateway preflight、连接状态、事件数量、最新事件、fallback/error/completed 和脱敏 endpoint/token 指纹。

当前主要遗留仍包含 live Gateway 心跳和重连。v0.12 不做后台保活、不新增桌面协议和不改变 Gateway 权限，而是先在手机端 live transport 路径补一个可测试的最小闭环：WebSocket 打开后做一次 ping 观测，transport 错误时按固定上限重试一次，并把尝试次数、ping 成功/失败和重连状态安全地展示到 health summary。

## 背景

当前架构：

- `ClawGatewayLiveRequest` 保存 endpoint、transport、requestPath、tokenFingerprint、headers、bodyBytes、taskID、command、actionCount、canAttemptLive 和 preflightMessage；它不保存 raw token 到 envelope。
- `URLSessionClawGatewayTransport.streamEvents(...)` 目前只做一次 WebSocket 连接、发送 envelope、本地 yield 一个 `.gatewayConnected`，然后读取桌面 Gateway 事件；抛错后由 `ClawStore.sendLatestClawMobileTaskOverLiveGateway(...)` fallback 到模拟事件流。
- `ClawGatewayLiveHealthSummary` 目前只从 request、connection state、events 和最新 session 派生连接健康；还没有重连次数、ping 结果或最后一次 transport 错误摘要。
- `ClawGatewaySessionPanel` 已在 Live request 附近展示 health row；iPad regular 工作台复用同一 panel。

本轮要补的是连接可用性，而不是扩大执行能力。

## 目标

- 为 live transport 增加有界重连配置，默认最多重试 1 次，总尝试次数最多 2 次。
- WebSocket resume 并发送 envelope 后做一次 ping 观测；ping 成功/失败只用于手机端 health summary 和 audit 文案，不作为执行授权依据。
- transport 每次尝试 yield 一个现有 `.gatewayConnected` 进度事件，summary 必须能区分：
  - 第 1 次连接打开并发送 envelope。
  - 第 N 次重连打开并重新发送 envelope。
  - ping 成功或 ping 失败。
- 不新增 `ClawGatewayEventKind`。本轮复用 `.gatewayConnected` 作为客户端连接/重连进度事件，桌面 Gateway 仍可推自己的 `.gatewayConnected`。
- `ClawGatewayLiveHealthSummary` 新增或等价派生：
  - `transportAttemptCount`
  - `reconnectCount`
  - `lastPingSucceeded`
  - `lastTransportErrorSummary`
  - `isReconnecting` 或 `hasReconnectAttempt`
  - 适合 UI 的 compact/detail/chip 文案
- `ClawStore` 的 `gatewayLiveHealthSummary` 仍只从现有 state 派生，不新增持久 source of truth；如需要额外输入，优先从 session 事件 summary 安全解析。
- transport 最终仍失败时，保持既有 fallback simulation 行为，并让 health summary 同时显示 fallback 和 transport error，不显示 raw token、Authorization、完整 path、headers、payload 或 `toolArguments`。
- UI 在既有 `ClawGatewayLiveHealthRow` chip 中显示重连次数、ping 状态和最后错误摘要的安全短文本；iPad regular 自然覆盖。
- 补 XCTest 和 Swift logic smoke 覆盖重连、ping、最终 fallback 和敏感内容不泄露。
- 同步 README、协议、flow、flowchart、update_log。

## 非目标

- 不新增 Gateway event kind、action kind 或 artifact kind。
- 不改变 `claw.computer.control.v1` schema。
- 不要求桌面 Gateway 新增业务协议、心跳消息、后台保活、配对服务或长轮询。
- 不在 iOS 后台静默保持 WebSocket。
- 不扩大 Shell、浏览器、屏幕、窗口、Accessibility、AppleScript、桌面 App、文件或网络权限。
- 不修改 action allowlist、workspace 限制、token 校验或审批闸门。
- 不把自然语言直接交给 Shell、AppleScript、浏览器或桌面自动化。
- 不把 live health、ping 或 retry 信息写入 envelope。
- 不读取 Gateway `file://` artifact payload。
- 不显示 raw token、Authorization header、完整 headers、完整 workspace path、`toolArguments`、命令输出、网页正文、截图内容、草稿正文或联系人。
- 不引入第三方依赖，不下载模型、大数据、历史 artifact 或完整 build cache。

## 实现步骤

1. 开始前同步和检查
   - `git fetch origin`
   - 确认当前分支是 `main`，目标远端是 `origin/main`。
   - `git status --short --branch` 必须无无关改动；无法判断归属时暂停。

2. 增加 transport retry 配置
   - 在 `Claw/Services/ClawStore.swift` 中新增轻量配置类型，例如 `ClawGatewayTransportRetryPolicy: Equatable, Sendable`。
   - 字段建议：
     - `maxRetries: Int`
     - `retryDelayNanoseconds: UInt64`
     - `pingAfterConnect: Bool`
   - 默认策略：`maxRetries = 1`、短 delay、`pingAfterConnect = true`。
   - 不引入 timers/source of truth 到 UI；transport 内部使用。

3. 更新 `URLSessionClawGatewayTransport`
   - 支持初始化注入 retry policy，保持默认构造不破坏调用点。
   - `streamEvents(...)` 内部按 attempt 循环：
     - 创建 WebSocket。
     - 设置 headers。
     - resume。
     - send envelope。
     - 如 `pingAfterConnect`，调用 `socket.sendPing` 的 async 包装。
     - yield `.gatewayConnected`，summary 包含 `attempt=N`、`reconnect=M`、`ping=ok/failed/skipped`，不得包含 raw token/header/envelope。
     - 读取桌面事件直到 `.sessionCompleted`。
   - 若某次 attempt 在收到任何桌面 Gateway 业务事件前抛错：
     - 如果还有 retry quota，yield 一个 `.gatewayConnected` 或安全 progress event summary，说明将进行第 N 次重连；随后重试。
     - 如果没有 retry quota，throw 最后错误，由 Store 走既有 fallback。
   - 注意不要让同一个失败 attempt 生成 `sessionCompleted`；最终 fallback 仍由 Store 产生。
   - 避免递归和无限循环；总尝试次数固定。

4. 更新 health summary
   - 在 `ClawGatewayLiveHealthSummary` 增加字段或等价派生：
     - `transportAttemptCount`
     - `reconnectCount`
     - `lastPingSucceeded`
     - `lastTransportErrorSummary`
     - `hasReconnectAttempt`
   - 从 session events 的 `.gatewayConnected` summary 中安全解析 `attempt=...`、`reconnect=...`、`ping=...`、`transportError=...` 等键值。
   - 解析函数必须容错旧事件、桌面 Gateway 自己的 `.gatewayConnected` 和无 metadata 情况。
   - `safeSummary` 继续遮蔽 Authorization、Bearer、file URL、workspace、local path；新增对 `token=...`、`password=...`、`secret=...`、`headers=...` 等 query/键值的遮蔽。
   - `compactStatus`/`detailLine` 和 chips 要能表达：
     - “已重连 1 次”
     - “ping 正常”
     - “ping 失败但仍等待事件”
     - “连接失败后回退模拟”

5. 更新 UI
   - 在 `ClawGatewayLiveHealthRow.chips` 增加：
     - 重连次数 chip。
     - ping 状态 chip。
     - 最后 transport error chip，文本必须短且脱敏。
   - 不显示 raw endpoint query、headers、token、payload。
   - 不新增复杂交互，不新增设置页。

## 验收标准

- 有界重连最多按策略执行，默认最多重试 1 次，不会无限循环。
- transport progress 使用现有 `.gatewayConnected`，不新增 event kind。
- health summary 能显示 attempt/reconnect/ping/error/fallback，且旧事件可降级。
- ping/retry 信息只用于 UI 复核和审计文案，不进入 envelope，不成为执行计划来源。
- 任何 summary/detail/chip 不泄露 raw token、Authorization、headers、workspace path、artifact payload、`toolArguments` 或自然语言转执行内容。
- v0.11 的 live health、fallback、completed、streaming 测试继续通过。
- 本地轻量检查通过，云端 artifact 经 Agent C 复判通过。

## 风险

- `URLSessionWebSocketTask.sendPing` 是平台 API；本地 logic smoke 不直接覆盖真实 socket，需要用云端 build 和 mock/summary parsing 保证编译与逻辑。
- 重连后桌面端可能已经收到第一次 envelope；本轮只在第一次连接未收到任何桌面 Gateway 业务事件前做客户端重试提示，仍可能产生桌面端重复任务风险。因此 UI/summary 必须标记重连，后续需要 server-side task idempotency。
- 如果网络错误文案包含 endpoint query 或系统路径，必须通过 `safeSummary` 遮蔽。
- 本轮不是后台保活；App 挂起后 iOS 仍可能断开 WebSocket。

## 禁止项

- 禁止新增或伪造桌面 Gateway 权限。
- 禁止自然语言直连 Shell、AppleScript 或真实桌面动作。
- 禁止把旧 artifact、旧 output 或本地输出冒充最新云端结果。
- 禁止无 Agent C artifact 复判时宣布本轮完成。
