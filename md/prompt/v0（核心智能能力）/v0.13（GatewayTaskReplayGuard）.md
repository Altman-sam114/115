# v0.13（Gateway task replay guard / server-side idempotency）Agent B 实现提示词

## 版本

- 版本：v0.13
- 阶段：v0（核心智能能力）
- 基线：当前 `main`，HEAD `42392bedde8b3e305bc949a988ba217c5ea979dd`
- 提交信息：`v0.13: Gateway task replay guard`
- 版本依据：v0.12 已实现 Live Gateway 有界重连与 ping 可观测性，但遗留 server-side task idempotency；本轮补 Gateway 端 replay guard，降低同一 envelope 被 live retry 重复执行的风险。

## 背景

v0.12 的客户端 transport 只在尚未收到桌面 Gateway 业务事件前重发 envelope，并把 attempt/reconnect/ping 写入健康摘要。该策略降低了客户端重复发送风险，但桌面 Gateway 仍需要自己的任务重放防护，避免同一 `task.id` 在同一 Gateway 进程生命周期内再次触发文件、Shell、浏览器或桌面 App handler。

## 目标

- 在 `Tools/claw-gateway-server.mjs` 增加 Gateway 进程内 task replay guard。
- 对同一 envelope/task 的重复提交做 server-side 去重，不再次执行 action handlers，不重复写业务 artifact，不重复执行 Shell/文件/浏览器/桌面 App 动作。
- 重复 envelope 仍返回可审计事件流：复用既有 `auditLog` artifact、`artifactStored`、`actionSkipped`、`sessionCompleted`。
- replay audit payload/metadata 只能包含安全摘要：task id、短 envelope/task digest、首次处理 session id、replay count、action count/kinds、原因；不得包含 raw token、Authorization、自然语言 instruction、`toolArguments`、命令输出、网页正文、截图内容、草稿正文或完整 workspace path。
- Direct smoke 和 WebSocket smoke 覆盖重复发送同一 envelope 不会重复执行业务动作。
- 同步更新 `README.md`、`Docs/claw-mobile-gateway-protocol.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md`。
- Swift 默认不改；除非发现 reducer 无法消费既有 `actionSkipped/sessionCompleted`，否则不要改 Swift。

## 非目标

- 不新增 event kind、action kind、artifact kind。
- 不改变 `claw.computer.control.v1` schema。
- 不新增 Gateway 权限，不扩大 Shell/browser/screen/window/desktop/file/network 能力。
- 不做持久化 idempotency 数据库、跨进程/重启去重、配对服务、后台保活或真实心跳协议。
- 不把自然语言直接交给 Shell、AppleScript 或桌面自动化。
- 不删除 smoke 断言，不绕过 allowlist、workspace、token 或审批闸门。

## 当前架构依据

- `Tools/claw-gateway-server.mjs` 的 WebSocket 和 `--emit-events` 入口都调用 `makeGatewayEvents(envelope, options)`。
- `makeGatewayEvents` 目前每次新建 session workspace，随后写 `gateway-capability-snapshot.json`，再逐个 action 调用 `runAction`。
- `ClawGatewayEvent` 已有 `actionSkipped`，`ClawGatewayArtifactKind` 已有 `auditLog`；手机端 reducer 已能消费 session-level `artifactStored` 和 skipped action result。
- `Tools/claw-gateway-smoke.mjs` 覆盖 WebSocket server path，`Tools/claw-gateway-direct-smoke.mjs` 覆盖 `--emit-events` direct path。

## 实现步骤

1. 在 Gateway options/config 上挂进程内 replay cache，例如 `taskReplayCache: new Map()`，并设置小上限避免无界增长。
2. `validateEnvelope()` 补充 `task.id` 非空字符串检查，避免缺失 id 的 envelope 撞同一 replay key。
3. 构建安全 replay key/digest：使用 `schemaVersion`、`task.id`、`gateway.tokenFingerprint`、排序后的 action ids/kinds；不要保存完整 envelope，不包含 `createdAt`、`instruction`、`toolArguments` 或 raw token。
4. 首次处理时先登记 replay record，再正常执行原有 Gateway 路径；执行完成后记录 status、event count、最终 sequence 和 completedAt。
5. 后续命中同一 key 时，新建 replay session，返回 `gatewayConnected`、session-level `task-replay-guard.json` `auditLog`、每个 action 的 `actionSkipped` 和 `sessionCompleted`。
6. replay path 不得调用 `runAction`，不得写 `browserTrace/fileDiff/commandOutput/agentTrace/screenshot/messageDraft` 等业务 artifact。
7. `--emit-events` 和 WebSocket live path 必须走同一 replay guard。
8. 更新 direct/WebSocket smoke：重复发送同一 envelope 后断言 replay 事件只包含现有 kind，业务 artifact 没有翻倍，replay auditLog 脱敏且 actionSkipped 数量等于 action 数。
9. 更新 README、协议、flow、flowchart 和 update_log，说明这是进程内 server-side idempotency，不是跨重启 exactly-once。

## 风险和禁止项

- 进程内 Map 只能防同一 Gateway 进程生命周期内的重复提交；重启后不会保留。
- digest/canonicalization 若包含非稳定字段会误判；必须避开 `createdAt` 等时间字段。
- replay path 若误调用 `runAction` 会重新写文件或执行允许的 Shell/桌面动作，这是本轮核心风险。
- WebSocket `--once` 现有测试结构可能需要小幅调整，避免为了测试破坏生产关闭语义。
- 禁止新增 Gateway 权限、删除 smoke 断言、伪造测试通过、泄露 token/header/toolArguments/instruction、或把 replay guard 描述成跨进程 exactly-once。
