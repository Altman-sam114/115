# v0.14（手机端 Replay Guard 复核摘要）Agent B 实现提示词

## 版本

- 版本：v0.14
- 阶段：v0（核心智能能力）
- 基线：`main` / `origin/main`，HEAD `d6a57c60bf5d5c291b42a6036a3bd83d683a88d8`
- 提交信息：`v0.14: 手机端Replay Guard复核摘要`
- 版本依据：v0.13 已在桌面 Gateway 端实现进程内 task replay guard，并通过 `task-replay-guard.json` `auditLog` metadata 回传安全摘要；手机/iPad presentation layer 仍没有专门复核入口。本轮把 replay guard 结果接到 Mission Run 和 Gateway session 复核界面。

## 背景

当前手机端已能从 `agentTrace` 和 `gateway-capability-snapshot.json` 的 artifact metadata 派生安全复核摘要，但 v0.13 的 `task-replay-guard.json` 仍只表现为普通 `auditLog`。用户在 iPad/宽屏工作台复核时，需要直接看到“重复任务已被进程内 replay guard 跳过、未重新执行 handler”的结论，而不是打开或读取 Gateway `file://` payload。

## 目标

- 新增 Swift presentation model `ClawGatewayTaskReplayGuardReviewSummary`，只解析 artifact metadata。
- 在 `ClawMissionRunSummary` 中加入 replay guard 复核摘要。
- 在 Mission Run 面板和 Gateway session card 中展示 replay guard 复核 row。
- XCTest 和 Swift logic smoke 覆盖 metadata 解析、legacy fallback、session-level artifact 合并、Mission Run 派生和敏感内容不泄露。
- 同步 README、协议、flow、flowchart、update_log。

## 非目标

- 不新增 `ClawGatewayEventKind`、`ClawGatewayArtifactKind`、action kind 或 schema 字段。
- 不修改 Gateway JS replay guard 行为，不扩大 Shell/browser/screen/window/desktop/file/network 权限。
- 不读取 Gateway `file://` artifact payload，不展示 raw token、Authorization、`instruction`、`toolArguments`、完整 workspace path 或业务 artifact 内容。
- 不把 replay guard 描述成跨进程、跨重启或分布式 exactly-once。
- 不重构 SwiftUI 大文件结构；只做本轮必要的局部展示。

## 当前架构依据

- `ClawGatewayArtifact.metadata` 是 `[String: String]?`，已用于 AgentTrace 和 Gateway capability review。
- `ClawGatewaySession.allArtifacts` 已包含 `sessionArtifacts + action artifacts`。
- Reducer 已把无 action 绑定的 `artifactStored` 存入 `sessionArtifacts`，不创建伪 action result。
- v0.13 Gateway replay artifact metadata 键包括：`replayGuard`、`decision`、`taskID`、`replayDigest`、`digestMatchesFirst`、`firstSessionID`、`originalStatus`、`replayCount`、`actionCount`、`actionKinds`、`safetyFlags`。
- Mission Run 和 Gateway session card 已展示 `ClawGatewayCapabilityReviewRow` 与 `ClawAgentTraceReviewRow`。

## 实现步骤

1. 在 `Claw/Core/ClawModels.swift` 中新增 `ClawGatewayTaskReplayGuardReviewSummary`：
   - 过滤 `artifact.kind == .auditLog` 且 `artifact.title == "task-replay-guard.json"` 或 `metadata["replayGuard"] == "taskReplayGuard"`。
   - 解析 metadata 字段，使用现有 `ClawArtifactMetadataParser`。
   - 提供 `compactStatus`，强调进程内 replay guard、重复任务跳过和原始状态。
   - 提供短显示字段，例如短 task id / digest / first session id，避免 UI 过宽。
2. 给 `ClawMissionRunSummary` 增加可选字段 `gatewayTaskReplayGuardReview`。
3. 在 `Claw/Services/ClawStore.swift` 的 `missionRunSummary` 中派生 replay guard review。
4. 在 `Claw/Views/ContentView.swift` 新增 `ClawGatewayTaskReplayGuardReviewRow`，复用现有 row/chip 风格：
   - Mission Run 面板展示在 Gateway 能力摘要之后、AgentTrace 摘要之前。
   - Gateway session card 同样展示。
   - chip 展示 decision、digest match、replay/action count、original status、action kinds、safety flags。
5. 在 `ClawTests/ClawTests.swift` 增加：
   - replay guard metadata 解析测试。
   - legacy/fallback artifact 测试。
   - session-level replay audit artifact 不创建 fake result。
   - Mission Run summary 能派生 replay guard review。
   - 摘要字段不含 token/header/toolArguments/file path。
6. 在 `Tools/LogicSmoke.swift` 增加同类轻量断言。
7. 更新 README、协议、flow、flowchart、update_log。

## 验收标准

- Mission Run 和 Gateway session card 能展示 replay guard 摘要。
- 摘要只从 metadata 派生，不读取 Gateway `file://` payload。
- replay guard row 不显示 raw token、Authorization、`instruction`、`toolArguments`、完整 workspace path 或业务 artifact 内容。
- 旧 artifact 缺 metadata 时仍可降级显示“metadata 待同步”。
- Swift logic smoke 和云端 CI 通过，Agent C artifact 复判通过。

## 风险和禁止项

- replay guard 只能说明同一 Gateway 进程内重复任务被跳过，不能表述为全局 exactly-once。
- metadata 中的 UUID/digest 应短显，避免 iPad/compact UI 过宽。
- 禁止新增协议种类或读取 artifact payload 来补充展示。
- 禁止修改 Gateway 权限、绕过审批、删除测试断言或伪造 CI 结果。
