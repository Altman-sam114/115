# v0.15（macOS Accessibility 观察摘要）Agent B 实现提示词

## 版本

- 版本：v0.15
- 阶段：v0（核心智能能力）
- 基线：`main` / `origin/main`，HEAD `e68d8a88c8a8bb8af75f0d9b1725bf4a00c8ef81`
- 提交信息：`v0.15: macOS Accessibility观察摘要`
- 版本依据：v0.14 已完成手机端 Replay Guard 复核摘要；当前遗留项包含完整 macOS Accessibility tree。本轮只推进 Gateway 观察层的安全摘要和可观测能力状态，不新增 action/event/artifact kind。

## 背景

当前 `observeScreen` 已通过结构化 `toolArguments` 生成 `screenshot` JSON 观察和既有 `accessibilityTree` artifact，但真实内容主要来自 dry-run/window metadata。需要在桌面 Gateway 上补一个受显式 env 开关控制的 macOS Accessibility 观察摘要：默认仍 dry-run/metadata-only；启用后只采集前台 App/窗口和有限候选控件摘要，用于观察和审计，不做点击、输入或自动执行。

## 目标

- 在 `Tools/claw-gateway-server.mjs` 的 `observeScreen` 路径中加入 macOS Accessibility 观察摘要。
- 新增显式环境开关 `CLAW_ALLOW_ACCESSIBILITY_OBSERVE=1`；默认不开启时仍为 dry-run/metadata-only。
- 真实读取必须只由结构化 `toolArguments` 控制，例如 `includeAccessibilityTree`、`maxCandidateControls`、`includeWindowTitles`、`redaction`，不得使用自然语言 instruction 驱动 AppleScript/AX。
- 复用现有 `observeScreen` action 和既有 `accessibilityTree` / `screenshot` / `auditLog` artifact，不新增 action/event/artifact kind。
- capability snapshot 增加 `accessibilityTreeState` 和安全 metadata，供 Agent C/手机端审计。
- 手机端 Gateway 能力复核摘要展示 Accessibility tree 状态。
- 更新 direct/WebSocket smoke、本地 node check、README、协议、flow、flowchart、update_log。

## 非目标

- 不新增 action kind、event kind、artifact kind 或 envelope schema 字段。
- 不实现点击、输入、表单提交、控件 press、桌面 App 操作或浏览器控制。
- 不让自然语言 instruction 直连 AppleScript、Shell、System Events 或 Accessibility。
- 不让 iOS 描述为能读取其他 App 私有数据；真实观察只发生在用户授权的桌面 Gateway。
- 不读取或上传完整屏幕内容、raw token、Authorization header、完整 workspace path、网页正文、命令输出、草稿正文。
- 不引入第三方依赖或 native Swift/ObjC bridge；本轮保持 Node 原型内的轻量 macOS 命令/脚本实现。

## 当前架构依据

- `ClawMobileActionKind.observeScreen` 和 `ClawGatewayArtifactKind.accessibilityTree` 已存在。
- `ClawMobileBridge.toolArguments(.observeScreen)` 已生成 `includeAccessibilityTree=true`、`includeWindowTitles=true`、`maxCandidateControls`、`redaction=maskSensitiveText`。
- `observeScreenAction` 当前写 `screen-observation-N.json` 和 `accessibility-tree-N.json`。
- `collectWindowMetadata` 已受 `CLAW_ALLOW_WINDOW_METADATA` 控制；真实路径使用结构化策略，不应扩大为执行动作。
- `gateway-capability-snapshot.json` 已记录 screen/window/desktop 等 capability state，metadata 有固定 allowlist，smoke 会断言。
- `extractData` 和 `runAgentLoop` 已消费 `accessibilityTree`，因此 payload 结构需向后兼容。

## 实现步骤

1. 在 `Tools/claw-gateway-server.mjs` 增加配置 `allowAccessibilityObserve: process.env.CLAW_ALLOW_ACCESSIBILITY_OBSERVE === "1"` 和 help 文案。
2. 扩展 capability snapshot：
   - `policies.accessibilityTree.enabled`
   - `capabilities.accessibilityTree.state`，取值 `real` / `dry-run` / `unavailable`
   - metadata key `accessibilityTreeState`
3. 重构 `observeScreenAction`：
   - 保留现有截图/window metadata 行为。
   - 始终按既有行为写 `accessibilityTree` artifact，但当 `includeAccessibilityTree == "false"` 时只写 not-requested 摘要。
   - 默认 mode 仍为 `dry-run` 或 `window-metadata`。
   - 仅当 `CLAW_ALLOW_ACCESSIBILITY_OBSERVE=1`、`process.platform === "darwin"` 且结构化参数允许时，调用新的只读采集函数。
4. 新增只读采集函数：
   - 只采集前台 app/window、role、title/label 的短摘要、focused/enabled、候选控件数量。
   - 严格按 `maxCandidateControls` 限制，硬上限 50。
   - 对文本做 `normalizeText`、长度截断和敏感策略标记；不要采集密码字段值。
   - 失败时返回 `mode=accessibility-unavailable` 或 `accessibility-failed`，写明可能需要 macOS Accessibility 权限。
5. 确保 artifact payload 和 metadata 安全：
   - 不包含 raw token、Authorization、自然语言 instruction、完整 `toolArguments`、完整 workspace path、截图二进制、命令输出。
   - 只使用固定 AppleScript，不拼接自然语言或任意选择器。
6. 更新 Gateway direct/WebSocket smoke：
   - 默认 dry-run 断言仍通过。
   - capability snapshot 包含 `accessibilityTreeState`。
   - 启用 `CLAW_ALLOW_ACCESSIBILITY_OBSERVE=1` 的场景不要求 CI 有权限成功读取真实 AX，但必须产出合法 mode 和可审计 artifact。
7. 更新 Swift model/UI/XCTest/LogicSmoke：
   - `ClawGatewayCapabilityReviewSummary.accessibilityTreeState`
   - Gateway capability row 展示 `ax <state>` chip
   - 测试覆盖 metadata 解析和 legacy fallback
8. 更新 README、协议、flow、flowchart、update_log。

## 验收标准

- 默认不设置 `CLAW_ALLOW_ACCESSIBILITY_OBSERVE` 时，Gateway 不做真实 AX 读取，smoke 仍通过。
- 设置显式开关时，Gateway 只按结构化 `toolArguments` 采集有限只读摘要；无权限时生成可审计失败/不可用 artifact，而不是崩溃。
- 不新增 action/event/artifact kind；仍复用 `observeScreen` 和既有 artifact。
- capability snapshot 和 metadata 能表达 Accessibility tree 状态，且不泄漏敏感字段。
- 文档明确 iOS 端不能读取其他 App 私有数据，桌面读取依赖用户授权 Gateway 和 macOS 权限。
- 本地轻量检查通过，push 后 GitHub Actions 通过，Agent C artifact 复判通过。

## 风险和禁止项

- macOS Accessibility 权限在 CI/无 GUI 环境大概率不可用，测试必须接受 `unavailable/failed` 的可审计结果。
- AppleScript/System Events 只能用于固定只读脚本，不得拼接自然语言或未约束选择器。
- 禁止采集密码字段值、raw token、Authorization、完整 workspace path、完整网页正文、命令输出或草稿正文。
- 禁止把 Accessibility 摘要描述成 iOS 能力；它是用户授权桌面 Gateway 能力。
- 禁止删除 smoke 断言、绕过 allowlist/审批闸门、伪造 CI 结果或混入无关重构。
