# v0.16（手机端 Accessibility Artifact 复核摘要）Agent B 实现提示词

## 版本

- 版本：v0.16
- 阶段：v0（核心智能能力）
- 基线：`main` / `origin/main`，HEAD `20ab08a311a78fa59ce4698caccbc8dd1e000d59`
- 提交信息：`v0.16: 手机端Accessibility复核摘要`
- 版本依据：v0.15 已让 Gateway `observeScreen` 产出受控 `accessibilityTree` metadata，并通过云端 artifact 复判。本轮把这些安全 metadata 带到手机/iPad 复核层，推进“像 Claw/OpenClaw 一样能观察电脑”的用户可解释闭环。

## 背景

当前手机端已能展示 Gateway capability、Replay Guard 和 AgentTrace 复核摘要，但 `accessibilityTree` artifact 只作为普通 artifact chip 出现。v0.15 的 Gateway 已在 `accessibilityTree` artifact event metadata 中写入 `accessibilityTree=observeSummary`、`mode`、`accessibilityPolicy`、`maxCandidateControls`、`nodeCount`、`candidateControlCount`、`platform`、`redaction` 和 `safetyFlags`。手机端需要只基于这些 metadata 展示 Accessibility 观察摘要，说明本次观察是 dry-run、window-metadata、accessibility-summary、accessibility-failed、accessibility-unavailable 或 not-requested。

## 目标

- 新增手机端 `ClawGatewayAccessibilityReviewSummary`，筛选 `ClawGatewayArtifact.kind == .accessibilityTree`；metadata 合法时解析字段，否则显示 legacy fallback。
- `ClawMissionRunSummary` 增加 accessibility review 字段，Mission Run 首屏、Gateway session card、单个 result row 都能展示最新 Accessibility 观察复核摘要。
- UI 展示 mode、policy、节点数、候选控件数、max controls、platform、redaction、safety flags、脱敏状态和 metadata 缺失 fallback。
- 模拟事件流给 `accessibilityTree` artifact 附带安全 metadata，覆盖 dry-run 基线。
- XCTest 和 Swift logic smoke 覆盖 metadata 解析、legacy fallback、Mission Run 派生、敏感字段不显示。
- 更新 README、协议、flow/flowchart、update_log。

## 非目标

- 不新增 action kind、event kind、artifact kind 或 envelope schema 字段。
- 不修改 Gateway 真实 Accessibility 采集逻辑，不增加 AppleScript、Shell、浏览器或桌面控制权限。
- 不读取 Gateway `file://` artifact payload；手机端只使用 artifact event metadata。
- 不展示 raw token、Authorization header、完整 workspace path、`toolArguments`、命令输出、截图内容、网页正文、草稿正文或密码字段值。
- 不把 Accessibility 摘要描述成 iOS 自己能读取其他 App 私有数据；真实观察只发生在用户授权桌面 Gateway。

## 当前架构依据

- `ClawGatewayArtifactKind.accessibilityTree` 已存在。
- v0.15 Gateway `accessibilityTreeMetadata()` 已输出安全 metadata。
- `ClawAgentTraceReviewSummary`、`ClawGatewayCapabilityReviewSummary`、`ClawGatewayTaskReplayGuardReviewSummary` 是可复用模式：从 artifact metadata 派生 summary，不读取 `file://`。
- `ClawMissionRunSummary` 当前已有 `agentTraceReview`、`gatewayCapabilityReview`、`gatewayTaskReplayGuardReview`。
- `ContentView.swift` 中 Mission Run、Gateway session card 和 result row 已分别展示 AgentTrace/ReplayGuard/Capability review row。
- `ClawGatewaySimulator.makeResult(.observeScreen)` 当前产出 `accessibilityTree` artifact，但 metadata 缺失。

## 实现步骤

1. 在 `Claw/Core/ClawModels.swift` 新增 `ClawGatewayAccessibilityReviewSummary`：
   - 字段建议：`treeCount`、`latestTitle`、`hasMetadata`、`mode`、`accessibilityPolicy`、`includeAccessibilityTree`、`maxCandidateControls`、`nodeCount`、`candidateControlCount`、`platform`、`redaction`、`safetyFlags`、`isRedacted`。
   - `compactStatus` 示例：`ax dry-run · controls 2/12 · redaction maskSensitiveText`；metadata 缺失时显示待同步。
   - `latest(from session:)` 应读取 `session.allArtifacts`；`latest(from artifacts:)` 应只筛 `kind == .accessibilityTree`。
   - 所有字符串字段必须通过安全清理/脱敏，不读取未知敏感 metadata。
2. 扩展 `ClawMissionRunSummary`：
   - 增加 `gatewayAccessibilityReview: ClawGatewayAccessibilityReviewSummary?`。
   - 在 summary 构造处从最新 session 派生。
3. 更新 `Claw/Services/ClawStore.swift`：
   - `ClawGatewaySimulator.makeResult(.observeScreen)` 的 `accessibilityTree` artifact 附带 v0.15 同款安全 metadata。
4. 更新 `Claw/Views/ContentView.swift`：
   - 新增 `ClawGatewayAccessibilityReviewRow`，风格与现有 review row 一致，使用 `PhoneAgentTag`、`Label`、系统图标和 `.accessibilityElement(children: .combine)`。
   - Mission Run panel、Gateway session card、单个 Gateway result row 都展示该 row。
   - 不做卡片嵌套；不要新增说明性大段文案。
5. 更新测试：
   - `ClawTests.swift` 覆盖 metadata 解析、legacy fallback、Mission Run 派生、敏感 metadata 脱敏。
   - `Tools/LogicSmoke.swift` 覆盖 simulator dry-run accessibility review。
6. 更新文档：
   - README：说明手机/iPad 只从 `accessibilityTree` artifact metadata 展示观察摘要。
   - 协议：说明 metadata key 用途和不读取 `file://` payload。
   - flow/flowchart：把 Accessibility review 加入核心数据流和 UI 复核。
   - update_log：追加 v0.16 记录。

## 验收标准

- Mission Run / Gateway session / result row 能展示 Accessibility 复核摘要。
- metadata 缺失的旧 `accessibilityTree` artifact 降级为“metadata 待同步”，不崩溃。
- UI 不显示 raw token、Authorization、`toolArguments`、完整 `file://`、完整 workspace path 或敏感值。
- 不新增协议字段、action/event/artifact kind，不扩大 Gateway 权限。
- 本地轻量检查通过，push 后 GitHub Actions 通过，Agent C artifact 复判通过。

## 风险和禁止项

- 禁止读取 `file://` payload 来补字段；metadata 不足时必须显示待同步。
- 禁止把 Accessibility 摘要说成 iOS 本地能力。
- 禁止删除现有 smoke/XCTest 断言让测试通过。
- 禁止混入 Gateway 真实控制、点击、输入或 AppleScript 改动。
- 禁止把旧 artifact、旧 output 或本地结果冒充最新 CI artifact。
