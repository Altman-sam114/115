# v0.17（手机端 Artifact Metadata 详情复核）Agent B 实现提示词

## 版本

- 版本：v0.17
- 阶段：v0（核心智能能力）
- 基线：`main` / `origin/main`，HEAD `f70318c01809dd067db6263071e39d0ea7a9199c`
- 提交信息：`v0.17: 手机端Artifact metadata复核`
- 版本依据：v0.16 已让手机端展示 Accessibility artifact metadata 摘要并通过云端 artifact 复判。本轮继续补齐手机/iPad 复核体验，让普通 Gateway artifact 也能展示安全 metadata 详情，但仍不读取 `file://` payload。

## 背景

当前 Mission Run 和 Gateway result row 只把 artifact 展示为 kind chip；capability、Accessibility、Replay Guard 和 AgentTrace 有专项复核行。用户要让 Claw 更像 OpenClaw 式电脑智能体，需要在手机/iPad 上能快速判断 artifact 是否带 metadata、是否脱敏、有哪些安全字段，而不是只能看到“截图/控件树/浏览轨迹”等类型。

## 目标

- 新增手机端 `ClawGatewayArtifactMetadataReviewSummary`，只从 `ClawGatewayArtifact.metadata`、artifact kind/title 和 `isRedacted` 派生，不读取 `reference` 指向的 payload。
- Mission Run summary 增加 artifact metadata review 字段；Mission Run、Gateway session card 和单个 result row 展示 metadata 复核行。
- UI 展示 artifact 总数、带 metadata 数、脱敏数、最近 artifact kind/title、最近 metadata 安全键值、safety flags 和 metadata 缺失 fallback。
- 字符串必须脱敏：不得展示 raw token、Authorization header、完整 `file://`、完整 workspace path、`toolArguments`、命令输出、网页正文、截图内容、草稿正文或密码字段值。
- XCTest 和 Swift logic smoke 覆盖 metadata 存在、metadata 缺失、敏感字段脱敏、Mission Run 派生和 simulator dry-run 基线。
- 更新 README、协议、flow/flowchart、update_log。

## 非目标

- 不新增 action kind、event kind、artifact kind 或 envelope schema 字段。
- 不修改 Gateway JS，不新增 artifact payload 字段，不改变 artifact 存储路径。
- 不实现完整 JSON payload viewer，不打开或读取 Gateway `file://` artifact 内容。
- 不新增真实电脑权限，不改变浏览器、Shell、AppleScript、Accessibility 或桌面 App 执行策略。
- 不让 iOS 声称能静默读取电脑文件、其他 App 私有数据或 Gateway artifact payload。

## 当前架构依据

- `ClawGatewayArtifact` 已有 `metadata: [String: String]?`，且旧 JSON 缺字段可 decode。
- `ClawGatewaySession.allArtifacts` 已统一 session-level artifacts 和 action-bound artifacts。
- `ClawArtifactMetadataParser`、`ClawSensitiveTextRedactor`、`ClawGatewayAccessibilityReviewSummary` 和 `ClawGatewayTaskReplayGuardReviewSummary` 已有 metadata 解析与脱敏模式。
- `ClawMissionRunSummary` 当前已有 capability、Accessibility、Replay Guard 和 AgentTrace review 字段。
- `ContentView.swift` 已有 `ClawGatewayCapabilityReviewRow`、`ClawGatewayAccessibilityReviewRow`、`ClawGatewayTaskReplayGuardReviewRow`、`ClawAgentTraceReviewRow` 可作为 UI 风格参考。

## 实现步骤

1. 在 `Claw/Core/ClawModels.swift` 新增 `ClawGatewayArtifactMetadataReviewSummary`：
   - 字段建议：`artifactCount`、`metadataArtifactCount`、`redactedArtifactCount`、`latestKind`、`latestTitle`、`hasMetadata`、`safeMetadataPairs`、`safetyFlags`、`isLatestRedacted`。
   - `safeMetadataPairs` 只取最近 metadata 的前几个安全键值，键和值都要清理和脱敏；未知敏感字段值必须变成 `secret-redacted`、`header-redacted`、`file-redacted`、`path-redacted` 等安全文本。
   - `compactStatus` 要能说明 metadata 覆盖情况；metadata 缺失时显示“metadata 待同步”。
   - `latest(from session:)` 使用 `session.allArtifacts`；`latest(from artifacts:)` 用传入 artifact 列表。
2. 扩展 `ClawMissionRunSummary` 和 `ClawStore.missionRunSummary`：
   - 增加 `artifactMetadataReview: ClawGatewayArtifactMetadataReviewSummary?`。
   - 从最新 Gateway session 派生，空 session/空 artifact 时返回 nil。
3. 更新 `Claw/Views/ContentView.swift`：
   - 新增 `ClawGatewayArtifactMetadataReviewRow`，使用 `PhoneAgentTag`、`Label`、横向 chip；保持紧凑，不做卡片嵌套。
   - Mission Run panel、Gateway session card、单个 Gateway result row 都展示该 row。
   - 文案强调 metadata-only；不要提示用户打开 payload。
4. 更新测试：
   - `ClawTests.swift` 覆盖 metadata 解析、metadata 缺失 fallback、敏感 metadata 脱敏、Mission Run 派生。
   - `Tools/LogicSmoke.swift` 覆盖 simulator 产生的 metadata review。
5. 更新文档：
   - README：说明手机/iPad 支持 artifact metadata-only 详情复核。
   - `Docs/claw-mobile-gateway-protocol.md`：说明该 review 不新增协议字段、不读取 payload。
   - `md/flow/flow.md` 和 `md/flow/flowchart.md`：加入 artifact metadata review summary。
   - `update_log.md`：追加 v0.17 记录。

## 验收标准

- Mission Run / Gateway session / result row 能展示 artifact metadata 复核摘要。
- metadata 缺失时显示 fallback，不崩溃。
- UI 不显示 raw token、Authorization、`toolArguments`、完整 `file://`、完整 workspace path 或敏感值。
- 不新增协议字段、action/event/artifact kind，不扩大 Gateway 权限。
- 本地轻量检查通过，push 后 GitHub Actions 通过，Agent C artifact 复判通过。

## 风险和禁止项

- 禁止读取 `file://` payload 来补字段；metadata 不足时必须显示待同步。
- 禁止把 artifact metadata review 描述成完整 artifact 内容查看器。
- 禁止删除现有 smoke/XCTest 断言让测试通过。
- 禁止混入 Gateway 真实控制、点击、输入或 AppleScript 改动。
- 禁止把旧 artifact、旧 output 或本地结果冒充最新 CI artifact。
