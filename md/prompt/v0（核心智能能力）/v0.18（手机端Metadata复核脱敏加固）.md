# v0.18（手机端 Metadata 复核脱敏加固）Agent B 实现提示词

## 版本

- 版本：v0.18
- 阶段：v0（核心智能能力）
- 基线：`main` / `origin/main`，HEAD `4ebae2dfcc0f95f3f5ed4da7b0572d59186eaabc`
- 提交信息：`v0.18: 手机端metadata复核脱敏加固`
- 版本依据：v0.17 已加入通用 Artifact metadata-only 复核。子 Agent 复核发现专用 AgentTrace / Gateway capability 摘要仍直接展示部分 `cleanValue` 字段，未统一走 v0.17 的敏感值脱敏路径。本轮优先修安全基线，再继续扩展浏览器或提取复核。

## 背景

手机端 Mission Run 和 iPad 多栏工作台会展示 AgentTrace、Gateway capability、Accessibility、Replay Guard 和通用 Artifact metadata 复核摘要。通用 artifact 行已隐藏未知键并脱敏敏感值，但 AgentTrace 与 Gateway capability 专用摘要仍可能把异常 Gateway metadata 中的 Authorization、Bearer token、`file://`、workspace path 或命令/页面内容片段展示到 UI。

## 目标

- 在 `Claw/Core/ClawModels.swift` 中集中一套 metadata 安全显示工具，统一处理 `cleanValue`、list 解析、敏感值脱敏和路径/header/token 替换。
- 让 AgentTrace、Gateway capability、Accessibility、Replay Guard 和通用 Artifact metadata review 的所有 UI 可见字符串都走同一安全路径。
- 新增 XCTest 和 Swift logic smoke 覆盖恶意 AgentTrace / capability metadata，不显示 raw token、Authorization、Bearer、`file://`、`/private`、workspace path、`toolArguments` 或敏感 header 内容。
- 同步 README、协议、flow/flowchart 和 update_log，明确专用复核摘要也只展示安全 metadata，不读取 payload。

## 非目标

- 不新增 action kind、event kind、artifact kind 或 envelope schema 字段。
- 不修改 Gateway JS，不新增 Gateway metadata 字段，不改变 artifact payload。
- 不实现 artifact payload viewer，不读取 Gateway `file://` reference。
- 不新增真实电脑权限、浏览器控制、Shell 执行、AppleScript 或 Accessibility 动作。
- 不解决 artifact “latest” 的完整时间序问题；如需处理只可记录为遗留，不在本轮引入跨模型 schema 变更。

## 当前架构依据

- `ClawGatewayArtifactMetadataReviewSummary` 已有安全键 allowlist 和 `safeMetadataValue` / `safeMetadataList` 模式。
- `ClawGatewayAccessibilityReviewSummary` 与 `ClawGatewayTaskReplayGuardReviewSummary` 已有类似脱敏 helper，但存在重复。
- `ClawAgentTraceReviewSummary` 和 `ClawGatewayCapabilityReviewSummary` 当前直接使用 `ClawArtifactMetadataParser.cleanValue` / `listValue`，需要改为安全显示路径。
- `ContentView.swift` 的专用复核 row 会直接展示这些 summary 字段，因此模型层必须保证字段已经适合 UI 显示。

## 实现步骤

1. 在 `Claw/Core/ClawModels.swift` 新增集中 helper，例如 `ClawArtifactMetadataDisplaySanitizer`：
   - `safeValue(_:) -> String?`
   - `safeList(_:) -> [String]`
   - 保留 v0.17 的替换语义：Authorization/header、Bearer token、token/password/secret、headers、workspace、`file://`、path 等必须变为安全占位。
2. 将通用 `ClawGatewayArtifactMetadataReviewSummary`、`ClawAgentTraceReviewSummary`、`ClawGatewayAccessibilityReviewSummary`、`ClawGatewayCapabilityReviewSummary`、`ClawGatewayTaskReplayGuardReviewSummary` 改为复用集中 helper。
3. AgentTrace 字段必须脱敏：
   - `latestTitle`
   - `satisfiedSignals`
   - `missingSignals`
   - `selectedNextActionKind`
   - `riskTags`
   - `stopReason`
   - `handoffSummary`
4. Gateway capability 字段必须脱敏：
   - `latestTitle`
   - `tokenFingerprint`
   - `allowedActionKinds`
   - `workspaceState`
   - `shellState`
   - `browserControlState`
   - `browserNetworkState`
   - `screenCaptureState`
   - `windowMetadataState`
   - `accessibilityTreeState`
   - `desktopControlState`
   - `safetyFlags`
   - `platform`
5. 更新测试：
   - XCTest 增加恶意 AgentTrace metadata 脱敏测试。
   - XCTest 增加恶意 Gateway capability metadata 脱敏测试。
   - Swift logic smoke 增加至少一个专用复核摘要恶意 metadata 脱敏断言。
6. 更新文档：
   - README、协议、flow/flowchart 说明专用 review summary 统一 metadata-only 脱敏。
   - `update_log.md` 追加 v0.18 记录。

## 验收标准

- AgentTrace / Gateway capability / Accessibility / Replay Guard / Artifact metadata review 不显示 raw token、Authorization、Bearer、`file://`、完整 workspace path、`toolArguments` 或明显敏感 payload。
- 旧 metadata 缺失路径仍显示 fallback，不崩溃。
- 不新增协议字段、不修改 Gateway 权限、不读取 artifact payload。
- 本地轻量检查通过，push 后 GitHub Actions 通过，Agent C artifact 复判通过。

## 风险和禁止项

- 禁止只修 UI 文案而不修模型层字段；UI 可见字段必须在模型层已脱敏。
- 禁止删除既有 smoke/XCTest 断言让测试通过。
- 禁止混入浏览器控制、真实点击、输入、Shell 或 AppleScript 改动。
- 禁止把旧 artifact、旧 output 或本地结果冒充最新 CI artifact。
