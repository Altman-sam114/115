# v0.19（提取完整性复核）Agent B 实现提示词

## 版本

- 版本：v0.19
- 阶段：v0（核心智能能力）
- 基线：`main` / `origin/main`，HEAD `32ff6e33e701683381b633aaf7389b5b7b1a13f0`
- 提交信息：`v0.19: 提取完整性复核`
- 版本依据：v0.18 已统一手机端 metadata 复核脱敏。本轮继续朝 OpenClaw 式 mac/iPad 控制台推进，让用户能在继续草稿、表单或最终提交前判断结构化提取是否有足够 artifact 依据。

## 背景

Gateway `extractData` 已消费 browser trace、file diff、command output、screen observation、accessibility tree 等 session artifact，并生成结构化输出。但手机/iPad 端目前只能看到通用 artifact 和 agent trace 摘要，不能直接判断提取是否 artifact-grounded、来源覆盖是否足够、行数是否为空、是否通过完整性检查。

## 目标

- Gateway 给既有 `extractData` 输出 artifact 增加安全 metadata，不新增 artifact kind/event kind/action kind。
- 手机端新增 `ClawGatewayExtractionCompletenessReviewSummary`，只从 artifact metadata 派生完整性复核摘要，不读取 `file://` payload。
- Mission Run、Gateway session card 和单个 result row 展示提取完整性复核行。
- XCTest、Swift logic smoke、Gateway smoke 覆盖 metadata 存在、metadata 缺失 fallback、来源计数、空结果、敏感 metadata 脱敏。
- 同步 README、协议、flow/flowchart、update_log。

## 非目标

- 不改变 `extractData` 的核心提取算法。
- 不新增 artifact kind；继续复用当前输出 artifact。
- 不读取 artifact `reference` 或打开 Gateway `file://` payload。
- 不展示 row 内容、网页正文、命令输出、URL/path、草稿正文、联系人、Authorization/header、token、cookie 或 `toolArguments`。
- 不新增真实浏览器控制、Shell 执行、AppleScript、Accessibility 动作或桌面权限。
- 不修正 artifact 全局时间顺序；该问题保留为 v0.18 遗留。

## 当前架构依据

- `Tools/claw-gateway-server.mjs` 的 `extractDataAction()` 已生成 `mode: "artifact-grounded-extraction"`、`validateCompleteness`、`sourceArtifacts` 和 `rows`。
- `writeArtifact()` 已支持可选 metadata。
- `ClawGatewayArtifact` 已有可选 `metadata`。
- `ClawArtifactMetadataDisplaySanitizer` 已提供 UI 安全字符串脱敏。
- `ClawMissionRunSummary` 已聚合多个 metadata-only review；`ContentView.swift` 已有复核 row 样式。

## Gateway metadata 设计

为 `extractData` 输出 artifact 增加 metadata，建议键：

- `extractionReview=artifactGrounded`
- `mode`
- `validateCompleteness`
- `rowCount`
- `completenessStatus`：`complete` / `empty` / `partial`
- `browserTraceCount`
- `fileDiffCount`
- `commandOutputCount`
- `screenObservationCount`
- `accessibilityTreeCount`
- `messageDraftCount`
- `sourceArtifactKinds`
- `safetyFlags`

metadata 只允许短安全字符串和计数。`sourceArtifactKinds` 只能是 kind 名称列表，不得带文件名、路径、URL、正文、命令或 row 内容。`safetyFlags` 至少表达 metadata-only、row-content-omitted、source-values-omitted、tool-arguments-omitted、artifact-payload-not-read。

## 实现步骤

1. `Tools/claw-gateway-server.mjs`
   - 新增 `extractionCompletenessMetadata(extraction)` 或等价 helper。
   - `extractDataAction()` 写 artifact 时传 metadata。
   - metadata 禁止 row 内容、URL/path、命令输出、页面正文、`toolArguments`。
2. `Tools/claw-gateway-direct-smoke.mjs` / `Tools/claw-gateway-smoke.mjs`
   - 验证 `extractData` artifact event metadata 存在。
   - 验证计数字段与 payload `sourceArtifacts` / `rows.length` 一致。
   - 验证 metadata 不包含 forbidden strings。
3. `Claw/Core/ClawModels.swift`
   - 新增 `ClawGatewayExtractionCompletenessReviewSummary`。
   - 从 metadata `extractionReview=artifactGrounded` 或 `mode=artifact-grounded-extraction` 的 artifact 派生。
   - 字段建议：`extractionCount`、`latestTitle`、`hasMetadata`、`mode`、`validateCompleteness`、`rowCount`、`completenessStatus`、各来源计数、`sourceArtifactKinds`、`safetyFlags`、`isRedacted`。
   - 所有字符串走 `ClawArtifactMetadataDisplaySanitizer`。
4. `Claw/Services/ClawStore.swift`
   - `ClawMissionRunSummary` 增加 `gatewayExtractionCompletenessReview` 字段，从 latest session 派生。
   - Gateway simulator 的 `extractData` artifact 添加同类安全 metadata，便于手机端和 logic smoke 覆盖。
5. `Claw/Views/ContentView.swift`
   - 新增 `ClawGatewayExtractionCompletenessReviewRow`。
   - 在 Mission Run、Gateway session card、Gateway result row 展示。
6. `ClawTests/ClawTests.swift` / `Tools/LogicSmoke.swift`
   - 覆盖 Mission Run 派生、metadata fallback、恶意 metadata 脱敏和空/完整状态。
7. 文档
   - README、协议、flow/flowchart、update_log 说明 extraction completeness review 是 metadata-only，不读取 payload、不新增权限。

## 验收标准

- `extractData` artifact event metadata 覆盖来源计数、row count、完整性状态和 safety flags。
- 手机端 Mission Run / Gateway session / result row 能展示提取完整性复核。
- metadata 缺失时降级为“metadata 待同步”。
- UI 不显示 row 内容、网页正文、命令输出、URL/path、草稿正文、联系人、Authorization/header、token、cookie 或 `toolArguments`。
- 不新增协议字段、action/event/artifact kind，不扩大 Gateway 权限。
- 本地轻量检查通过，push 后 GitHub Actions 通过，Agent C artifact 复判通过。

## 风险和禁止项

- 禁止把 extraction review 描述成完整 artifact viewer。
- 禁止把 rows、正文、命令输出、URL/path 或 `toolArguments` 放入 metadata。
- 禁止为了 smoke 通过删除现有断言。
- 禁止混入浏览器控制、点击、输入、表单提交、Shell 真执行或 AppleScript 行为变更。
