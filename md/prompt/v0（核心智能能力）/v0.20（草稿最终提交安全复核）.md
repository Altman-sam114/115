# v0.20（草稿最终提交安全复核）Agent B 实现提示词

## 版本

- 版本：v0.20
- 阶段：v0（核心智能能力）
- 基线：`main` / `origin/main`，HEAD `01b5dcf1d4713bcdf1e54dcad3406dfa5cad17d0`
- 提交信息：`v0.20: 草稿最终提交安全复核`
- 版本依据：v0.19 已让手机端判断结构化提取是否 artifact-grounded。本轮继续朝 OpenClaw 式 mac/iPad 控制台推进，在草稿、桌面 App 输入和最终发送/提交前给用户一个 metadata-only 的安全复核摘要。

## 背景

当前 Gateway 已有两类最终提交前停顿：

- `composeMessage` / `composeEmail` 会生成 `messageDraft` artifact 并返回 `waitingForApproval`。
- `operateDesktopApp` 会根据 `finalSubmitRequiresApproval` 和 key allowlist 停在最终提交前。

但手机/iPad 端目前只能看到通用 artifact metadata 和 AgentTrace 风险标签，不能直接判断本轮是否停在最终提交闸门、草稿正文是否被省略、是否需要用户确认、是否有提交键被阻断。

## 目标

- Gateway 给既有 `messageDraft` 和 `operateDesktopApp` 相关 artifact 增加安全 metadata，不新增 artifact kind/event kind/action kind。
- 手机端新增 `ClawGatewayDeliverySafetyReviewSummary`，只从 artifact metadata 派生草稿/最终提交安全复核摘要，不读取 `file://` payload。
- Mission Run、Gateway session card 和单个 result row 展示 Delivery Safety 复核行。
- XCTest、Swift logic smoke、Gateway smoke 覆盖 metadata 存在、metadata 缺失 fallback、最终提交闸门、提交键阻断、敏感 metadata 脱敏。
- 同步 README、协议、flow/flowchart、update_log。

## 非目标

- 不实现真实消息发送、邮件发送、表单提交或桌面 App 最终提交。
- 不新增 action kind、artifact kind、event kind 或 schema 字段。
- 不读取 artifact `reference` 或打开 Gateway `file://` payload。
- 不展示草稿正文、粘贴文本、网页正文、命令输出、URL/path、联系人、Authorization/header、token、cookie 或 `toolArguments`。
- 不新增真实浏览器控制、Shell 执行、AppleScript、Accessibility 动作或桌面权限。
- 不修正 artifact 全局时间顺序；仍沿用当前 session/result artifacts 派生方式。

## 当前架构依据

- `Tools/claw-gateway-server.mjs` 的 `writeArtifact()` 已支持可选 metadata。
- `messageDraftAction()` 已写 `messageDraft` artifact 并返回 `waitingForApproval`。
- `desktopAppAction()` 已计算 `finalSubmitRequiresApproval`、`keyPlan.allowed`、`keyPlan.blocked`，并在 dry-run/real/blocked 路径写 screenshot JSON artifact。
- `ClawGatewayArtifact` 已有可选 `metadata`。
- `ClawArtifactMetadataDisplaySanitizer` 已提供 UI 安全字符串脱敏；v0.19 已对专用 review 使用白名单。
- `ClawMissionRunSummary` 已聚合多个 metadata-only review；`ContentView.swift` 已有复核 row 样式。

## Gateway metadata 设计

为 `messageDraft` 和 `operateDesktopApp` 相关 artifact 增加 metadata，建议键：

- `deliveryReview=finalSubmitGate`
- `mode`：`message-draft-pending-approval` / `desktop-control-dry-run` / `desktop-control-paused` / `desktop-control-policy-blocked` / `desktop-control-unavailable` / `desktop-control-failed` / `desktop-control-completed`
- `actionKind`：`composeMessage` / `composeEmail` / `operateDesktopApp`
- `targetKind`：`message` / `email` / `desktopApp`
- `finalSubmitRequiresApproval`
- `userApprovalRequired`
- `draftBodyOmitted`
- `pasteTextOmitted`
- `submitBlocked`
- `allowedKeyCount`
- `blockedKeyCount`
- `blockedSubmitKeyCount`
- `safetyFlags`

metadata 只允许短安全字符串、布尔值和计数。`safetyFlags` 至少表达 `metadata-only`、`draft-body-omitted`、`paste-text-omitted`、`final-submit-gated`、`user-approval-required`、`tool-arguments-omitted`、`artifact-payload-not-read`。不得把草稿正文、粘贴文本、按键原文、URL/path、命令输出或 `toolArguments` 放入 metadata。

## 实现步骤

1. `Tools/claw-gateway-server.mjs`
   - 新增 `deliverySafetyMetadata(action, details)` 或等价 helper。
   - `messageDraftAction()` 写 `messageDraft` artifact 时传 metadata。
   - `desktopAppAction()` 在 dry-run、unavailable、missing target、policy-blocked、real result 路径写 artifact 时传 metadata。
   - metadata 只能包含枚举、布尔和计数；禁止草稿正文、paste text、key sequence 原文、`toolArguments`。
2. `Tools/claw-gateway-direct-smoke.mjs` / `Tools/claw-gateway-smoke.mjs`
   - 验证 `operateDesktopApp` 或 `messageDraft` artifact event metadata 存在。
   - 验证 final-submit gate、approval required、draft/paste omission、blocked submit key count。
   - 验证 metadata 不包含 forbidden strings。
3. `Claw/Core/ClawModels.swift`
   - 新增 `ClawGatewayDeliverySafetyReviewSummary`。
   - 从 metadata `deliveryReview=finalSubmitGate` 的 artifact，或 legacy `messageDraft` artifact 派生。
   - 字段建议：`reviewCount`、`latestTitle`、`hasMetadata`、`mode`、`actionKind`、`targetKind`、`finalSubmitRequiresApproval`、`userApprovalRequired`、`draftBodyOmitted`、`pasteTextOmitted`、`submitBlocked`、key counts、`safetyFlags`、`isRedacted`。
   - 所有字符串走白名单或 `ClawArtifactMetadataDisplaySanitizer`，列表用固定 allowlist。
   - 通用 artifact metadata review allowlist 增加本轮安全键。
4. `Claw/Services/ClawStore.swift`
   - `ClawMissionRunSummary` 增加 `gatewayDeliverySafetyReview` 字段，从 latest session 派生。
   - Gateway simulator 的 `operateDesktopApp` 和 `composeMessage` / `composeEmail` artifact 添加同类安全 metadata。
5. `Claw/Views/ContentView.swift`
   - 新增 `ClawGatewayDeliverySafetyReviewRow`。
   - 在 Mission Run、Gateway session card、Gateway result row 展示。
   - metadata 缺失时降级为“metadata 待同步”，不能显示假阳性的“已安全”。
6. `ClawTests/ClawTests.swift` / `Tools/LogicSmoke.swift`
   - 覆盖 Mission Run 派生、metadata fallback、恶意 metadata 脱敏、final submit gate 和 blocked submit key counts。
7. 文档
   - README、协议、flow/flowchart、update_log 说明 delivery safety review 是 metadata-only，不读取草稿正文或 payload、不新增权限。

## 验收标准

- `messageDraft` 和 `operateDesktopApp` 相关 artifact event metadata 覆盖 final-submit gate、approval required、正文省略、paste text 省略和 key block counts。
- 手机端 Mission Run / Gateway session / result row 能展示 Delivery Safety 复核。
- metadata 缺失时降级为“metadata 待同步”。
- UI 不显示草稿正文、粘贴文本、按键原文、URL/path、网页正文、命令输出、联系人、Authorization/header、token、cookie 或 `toolArguments`。
- 不新增协议字段、action/event/artifact kind，不扩大 Gateway 权限。
- 本地轻量检查通过，push 后 GitHub Actions 通过，Agent C artifact 复判通过。

## 风险和禁止项

- 禁止把 delivery safety review 描述成已发送或可自动发送。
- 禁止把草稿正文、paste text、按键序列原文或 `toolArguments` 放入 metadata。
- 禁止为了 smoke 通过删除现有断言。
- 禁止混入真实发送、点击提交、表单提交、Shell 真执行或 AppleScript 行为变更。
