# v0.21（浏览器控制计划复核）Agent B 实现提示词

## 版本

- 版本：v0.21
- 阶段：v0（核心智能能力）
- 基线：`main` / `origin/main`，HEAD `8b2ebdcbfd6cce1110e3ae50024839d9a53e99c2`
- 提交信息：`v0.21: 浏览器控制计划复核`
- 版本依据：v0.19 已让手机端复核结构化提取是否有 artifact 依据，v0.20 已让手机端复核草稿/最终提交是否停在审批闸门。本轮继续补齐 OpenClaw 式 mac/iPad 控制台的关键审计面：在浏览器打开、搜索、网络抓取或策略阻断前后，让手机/iPad 端只用安全 metadata 复核浏览器控制计划和策略结果。

## 背景

当前 Gateway 的 `controlBrowser` 已经能生成两类既有 artifact：

- `browserTrace`：包含本地 HTML、URL trace、标题、链接、表格、表单、候选控件、文本预览和浏览器控制 payload。
- `screenshot`：`browser-control-*.json`，记录浏览器打开/搜索计划、dry-run、policy blocked、host blocked、unavailable、opened 或 failed 等状态。

这些 payload 内部会包含 raw URL、search query、HTML/page text、form fields、candidate control labels、stdout/stderr 和结构化参数等敏感或高噪声内容。手机/iPad 端目前只能通过通用 artifact metadata review、提取完整性 review 和 Gateway capability review 间接判断浏览器相关状态，缺少一个专门的 metadata-only 浏览器控制计划复核摘要。

## 目标

- Gateway 给既有 `controlBrowser` 相关 artifact event 增加安全 metadata，不新增 artifact kind、event kind、action kind 或 schema 字段。
- 手机端新增 `ClawGatewayBrowserControlReviewSummary`，只从 artifact event metadata 派生浏览器控制计划复核摘要，不读取 `file://` payload。
- Mission Run、Gateway session card 和单个 result row 展示 Browser Control 复核行，帮助用户判断浏览器控制是否只是 dry-run、是否请求打开/搜索、网络/host/app allowlist 是否参与、是否被策略阻断。
- Gateway simulator 为 `controlBrowser` 输出同类安全 metadata，使本地 Swift summary/UI 能覆盖模拟路径。
- XCTest、Swift logic smoke、Gateway direct/WebSocket smoke 覆盖 metadata 存在、metadata 缺失 fallback、策略阻断、敏感 metadata 脱敏和 forbidden strings 不外显。
- 同步 README、协议、flow/flowchart、update_log。

## 非目标

- 不实现真实浏览器点击、表单填写、标签管理、Playwright/browser-use 控制器或视觉定位。
- 不扩大 macOS 浏览器控制权限，不绕过 `CLAW_ALLOW_BROWSER_CONTROL`、browser app allowlist 或 host allowlist。
- 不新增 action kind、artifact kind、event kind、schema 字段或 envelope 字段。
- 不读取 artifact `reference`，不打开 Gateway `file://` payload，不把 payload 内容复制到 UI。
- 不把 raw URL、search query、HTML/page text、form fields、candidate control labels、headers、Authorization、tokens、cookies、`toolArguments`、`file://`、workspace path、stdout/stderr 或命令输出放入 metadata 或 UI。
- 不把浏览器控制计划复核描述成已经完成点击、填写、提交或外部网络访问成功。
- 不修正 artifact 全局时间顺序；仍沿用当前 `sessionArtifacts + results.artifacts` 派生方式。

## Gateway metadata 设计

新增 helper，例如 `browserControlReviewMetadata(action, browserInput, plan, details)`。metadata 只能包含枚举、布尔值、计数和固定 safety flags；不得包含任何 raw URL/query/body/label/path/header/token。

建议 metadata 键：

- `browserReview=controlPlan`
- `mode`：沿用安全枚举，例如 `browser-control-not-requested`、`browser-control-dry-run`、`browser-control-unavailable`、`browser-control-policy-blocked`、`browser-control-host-blocked`、`browser-control-opened`、`browser-control-failed`
- `actionKind=controlBrowser`
- `browserControlPolicy`：`not-requested` / `dry-run` / `enabled`
- `browserControlRequested`
- `openInBrowserRequested`
- `targetURLPresent`
- `searchQueryPresent`
- `localHTMLInput`
- `networkFetchAttempted`
- `networkBlocked`
- `appAllowlistEnforced`
- `hostAllowlistEnforced`
- `appAllowed`
- `targetHostAllowed`
- `executed`
- `resultStatus`：`skipped` / `succeeded` / `failed`
- `platform`：安全平台枚举
- `safetyFlags`

建议 `safetyFlags` 至少包含：

- `metadata-only`
- `tool-arguments-omitted`
- `target-url-omitted`
- `search-query-omitted`
- `html-content-omitted`
- `page-content-omitted`
- `form-fields-omitted`
- `candidate-controls-omitted`
- `headers-omitted`
- `browser-app-name-omitted`
- `network-allowlist-enforced`
- `browser-app-allowlist-enforced`
- `artifact-payload-not-read`

metadata 不应包含 `browserApp`、`targetURL`、`searchQuery`、`source`、`instruction`、`inputPreview`、`browserGoal`、`appAllowlist`、`hostAllowlist`、`stdout`、`stderr`、`html`、`textPreview`、`forms`、`candidateControls`、`toolArguments` 或任何 path/header/token 字段。

## 实现步骤

1. `Tools/claw-gateway-server.mjs`
   - 新增浏览器控制复核 metadata helper。
   - 在 `controlBrowserAction()` 写 `browserTrace` artifact 时附同一类安全 metadata。
   - 在 `browserControlArtifact()` 的所有返回路径为 `browser-control-*.json` artifact 附 metadata：not requested、dry-run、unavailable、app policy blocked、host blocked、opened、failed。
   - helper 从 `browserInput.mode`、`plan.requested`、`plan.targetURL` 是否存在、`plan.searchQuery` 是否存在、config allowlist 是否启用、当前 result status/mode 派生布尔和枚举；禁止把具体 URL、query、HTML、host、app name、allowlist 值、stdout/stderr 或 `toolArguments` 写进 metadata。
2. `Tools/claw-gateway-direct-smoke.mjs` / `Tools/claw-gateway-smoke.mjs`

   - 验证 dry-run local HTML 浏览器 artifact metadata 存在，`browserReview=controlPlan`、`mode=browser-control-dry-run`、`browserControlRequested=true`、`searchQueryPresent=true`、`targetURLPresent=true`、`localHTMLInput=true`、allowlist safety flags 存在。

## 验收标准

- `controlBrowser` 相关 `browserTrace` 和 `browser-control-*.json` artifact event 都能携带 `browserReview=controlPlan` 安全 metadata。
- 手机端 Mission Run、Gateway session card 和 result row 能展示 Browser Control 复核摘要。
- metadata 缺失时降级为“metadata 待同步”。
- Browser Control review 能表达 requested、URL/query 是否存在、local HTML、network blocked/fetch attempted、app/host allowlist enforced、policy/result status、executed 和 safety flags。
- UI 和通用 metadata review 不显示 raw URL、search query、HTML/page text、form fields、candidate control labels、headers、Authorization、tokens、cookies、`toolArguments`、`file://`、workspace path、stdout/stderr 或命令输出。
- 不新增协议字段、action/event/artifact kind，不扩大 Gateway 权限，不实现真实点击/表单/提交。
- 本地轻量检查通过，push 后 GitHub Actions 通过，Agent C artifact 复判通过。

## 风险和禁止项

- 风险：`browserTrace` payload 本身包含敏感浏览内容，因此本轮必须保证手机端 review 只看 metadata，不读取 payload。
- 风险：browser control payload 目前包含 URL/query/app/allowlist/stdout/stderr 等字段，不能把这些字段直接复用为 metadata。
- 风险：metadata allowlist 如果过宽，会让通用 Artifact metadata review 外显敏感值；必须只加本轮固定安全键。
- 禁止把 raw URL、search query、HTML/page text、form fields、candidate control labels、headers、tokens、cookies、`toolArguments`、`file://`、workspace path、stdout/stderr 或命令输出放入 metadata。
- 禁止为了 smoke 通过删除现有断言或降低安全断言。
- 禁止混入真实浏览器点击、表单填写、最终提交、Shell 真执行、AppleScript 权限扩大或 Accessibility 动作变更。
- 禁止把 review 描述成浏览器任务已完成；它只能说明控制计划、策略和 artifact metadata 复核状态。
