# v0.22（文件变更安全复核）Agent B 实现提示词

## 版本

- 版本：v0.22
- 阶段：v0（核心智能能力）
- 基线：`main` / `origin/main`，HEAD `b91c9203d4bca85c521749882bcacaa7d2a42436`
- 提交信息：`v0.22: 文件变更安全复核`
- 版本分配依据：v0.19、v0.20、v0.21 已分别补齐结构化提取、草稿/最终提交和浏览器控制的 metadata-only 专用复核。本轮继续补 OpenClaw 式 mac/iPad 控制台的关键审计面：`manageFiles` 已能真实写 workspace 文件并生成 `fileDiff` artifact，但手机/iPad 端还缺少专门的文件变更安全复核摘要。

## 背景

当前 Gateway 的 `manageFiles` 会按结构化 `toolArguments.writePath` / `toolArguments.writeText` 在 session workspace 内写文件，并回传既有 `fileDiff` artifact。该 artifact payload 目前包含 workspace 内真实落点、session workspace、requested path 等执行细节；这些内容适合留在 Gateway artifact payload 供审计和后续 extraction 使用，但不适合直接展示到手机/iPad UI。

现有手机端有通用 Artifact metadata review、Extraction completeness、Browser Control、Delivery Safety、Gateway capability、Accessibility、Replay Guard 和 AgentTrace 复核摘要；文件变更只通过通用 artifact 行间接出现，用户无法在 Mission Run 首屏直接判断本轮文件写入是否受 session workspace 限制、是否阻断路径逃逸、是否只省略 raw path / diff / 内容后展示安全摘要。

本轮目标是补 metadata-only 的 File Change Safety review：让 Gateway 在既有 `fileDiff` artifact event 上附安全 metadata，手机/iPad 端只从 metadata 派生文件变更安全摘要，不读取 `file://` payload，不展示 raw path、workspace path、file content 或 diff content。

## 目标

- Gateway 给既有 `manageFiles` / `fileDiff` artifact event 增加安全 metadata，不新增 artifact kind、event kind、action kind 或 schema 字段。
- 覆盖成功写入路径，也覆盖路径逃逸或文件写入失败的可审计路径；失败路径可复用既有 `fileDiff` 或 `auditLog` artifact kind，但不得新增 kind。
- 手机端新增 `ClawGatewayFileChangeSafetyReviewSummary`，只从 artifact event metadata 派生文件变更安全摘要，不读取 Gateway `file://` payload。
- Mission Run、Gateway session card 和单个 result row 展示 File Change Safety 复核行。
- Gateway simulator 和 `Tools/ClawGatewayEventFixture.swift` 为 `manageFiles` 输出同类安全 metadata。
- XCTest、Swift logic smoke、Gateway direct/WebSocket smoke 覆盖 metadata 存在、metadata 缺失 fallback、路径逃逸阻断、敏感 metadata 脱敏和 forbidden strings 不外显。
- 同步 README、`Docs/claw-mobile-gateway-protocol.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和 `update_log.md`。

## 非目标

- 不实现完整 diff viewer、文件内容 viewer、回滚、目录授权 UI、文件移动/删除/上传或跨目录文件浏览。
- 不改变 `manageFiles` 的结构化输入契约，不把自然语言直接交给文件系统。
- 不扩大 workspace 权限，不允许路径逃逸到 session workspace 外。
- 不新增 action kind、artifact kind、event kind、schema 字段或 envelope 字段。
- 不读取 artifact `reference`，不打开 Gateway `file://` payload，不把 payload 内容复制到 UI。
- 不把 raw path、workspace path、`file://`、文件内容、diff 内容、token、headers、Authorization、cookies、`toolArguments`、stdout/stderr 或命令输出放入 metadata 或 UI。
- 不把文件变更复核描述成完整安全扫描、病毒扫描、权限修复、回滚保证或跨 Gateway 持久审计。
- 不修正 artifact 全局时间顺序；仍沿用当前 `sessionArtifacts + results.artifacts` 派生方式。

  - `ClawGatewayExtractionCompletenessReviewSummary`、`ClawGatewayBrowserControlReviewSummary`、`ClawGatewayDeliverySafetyReviewSummary` 可作为专用 metadata-only review 模板。

- `Claw/Services/ClawStore.swift`
  - `missionRunSummary` 从最新 session 派生各类 metadata-only review。
  - `ClawGatewaySimulator.makeResult` 的 `.manageFiles` 目前只返回无 metadata 的 `fileDiff` artifact。
- `Claw/Views/ContentView.swift`
  - Mission Run、Gateway session card 和 result row 已有 compact review row 样式，可复用 `Label`、`PhoneAgentTag`、横向 chip 和 `.accessibilityElement(children: .combine)`。
- `Tools/claw-gateway-direct-smoke.mjs` / `Tools/claw-gateway-smoke.mjs`

## Gateway metadata 设计

新增 helper，例如 `fileChangeSafetyMetadata(action, details)`。metadata 只能包含固定枚举、布尔值、计数和固定 safety flags；不得包含任何 raw path、workspace path、`file://`、文件内容、diff 内容、headers、token 或 `toolArguments`。

建议 metadata 键：

- `fileChangeReview=workspaceWrite`
- `mode`：`workspace-write` / `workspace-path-blocked` / `workspace-write-failed`
- `actionKind=manageFiles`
- `workspacePolicy=session-workspace-only`
- `workspaceScoped`
- `pathEscapeBlocked`
- `writeAttempted`
- `writeSucceeded`
- `createdFileCount`
- `modifiedFileCount`
- `deletedFileCount`
- `requestedPathPresent`
- `writeTextPresent`
- `contentOmitted`
- `diffOmitted`
- `rawPathOmitted`
- `resultStatus`：`succeeded` / `failed` / `skipped`
- `safetyFlags`

建议 `safetyFlags` 至少包含：

- `metadata-only`
- `tool-arguments-omitted`
- `raw-path-omitted`
- `workspace-path-omitted`
- `file-content-omitted`
- `diff-content-omitted`
- `session-workspace-only`
- `artifact-payload-not-read`

路径逃逸或被阻断时额外包含：

- `path-escape-blocked`
- `no-file-written`

写入失败但非路径逃逸时额外包含：

- `write-failed`

metadata 不应包含 `writePath`、`requestedPath`、`created`、`workspace`、`sessionWorkspace`、`absolutePath`、`relativePath`、文件名、目录名、文件内容、diff hunk、patch、stdout、stderr、headers、token、`toolArguments` 或任何 `file://` / `/Users/...` / workspace path 字段。

## 实现步骤

1. `Tools/claw-gateway-server.mjs`
   - 新增 `fileChangeSafetyMetadata(action, details)` 或等价 helper。
   - `manageFilesAction()` 成功写入时，为既有 `fileDiff` artifact 附 metadata。
   - metadata 从结构化状态派生：是否有 requested path、是否有 write text、是否尝试写入、是否成功、created/modified/deleted 计数、workspace policy、path escape 是否阻断、result status 和 safety flags。
   - 路径逃逸时，不要让 `safeWorkspacePath()` 抛出后完全缺少审计信息。建议在 `manageFilesAction()` 内捕获 `workspace_path_escape_blocked`，写一个 redacted 的既有 `fileDiff` 或 `auditLog` artifact，附 `mode=workspace-path-blocked` metadata，然后返回 failed/retryable 结果或重新抛出前保证事件可审计。无论采用哪种形式，都不得新增 artifact kind/event kind/schema 字段。
   - 写入失败时附 `mode=workspace-write-failed` metadata；payload 和 summary 可以保留错误类别，但 metadata/UI 不得包含 raw path、workspace path、文件内容、diff 内容或 `toolArguments`。
   - 不破坏现有 fileDiff payload 被 `extractData` 和 `runAgentLoop` 消费的行为。
2. `Tools/claw-gateway-direct-smoke.mjs` / `Tools/claw-gateway-smoke.mjs`
   - 增加 `assertFileChangeSafetyMetadata()` 或等价 helper。
   - 验证普通 `manageFiles` 成功写入的 `fileDiff` artifact metadata 存在，包含 `fileChangeReview=workspaceWrite`、`mode=workspace-write`、`actionKind=manageFiles`、`workspacePolicy=session-workspace-only`、`workspaceScoped=true`、`writeAttempted=true`、`writeSucceeded=true`、`createdFileCount=1`、`pathEscapeBlocked=false`、`contentOmitted=true`、`diffOmitted=true`、`rawPathOmitted=true` 和 omission flags。
   - 增加路径逃逸 case，例如结构化 `writePath` 请求 `../escape.txt` 或绝对路径；断言 Gateway 阻断该动作，并且 artifact event metadata 能表达 `mode=workspace-path-blocked`、`pathEscapeBlocked=true`、`writeSucceeded=false`、`resultStatus=failed`、`path-escape-blocked`、`no-file-written`。
   - 断言 metadata JSON 不包含 smoke 中的 raw `writePath`、workspace path、`file://`、文件正文、diff 内容、token、Authorization/header、`toolArguments`、stdout/stderr。
   - 保持既有 fileDiff、extraction、agentTrace 断言，不要删减安全断言来让 smoke 通过。
3. `Claw/Core/ClawModels.swift`
   - 新增 `ClawGatewayFileChangeSafetyReviewSummary`。
   - 从 `metadata["fileChangeReview"] == "workspaceWrite"` 的 `fileDiff` artifact 派生；legacy `fileDiff` 无 metadata 时仍返回 fallback summary，`hasMetadata=false` 且 compact status 包含“metadata 待同步”。
   - 建议字段：`reviewCount`、`latestTitle`、`hasMetadata`、`mode`、`actionKind`、`workspacePolicy`、`workspaceScoped`、`pathEscapeBlocked`、`writeAttempted`、`writeSucceeded`、`createdFileCount`、`modifiedFileCount`、`deletedFileCount`、`requestedPathPresent`、`writeTextPresent`、`contentOmitted`、`diffOmitted`、`rawPathOmitted`、`resultStatus`、`safetyFlags`、`isRedacted`。
   - 对 mode、actionKind、workspacePolicy、resultStatus、safetyFlags 使用固定 allowlist；所有 UI 可见字符串走 `ClawArtifactMetadataDisplaySanitizer` 或只用本地固定文案。
   - 扩展 `ClawGatewayArtifactMetadataReviewSummary.allowedMetadataKey`，只加入本轮安全键；不要加入 `writePath`、`requestedPath`、`workspace`、`created`、`content`、`diff`、`patch`、`toolArguments` 等 raw payload key。
   - `ClawMissionRunSummary` 增加 `gatewayFileChangeSafetyReview` 字段。
4. `Claw/Services/ClawStore.swift`
   - 在 `missionRunSummary` 构造中从最新 session 派生 `gatewayFileChangeSafetyReview`。
   - Gateway simulator 的 `.manageFiles` result 为 `fileDiff` artifact 添加同类安全 metadata。
   - 增加 simulator helper，例如 `fileChangeSafetyMetadata(mode:resultStatus:...)`，保持与 Gateway metadata 键一致。
5. `Tools/ClawGatewayEventFixture.swift`

   - 保持 fixture metadata 和 Gateway/simulator 键名一致。

   - 在 Mission Run、Gateway session card 和 Gateway result row 中展示；建议放在 Artifact metadata review 后、Extraction/Browser/Delivery 等专用 review 附近。
   - 显示内容应是安全复核：workspace policy、write attempted/succeeded、path escape blocked、created/modified/deleted count、内容/diff/path 已省略、safety flags。

   - 覆盖恶意 metadata：raw path、workspace path、`file://`、文件内容、diff hunk、patch、token、Authorization/header、cookie、`toolArguments`、stdout/stderr 不进入 summary 或 safe metadata pairs。

## 验收标准

- `manageFiles` 相关 `fileDiff` artifact event 能携带 `fileChangeReview=workspaceWrite` 安全 metadata。
- 成功写入路径显示 session workspace policy、write attempted/succeeded、计数、内容/diff/path 省略和 safety flags。
- 路径逃逸或写入失败路径有可审计 metadata，能表达 blocked/failed，且不会写入 workspace 外文件。
- 手机端 Mission Run、Gateway session card 和 result row 能展示 File Change Safety 复核摘要。
- metadata 缺失时降级为“metadata 待同步”。
- UI 和通用 metadata review 不显示 raw path、workspace path、`file://`、文件内容、diff 内容、patch、headers、Authorization、tokens、cookies、`toolArguments`、stdout/stderr 或命令输出。
- 不新增协议字段、action/event/artifact kind，不扩大 Gateway 权限，不实现回滚、目录浏览、完整 diff viewer 或文件内容 viewer。
- 本地轻量检查通过，push 后 GitHub Actions 通过，Agent C artifact 复判通过。

## 风险和禁止项

- 风险：`fileDiff` payload 当前包含真实 workspace/session path；本轮必须保证手机端 File Change Safety review 只看 metadata，不读取 payload。
- 风险：`manageFiles` 是真实写文件能力；路径逃逸失败路径必须保持阻断，不得为生成 metadata 而放宽 `safeWorkspacePath()`。
- 风险：metadata allowlist 如果过宽，会让通用 Artifact metadata review 外显敏感路径或内容；必须只加本轮固定安全键。
- 禁止把 raw path、workspace path、`file://`、文件名/目录名、文件内容、diff hunk、patch、headers、tokens、cookies、`toolArguments`、stdout/stderr 或命令输出放入 metadata 或 UI。
- 禁止为了 smoke 通过删除现有 fileDiff、extraction、agentTrace 或安全脱敏断言。
- 禁止混入真实文件删除/移动/上传、目录授权 UI、回滚、Shell 真执行、AppleScript 权限扩大、浏览器点击/表单或桌面 App 最终提交。
- 禁止把 review 描述成文件变更已完整安全审计或可回滚；它只能说明 workspace 写入计划、路径边界、阻断/成功状态和 metadata 安全省略状态。
