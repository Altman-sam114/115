## 背景

当前 Gateway 的 `runShellCommand` 只读取结构化 `toolArguments.shellCommand`，默认 dry-run；必须同时设置 `CLAW_ALLOW_SHELL=1` 和 `CLAW_SHELL_ALLOWLIST`，且命令 binary 命中 allowlist 后才会真实执行。Gateway 会在缺少结构化命令、解析失败、policy 阻断和 allowlist 真执行路径写入既有 `commandOutput` artifact。

这些 `commandOutput` payload 可能包含 raw command、stdout、stderr、cwd 或解析错误上下文，适合保留在 Gateway artifact payload 供审计和后续 extraction / agent loop 消费，但不能直接进入手机/iPad UI。现有手机端已经有通用 Artifact metadata review、File Change Safety、Extraction completeness、Browser Control、Delivery Safety、Gateway capability、Accessibility、Replay Guard 和 AgentTrace 复核摘要；Shell 目前只显示为 redacted `commandOutput` artifact 或提取来源计数，用户无法在 Mission Run 首屏直接判断命令是否来自结构化字段、是否被 allowlist/dry-run 策略拦截、是否真实执行、是否超时、exit code 级别是否可复核，以及 command/stdout/stderr/cwd 是否已从 UI metadata 中省略。

本轮目标是给既有 `runShellCommand` / `commandOutput` artifact event 增加 metadata-only 的 Shell Command Safety review：Gateway 只在 metadata 中返回固定枚举、布尔值、计数和 omission flags；手机端新增专用 summary 并在 Mission Run / Gateway session / result row 展示。手机端不得读取 `file://` payload，不得展示 raw command、stdout、stderr、cwd、path、token、header 或 `toolArguments`。

## 目标

- Gateway 给既有 `runShellCommand` 产生的 `commandOutput` artifact event 增加 metadata-only 的 Shell Command Safety metadata。
- 覆盖四类 Shell 路径：缺少结构化 `shellCommand`、解析失败、dry-run / allowlist policy 阻断、allowlist 真执行成功或失败。
- 手机端新增 `ClawGatewayShellCommandSafetyReviewSummary`，只从 artifact event metadata 或 legacy `commandOutput` artifact 派生 Shell 命令安全复核摘要，不读取 Gateway `file://` payload。
- Mission Run、Gateway session card 和单个 result row 展示 Shell Command Safety 复核行。
- Gateway simulator 和 `Tools/ClawGatewayEventFixture.swift` 为 `runShellCommand` 输出同类安全 metadata。
- direct smoke 和 WebSocket smoke 断言 metadata 固定键、策略状态、执行状态、omission flags，并确认 metadata 不泄露 raw command、stdout、stderr、cwd/path、token、Authorization/header、cookie 或 `toolArguments`。
- XCTest 和 Swift logic smoke 覆盖 Mission Run 派生、result 单独派生、legacy metadata fallback、fixture 派生和恶意 metadata 脱敏。
- 同步 `README.md`、`Docs/claw-mobile-gateway-protocol.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和 `update_log.md`。

## 非目标

- 不新增 action kind、artifact kind、event kind、schema 字段或 envelope 字段。
- 不扩大 Shell 权限，不改变 `CLAW_ALLOW_SHELL` / `CLAW_SHELL_ALLOWLIST` 策略，不新增 shell 执行后门。
- 不把自然语言 `instruction` 直接交给 Shell；Shell 仍只能读取结构化 `toolArguments.shellCommand`。
- 不新增命令模板系统、交互式 shell、shell session、后台进程、管道 DSL、sudo、环境变量编辑、目录授权 UI 或三阶段审批 UI。
- 不读取 artifact `reference`，不打开 Gateway `file://` payload，不把 `commandOutput` payload 内容复制到 UI。
- 不在 metadata/UI 展示 raw command、binary path、args、cwd、workspace/session path、stdout、stderr、exit output、shellCommand、token、Authorization/header、cookie、`toolArguments`、自然语言 instruction 或任何 file/path 值。
- 不把 metadata-only 复核描述成完整命令沙箱、恶意命令检测、权限隔离、持久审计、命令回滚或安全保证。
- 不修改提取/agent loop 对 `commandOutput` payload 的既有消费方式，除非是为了保持现有测试通过的最小适配。

## Gateway metadata 设计

新增 helper，例如 `shellCommandSafetyMetadata(action, details)`。metadata 只能包含固定枚举、布尔值、计数和固定 safety flags；不得包含 raw command、binary、args、cwd、stdout、stderr、path、token、header、cookie、自然语言 instruction 或 `toolArguments`。

建议 metadata 键：

- `shellReview=commandSafety`
- `mode`：`missing-structured-command` / `command-parse-failed` / `shell-policy-blocked` / `shell-executed`
- `actionKind=runShellCommand`
- `shellPolicy`：`dry-run` / `allowlist-required` / `allowlist-enabled`
- `structuredCommandPresent`
- `commandParsed`
- `allowlistConfigured`
- `allowlistMatched`
- `executionAttempted`
- `executed`
- `timedOut`
- `exitCodePresent`
- `exitCodeZero`
- `stdoutPresent`
- `stderrPresent`
- `commandOmitted`
- `stdoutOmitted`
- `stderrOmitted`
- `cwdOmitted`
- `resultStatus`：`succeeded` / `failed` / `skipped`
- `safetyFlags`

建议 `safetyFlags` 至少包含：

- `metadata-only`
- `structured-arguments-only`
- `tool-arguments-omitted`
- `command-omitted`
- `stdout-omitted`
- `stderr-omitted`
- `cwd-omitted`
- `artifact-payload-not-read`

按路径额外包含：

- 缺少结构化字段：`natural-language-not-executed`、`no-command-executed`
- 解析失败：`parse-failed`、`no-command-executed`
- policy 阻断：`shell-allowlist-enforced`、`dry-run-only`、`no-command-executed`
- allowlist 真执行：`shell-allowlist-enforced`，并通过 `executed=true`、`executionAttempted=true`、`exitCodePresent=true`、`exitCodeZero` 表达结果；不要在 metadata 中写 exit code 数值以外的输出内容，建议只写布尔 presence。

metadata 禁止键和值：

- 禁止任何 raw `shellCommand`、命令行、binary、args、`pwd`、`ls`、测试命令、工作目录、session workspace、`/Users/...`、`.build/...`、`file://`、stdout/stderr 内容、payload 行、natural-language `instruction`、`toolArguments`、token、Authorization/header、Bearer、cookie。
- 禁止为了方便 UI 在 metadata 中写 `commandPreview`、`binaryName`、`cwd`、`stdoutPreview`、`stderrPreview`、`outputPath`、`workingDirectory`、`rawCommand` 等字段。

## 验收标准

- `runShellCommand` 相关 `commandOutput` artifact event 能携带 `shellReview=commandSafety` 安全 metadata。
- 缺少结构化命令、解析失败、policy/dry-run 阻断和 allowlist 真执行路径均能表达安全状态，且不改变既有执行权限。
- 手机端 Mission Run、Gateway session card 和 result row 能展示 Shell Command Safety 复核摘要。
- metadata 缺失时降级为“metadata 待同步”。
- UI 和通用 metadata review 不显示 raw command、stdout、stderr、cwd/path、binary/args、`shellCommand`、token、Authorization/header、cookie、`toolArguments`、natural-language instruction 或 `file://`。
- Gateway simulator、fixture、direct smoke、WebSocket smoke、XCTest 和 LogicSmoke 都覆盖本轮 metadata-only Shell review。
- 不新增协议字段、action/event/artifact kind，不扩大 Gateway 权限，不实现交互式 shell、命令模板、后台进程或审批 UI。
- 本地轻量检查通过，push 后 GitHub Actions 通过，Agent C artifact 复判通过。

## 风险和禁止项

- 风险：`commandOutput` payload 可能包含 raw command、stdout、stderr 和 cwd；本轮必须保证手机端 Shell Command Safety review 只看 metadata，不读取 payload。
- 风险：allowlist 真执行路径容易把 `pwd`、stdout、workspace/session path 或 exit output 带进 metadata；metadata 只能表达 presence/boolean/status，不写值。
- 风险：通用 Artifact metadata review allowlist 如果过宽，会外显敏感命令或路径；只能加入本轮固定安全键。
- 风险：为了测试 allowlist 真执行而改变 Gateway policy 会扩大权限；只能在 smoke 的受控环境变量里验证现有 allowlist 行为。
- 禁止把 raw command、binary、args、cwd、workspace/session path、`file://`、stdout、stderr、payload 内容、headers、tokens、cookies、`toolArguments`、自然语言 instruction 放入 metadata 或 UI。
- 禁止把自然语言 instruction 交给 Shell。
- 禁止新增 action kind、artifact kind、event kind、schema/envelope 字段。
- 禁止扩大 Shell allowlist、默认开启真实 Shell、绕过审批、绕过 Gateway policy 或降低现有 smoke/XCTest 断言。
- 禁止只改代码不改相关文档。
- 禁止把旧 CI artifact 或本地输出冒充最新云端结果。
