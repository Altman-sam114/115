- 版本分配依据：v0.24 已把多类 metadata-only 复核摘要聚合为 Mission Run 复核优先队列。当前队列能提示“先看什么”，但不能让用户直接聚焦队列项对应的详细复核行。v0.25 在不改变 Gateway 协议的前提下补齐队列项聚焦和详情筛选体验，提升 iPhone/iPad/mac 复核效率。

## 背景

Claw 当前方向是 OpenClaw 式电脑接管智能体：手机端负责计划、风险展示、审批、envelope 发送和事件查看；真实电脑操作发生在用户授权的桌面 Gateway。Gateway 已输出多类 metadata-only 复核摘要，v0.24 已在 Mission Run 首屏展示按风险/可行动性排序的复核优先队列。

本轮要把队列从静态提示升级为可交互的聚焦入口：用户点击某个队列项后，Mission Run 只突出或筛选该 `reviewKind` 对应的详细复核行；点击“全部”恢复全量复核。该能力必须只基于既有 summary/reviewKind，不读取 artifact payload，不写 envelope，不改变 Gateway 权限。

## 目标

- Mission Run 复核优先队列项可点击，并显示当前聚焦项。
- 聚焦某个有详细 row 的 `reviewKind` 时，只显示对应详细复核 row；无聚焦时显示全部详细 row。
- `approval`、`gateway-status` 或过期/未知聚焦项没有对应详细 row 时，不清空详情，保持全量详细复核并显示固定提示。
- `ClawMissionRunSummary` 增加可测试的 detail focus helper，例如可用 detail review kind、聚焦后可见 kind 和是否显示某个 detail row。
- iPhone compact 与 iPad regular 工作台复用同一 Mission Run 面板行为。
- 队列按钮使用 SwiftUI `Button` 和系统图标，提供 VoiceOver 可读 label，避免 `onTapGesture`。
- 增加 XCTest 和 Swift logic smoke，覆盖默认无聚焦、有效聚焦、非 detail 聚焦回退、未知聚焦回退和敏感字符串不外显。
- 同步 README、协议、flow/flowchart、测试说明和 update_log。

## 非目标

- 不新增 action kind、event kind、artifact kind、schema 字段、envelope 字段或 Gateway API。
- 不改 Gateway 执行权限、allowlist、Shell 策略、浏览器策略、文件 workspace 策略或最终提交闸门。
- 不读取 artifact `reference`，不打开 Gateway `file://` payload，不展示 raw artifact 内容。
- 不展示 raw URL、search query、HTML/page text、path、workspace/sessionWorkspace、文件名/文件内容/diff/patch、raw command、binary/args、cwd、stdout/stderr、草稿正文、paste text、按键原文、token、Authorization/header、cookie、自然语言 instruction 或 `toolArguments`。
- 不把聚焦模式描述成安全裁决、恶意检测、完整审计、自动修复器或跨进程 exactly-once。
- 不删除 Gateway session card 或 result row 的既有详细复核信息。

## 实现步骤

1. `Claw/Core/ClawModels.swift`
   - 给 `ClawMissionRunSummary` 增加 extension/helper：
     - `availableDetailReviewKinds`：按固定顺序返回当前存在详细 row 的 `reviewKind`。
     - `detailReviewKinds(focusedOn:)`：无聚焦返回全部；有效 detail kind 返回单项；非 detail 或未知 kind 回退全部。
     - `shouldShowDetailReview(_:focusedOn:)`：供 SwiftUI 条件显示。
     - 可选 `reviewPriorityItem(focusedOn:)`：供 UI 获取当前队列项。
   - helper 只能用固定 `reviewKind` 和已存在 summary 是否为 nil，不读取 payload。
2. `Claw/Views/ContentView.swift`
   - `ClawMissionRunPanel` 增加 `@State private var focusedReviewKind: String?`。
   - 把 `ClawMissionReviewPriorityQueueView` 改为接收 `selectedReviewKind`、`onSelectReviewKind`、`onClearFocus`。
   - 队列 row 改成 `Button`，选中态用描边/背景/图标表示，不只靠颜色。
   - 详细 review rows 使用 `summary.shouldShowDetailReview("...", focusedOn:)` 过滤。
   - 聚焦到 `approval`、`gateway-status` 或未知项时展示固定提示，并保持全量详细 row。
3. `ClawTests/ClawTests.swift`
   - 在 `testMissionRunSummaryDerivesReviewPriorityQueue` 附近补断言：
     - idle 无 detail kind。
     - 无聚焦返回全部 detail kind。

     - 聚焦 `gateway-status` 和未知 kind 回退全部。
     - 聚焦相关可见字符串不包含敏感词。
4. `Tools/LogicSmoke.swift`
   - 补同类 Mission Run focus helper smoke。
   - Shell 场景补聚焦 `shell-safety` 只显示 Shell detail。
5. 文档

   - `Docs/claw-mobile-gateway-protocol.md`：Mission Run Presentation Layer 加 v0.25 说明，强调非协议字段。
   - `md/flow/flow.md`：核心数据流、UI 步骤、SwiftUI App 职责、状态对象和展示派生层加入聚焦详情。
   - `md/flow/flowchart.md`：Mission Run summary 和 UI 节点加入 Focused Priority Detail。

## 验收标准

- Mission Run 队列项可点击聚焦，且可恢复全部复核。
- 有详细 row 的 `reviewKind` 聚焦后只显示对应详细 row；非 detail/未知聚焦不清空详情。
- 聚焦模式不读取 payload，不外显 raw URL/path/command/stdout/stderr/diff/token/header/toolArguments。
- iPhone compact 和 iPad regular 多栏工作台复用同一行为。
- XCTest、LogicSmoke 和云端 CI artifact 复判通过。

## 风险与禁止项

- 风险：聚焦让用户误以为其他风险消失。UI 必须保留“全部”入口，并让非 detail 聚焦回退全量。
- 风险：队列 row 只靠颜色表达选中态。必须用图标、描边或文案辅助。
- 风险：把 selection 放入模型 source of truth。聚焦状态只能是本地 presentation state，不写 envelope/Gateway。
- 禁止把项目方向改成法律/法务产品。
- 禁止自然语言直连 Shell、AppleScript 或桌面动作。
- 禁止绕过 allowlist、审批闸门、workspace 限制和 artifact 审计。
- 禁止把旧 artifact、旧 output 或本地输出冒充最新云端结果。
