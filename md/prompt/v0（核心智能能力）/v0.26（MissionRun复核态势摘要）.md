- 版本分配依据：v0.24 已有复核优先队列，v0.25 已能聚焦队列项对应的详细复核。当前 Mission Run 仍缺少一眼判断“现在是否需要人工行动、metadata 是否缺失、最高优先项是什么”的复核态势摘要。本轮继续强化 iPhone/iPad/mac 复核工作台，但仍保持手机端 presentation-only。

## 背景

Claw 的方向是 OpenClaw 式电脑接管智能体：手机端负责计划、审批、风险和复核，桌面 Gateway 执行受控动作并回传事件/artifact。随着 metadata-only 复核类型增多，v0.24 和 v0.25 已分别解决“先看什么”和“如何聚焦看”。下一步要在队列前增加一个短态势摘要，让用户先知道当前复核是否准备好、缺什么、最高优先项是什么。

该摘要不能成为安全裁决、恶意检测或自动继续执行依据；它只从完整 `reviewPriorityQueue`、已有 detail review kind 和本地聚焦状态派生。

## 目标

- 新增 `ClawMissionRunReviewReadinessSummary` 或等价模型，用于表达人工复核态势。
- 在 `ClawMissionRunSummary` extension 中以 computed helper 派生摘要，避免新增 stored property 和扩大 Codable/init 改动面。
- 摘要字段至少覆盖：
  - 总优先项数量。
  - 可行动项数量。
  - critical/high 项数量。
  - metadata 待同步项数量。
  - 当前可用 detail review 数量。
  - 最高优先项 review kind/title/action hint。
  - 是否可复核、是否需要人工行动。
  - 聚焦项是否有对应 detail row。
- Mission Run UI 在 artifact chips 后、复核优先队列前展示短态势摘要。
- 摘要可提供一个“聚焦最高优先项”按钮，复用 v0.25 的聚焦机制；必须用 SwiftUI `Button`，不用 `onTapGesture`。
- compact iPhone 和 iPad regular 工作台复用同一 `ClawMissionRunPanel` 行为。
- XCTest 和 Swift logic smoke 覆盖 idle、正常 mission、Shell 高风险、count 一致性、聚焦态和敏感字符串不外显。
- 同步 README、协议、flow/flowchart、测试说明和 update_log。

## 非目标

- 不新增 action kind、event kind、artifact kind、schema 字段、envelope 字段或 Gateway API。
- 不改 Gateway 执行权限、allowlist、Shell 策略、浏览器策略、文件 workspace 策略或最终提交闸门。
- 不读取 artifact `reference`，不打开 Gateway `file://` payload，不展示 raw artifact 内容。
- 不展示 raw URL、search query、HTML/page text、path、workspace/sessionWorkspace、文件名/文件内容/diff/patch、raw command、binary/args、cwd、stdout/stderr、草稿正文、paste text、按键原文、token、Authorization/header、cookie、自然语言 instruction 或 `toolArguments`。
- 不把摘要称为自动安全裁决、恶意检测、完整审计或自动修复器。
- 不替代 Agent C artifact 验收，不改 Agent X 调度规则。
