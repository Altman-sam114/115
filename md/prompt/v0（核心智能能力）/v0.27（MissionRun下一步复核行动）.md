# v0.27（Mission Run下一步复核行动）Agent B 实现提示词

## 版本

- 版本：v0.27
- 阶段：v0（核心智能能力）
- 基线：`main` / `origin/main`，上一轮 v0.26 最新已知 commit `53325834a2fd0eb3767bdf0b93a1af1deb8f30b8`。
- 提交信息：`v0.27: Mission Run下一步复核行动`
- 版本分配依据：v0.24 已有复核优先队列，v0.25 已有聚焦模式，v0.26 已有复核态势摘要。当前 Mission Run 仍需要把“最高优先项/聚焦项/metadata 缺口/审批阻断”转成一个明确、可点击、可复核的下一步行动提示，帮助 iPhone/iPad 用户更快进入下一步人工判断。

## 背景

Claw 的方向是 OpenClaw 式电脑接管智能体：手机端负责计划、审批、风险展示、envelope 发送和事件/artifact 复核，桌面 Gateway 负责在用户授权和策略 allowlist 内执行或模拟执行。随着 Gateway metadata-only 复核类型增多，Mission Run 需要从“展示很多复核结果”继续进化到“告诉用户下一步该复核什么”。

本轮只做手机端 presentation-layer 派生和 UI 展示，不新增 Gateway 权限，也不把任何自然语言直接连到 Shell、AppleScript 或桌面自动化。

## 目标

- 新增 `ClawMissionRunNextReviewAction` 或等价模型，表达 Mission Run 的下一步人工复核行动。
- 在 `ClawMissionRunSummary` extension 中以 computed helper 派生：
  - `var nextReviewAction: ClawMissionRunNextReviewAction`
  - `func nextReviewAction(focusedOn reviewKind: String?) -> ClawMissionRunNextReviewAction`
- 行动建议必须优先考虑当前有效聚焦项；没有聚焦项时使用完整 `reviewPriorityQueue` 的最高优先项；没有队列但有 detail review 时给出抽查详细复核；完全 idle 时给出等待 Gateway 证据。
- 行动字段至少覆盖：
  - title/status/guidance/icon。
  - reviewKind/title/actionHint。
  - 是否可聚焦到 detail row。
  - 是否需要人工行动。
  - 是否存在 metadata 缺口。
  - primaryButtonTitle。
- Mission Run UI 在复核态势摘要后、复核优先队列前展示下一步复核行动卡片。
- 行动按钮复用 v0.25 聚焦机制；必须用 SwiftUI `Button`，不用 `onTapGesture`，按钮最小高度不低于 44。
- compact iPhone 和 iPad regular 工作台复用同一 `ClawMissionRunPanel` 行为。
- XCTest 和 Swift logic smoke 覆盖 idle、正常 mission、聚焦 delivery、状态类聚焦 fallback、Shell 高风险和敏感字符串不外显。
- 同步 README、协议、flow/flowchart、测试说明和 update_log。

## 非目标

- 不新增 action kind、event kind、artifact kind、schema 字段、envelope 字段或 Gateway API。
- 不改 Gateway 执行权限、allowlist、Shell 策略、浏览器策略、文件 workspace 策略或最终提交闸门。
- 不读取 artifact `reference`，不打开 Gateway `file://` payload，不展示 raw artifact 内容。
- 不展示 raw URL、search query、HTML/page text、path、workspace/sessionWorkspace、文件名/文件内容/diff/patch、raw command、binary/args、cwd、stdout/stderr、草稿正文、paste text、按键原文、token、Authorization/header、cookie、自然语言 instruction 或 `toolArguments`。
- 不把下一步行动称为自动执行计划、自动安全裁决、恶意检测、完整审计或自动修复器。
- 不替代 Agent C artifact 验收，不改 Agent X 调度规则。

## 实现步骤

1. `Claw/Core/ClawModels.swift`
   - 在 readiness summary 附近新增 `ClawMissionRunNextReviewAction: Equatable, Codable, Sendable`。
   - 在 `ClawMissionRunSummary` extension 中新增 next-action helper。
   - helper 必须基于完整 `reviewPriorityQueue` 和 `availableDetailReviewKinds`，不能只看 UI 的 `prefix(5)`。
   - helper 只拼接固定文案、计数、枚举标题和队列项已脱敏字段。
2. `Claw/Views/ContentView.swift`
   - 新增 `ClawMissionNextReviewActionView`。
   - 在 `ClawMissionRunPanel` 中计算 `nextReviewAction`，并放在 readiness summary 后。
   - 如果 `reviewKind` 存在，按钮调用 `focusReviewKind`；如果没有可聚焦项，按钮只展示不可用状态或不展示，不制造假动作。
   - VoiceOver label 要包含下一步复核行动、状态和复核项名称。
3. `ClawTests/ClawTests.swift`
   - 扩展 `testMissionRunSummaryDerivesReviewPriorityQueue`：
     - idle next action 不可复核、不需要人工行动、不含 review kind。
     - 正常 mission next action 指向 queue first。
     - 聚焦 `delivery-safety` 时 next action 指向 delivery 并可聚焦 detail。
     - 聚焦 `gateway-status` 或无 detail 的状态项时不能错误筛空 detail。
     - 可见字符串不包含敏感词。
   - Shell 场景断言 `shell-safety` next action 高优先且不泄露 stdout/stderr。
4. `Tools/LogicSmoke.swift`
   - 同步 XCTest 核心断言。
5. 文档
   - `README.md`：当前能力、手机端方向、Gateway presentation-only 长段、smoke 覆盖和完成情况加入下一步复核行动。
   - `Docs/claw-mobile-gateway-protocol.md`：Mission Run Presentation Layer 加 v0.27 段。
   - `md/flow/flow.md`：核心数据流、UI 步骤、SwiftUI App 职责、状态对象和展示派生层加入下一步复核行动。
   - `md/flow/flowchart.md`：Mission Run summary 和 UI 节点加入 Next Review Action。

## 验收标准

- Mission Run 能派生并展示下一步人工复核行动。
- 下一步行动优先使用有效聚焦项，否则使用完整队列最高优先项，否则安全降级为 detail review 抽查或等待证据。
- 下一步行动不读取 payload，不外显 raw URL/path/command/stdout/stderr/diff/token/header/toolArguments。
- iPhone compact 和 iPad regular 多栏工作台复用同一行为。
- XCTest、LogicSmoke、本地轻量检查和云端 CI artifact 复判通过。

## 风险与禁止项

- 风险：下一步行动被误解为自动执行建议。文案必须写“复核/查看/聚焦”，不能写“自动继续执行”。
- 风险：helper 只看前 5 个 UI 队列项。必须基于完整队列。
- 风险：按钮对无 detail 的状态项造成详情误筛空。必须复用 `focusUsesDetailReview` 和已有 notice 行为。
- 禁止把项目方向改成法律/法务产品。
- 禁止自然语言直连 Shell、AppleScript 或桌面动作。
- 禁止绕过 allowlist、审批闸门、workspace 限制和 artifact 审计。
- 禁止把旧 artifact、旧 output 或本地输出冒充最新云端结果。
