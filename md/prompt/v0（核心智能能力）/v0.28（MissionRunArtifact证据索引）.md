## 背景

Claw 的方向是 OpenClaw 式电脑接管智能体：手机端负责计划、审批、风险展示、envelope 发送和事件/artifact 复核，桌面 Gateway 负责在用户授权和策略 allowlist 内执行或模拟执行。随着 review kind 增多，用户需要先看到安全的 artifact 证据覆盖关系，再进入复核态势、下一步行动和详细复核。

本轮只做手机端 presentation-layer 派生和 UI 展示，不新增 Gateway 权限，不读取 artifact payload，也不把自然语言直接连到 Shell、AppleScript 或桌面自动化。

## 目标

- 新增 `ClawMissionRunArtifactEvidenceIndex` 和 item 等价模型，表达 Mission Run 的 artifact 证据覆盖。
- 在 `ClawMissionRunSummary` extension 中以 computed helper 派生：
  - `var artifactEvidenceIndex: ClawMissionRunArtifactEvidenceIndex`
  - `func artifactEvidenceIndex(focusedOn reviewKind: String?) -> ClawMissionRunArtifactEvidenceIndex`
- 索引来源只允许使用：
  - `artifactKinds`
  - `artifactMetadataReview` 的计数、metadata 覆盖、redaction 状态和安全字段
  - `availableDetailReviewKinds`
  - `reviewPriorityQueue`
  - 当前有效聚焦项
- 索引字段至少覆盖：
  - title/status/guidance/icon。
  - artifact kind 总数、metadata artifact count、redacted count。
  - covered review count / missing review count。
  - 当前聚焦 review kind/title 是否有证据。
  - items：每个 detail review kind 对应的安全 artifact kind 列表、是否有 evidence、是否 metadata ready、是否可聚焦。
- Mission Run UI 在 artifact chips 后、复核态势摘要前展示 Artifact 证据索引。
- 索引 item 可用 `Button` 聚焦对应 review kind；按钮最小高度不低于 44，不用 `onTapGesture`。
- compact iPhone 和 iPad regular 工作台复用同一 `ClawMissionRunPanel` 行为。
- XCTest 和 Swift logic smoke 覆盖 idle、正常 mission、聚焦 delivery、Shell 高风险、metadata/redaction count、证据覆盖映射和敏感字符串不外显。
- 同步 README、协议、flow/flowchart、测试说明和 update_log。

## 非目标

- 不新增 action kind、event kind、artifact kind、schema 字段、envelope 字段或 Gateway API。
- 不改 Gateway 执行权限、allowlist、Shell 策略、浏览器策略、文件 workspace 策略或最终提交闸门。
- 不读取 artifact `reference`，不打开 Gateway `file://` payload，不展示 raw artifact 内容。
- 不展示 raw URL、search query、HTML/page text、path、workspace/sessionWorkspace、文件名/文件内容/diff/patch、raw command、binary/args、cwd、stdout/stderr、草稿正文、paste text、按键原文、token、Authorization/header、cookie、自然语言 instruction 或 `toolArguments`。
- 不把证据索引称为完整 payload viewer、自动安全裁决、恶意检测、完整审计或自动修复器。
- 不替代 Agent C artifact 验收，不改 Agent X 调度规则。

## 风险与禁止项

- 风险：证据索引被误解为完整 artifact viewer。文案必须写“证据索引/metadata-only/复核入口”，不能写“查看内容”。
- 风险：helper 只看前 5 个 UI chip。必须基于完整 `artifactKinds`。
- 风险：review kind -> artifact kind 映射过度断言。只能表达“证据类型覆盖”，不能声称 payload 已人工核对。
- 禁止把项目方向改成法律/法务产品。
- 禁止自然语言直连 Shell、AppleScript 或桌面动作。
- 禁止绕过 allowlist、审批闸门、workspace 限制和 artifact 审计。
- 禁止把旧 artifact、旧 output 或本地输出冒充最新云端结果。
