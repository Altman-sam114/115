## 背景

Claw 的方向是 OpenClaw 式电脑接管智能体：手机端负责计划、审批、风险展示、envelope 发送和事件/artifact 复核，桌面 Gateway 负责在用户授权和策略 allowlist 内执行或模拟执行。iPad/regular 工作台应更像控制台：用户不应必须从长卡里逐段寻找 Gateway 状态、证据覆盖、复核态势和下一步行动。

本轮只做手机端 presentation-layer 派生和 SwiftUI 展示，不新增 Gateway 权限，不读取 artifact payload，也不把自然语言直接连到 Shell、AppleScript 或桌面自动化。

## 目标

- 新增 Mission Run Operator Strip 派生模型，表达 4 个稳定操作 lane：
  - `gateway`：当前阶段、回合进度、成功/失败/可重试计数。
  - `evidence`：Artifact 证据索引覆盖、metadata/redaction count。
  - `review`：复核态势、可行动/高优先/metadata 缺口。
  - `next`：下一步人工复核行动或等待 Gateway 证据。
- 在 `ClawMissionRunSummary` extension 中以 computed helper 派生：
  - `var operatorStrip: ClawMissionRunOperatorStrip`
  - `func operatorStrip(focusedOn reviewKind: String?) -> ClawMissionRunOperatorStrip`
- 派生来源只允许使用已有 presentation 数据：
  - `phaseTitle`、`progressCurrent`、`progressTotal`、result counts。
  - `artifactEvidenceIndex(focusedOn:)`。
  - `reviewReadinessSummary(focusedOn:)`。
  - `nextReviewAction(focusedOn:)`。
  - 当前有效聚焦项。
- Mission Run UI 在 stage track / progress 附近展示 Operator Strip，优先服务 iPad/regular 工作台，但 compact iPhone 也必须不溢出、不遮挡、不破坏现有单栏。
- `next` lane 如有可聚焦 `reviewKind`，可用 `Button` 聚焦对应复核；其他 lane 不执行 Gateway 动作，不打开 artifact payload。
- XCTest 和 Swift logic smoke 覆盖 idle、正常 mission、focused delivery、Shell 高风险、lane 顺序、聚焦状态和敏感字符串不外显。
- 同步 README、协议、flow/flowchart、测试说明和 update_log。

## 非目标

- 不新增 action kind、event kind、artifact kind、schema 字段、envelope 字段或 Gateway API。
- 不改 Gateway 执行权限、allowlist、Shell 策略、浏览器策略、文件 workspace 策略或最终提交闸门。
- 不读取 artifact `reference`，不打开 Gateway `file://` payload，不展示 raw artifact 内容。
- 不展示 raw URL、search query、HTML/page text、path、workspace/sessionWorkspace、文件名/文件内容/diff/patch、raw command、binary/args、cwd、stdout/stderr、草稿正文、paste text、按键原文、token、Authorization/header、cookie、自然语言 instruction 或 `toolArguments`。
- 不把 Operator Strip 称为自动执行控制台、自动安全裁决、恶意检测、完整审计、payload viewer 或自动修复器。
- 不替代 Agent C artifact 验收，不改 Agent X 调度规则。

## 验收标准

- Mission Run 能派生并展示 4 lane Operator Strip。
- Strip 能快速表达 Gateway、证据、复核和下一步状态，并与 evidence/readiness/next action helper 保持一致。
- `next` lane 只聚焦 review kind，不打开 payload，不展示 raw URL/path/command/stdout/stderr/diff/token/header/toolArguments。
- iPhone compact 和 iPad regular 多栏工作台复用同一行为且不溢出。
- XCTest、LogicSmoke、本地轻量检查和云端 CI artifact 复判通过。

## 风险与禁止项

- 风险：Operator Strip 重复已有长卡信息。实现必须是短态势带，不要复制完整 Evidence/Readiness/Next 文案。
- 风险：把 lane 误解为自动执行按钮。只有 `next` lane 可聚焦已有复核，不能执行 Gateway 动作。
- 风险：色彩或图标成为唯一状态来源。必须提供文字和 VoiceOver label。
- 禁止把项目方向改成法律/法务产品。
- 禁止自然语言直连 Shell、AppleScript 或桌面动作。
- 禁止绕过 allowlist、审批闸门、workspace 限制和 artifact 审计。
- 禁止把旧 artifact、旧 output 或本地输出冒充最新云端结果。
