## 背景

Claw 的方向是 OpenClaw 式电脑接管智能体：手机端负责计划、审批、风险展示、envelope 发送和事件/artifact 复核，桌面 Gateway 在用户授权和策略 allowlist 内执行或模拟执行。AgentTrace 是当前最接近“智能体回合思考”的 artifact，但它的手机端摘要还缺少一个稳定状态：当前是缺证据、等审批、可安全继续、已完成，还是已停止。

本轮只增加 metadata-only 的 handoff 状态和手机端展示/排序，不新增 Gateway 权限，不读取 artifact payload，也不把 AgentTrace 下一步建议变成自动执行计划。

## 目标

- Gateway `runAgentLoop` 在 `agentTrace` artifact event metadata 上新增固定枚举字符串 `handoffStatus`。
- `handoffStatus` 只从既有安全字段派生：
  - `readiness.canContinue`
  - `selectedNextAction.kind`
  - `selectedNextAction.requiresApproval`
  - `riskTags`
  - `stopReason`
- Swift `ClawAgentTraceReviewSummary` 解析并展示 `handoffStatus`。
- Mission Run 复核优先队列把 `handoffStatus` 纳入 AgentTrace 是否可行动的判断。
- SwiftUI `ClawAgentTraceReviewRow` 增加一个 handoff 状态 chip，帮助 iPad/宽屏工作台快速判断当前交接状态。
- Simulator 的 `agentTrace` metadata 同步包含 `handoffStatus`，保证无 live Gateway 时可验证。
- XCTest、Swift logic smoke、Gateway direct/WebSocket smoke 覆盖 metadata、UI summary 派生和敏感字符串不外显。
- 同步 README、协议、flow/flowchart、测试说明和 update_log。

- 不改变 `runAgentLoop` artifact payload 的既有结构，除非为了在 artifact 内部记录同名 `handoffStatus` 且保持向后兼容。
- 不改 Gateway 执行权限、allowlist、Shell 策略、浏览器策略、文件 workspace 策略或最终提交闸门。
- 不读取 artifact `reference`，不打开 Gateway `file://` payload，不展示 raw artifact 内容。
- 不展示 raw URL、search query、HTML/page text、path、workspace/sessionWorkspace、文件内容/diff/patch、raw command、binary/args、cwd、stdout/stderr、草稿正文、paste text、按键原文、token、Authorization/header、cookie、自然语言 instruction 或 `toolArguments`。
- 不把 `handoffStatus` 称为自动执行 readiness、自动安全裁决、完整审计、payload viewer 或自动修复器。
- 不替代 Agent C artifact 验收，不改 Agent X 调度规则。

## 建议状态枚举

使用固定 ASCII 枚举，避免 UI 解析自然语言：

- `needs-evidence`：`readiness.canContinue == false`。
- `waiting-for-approval`：`selectedNextAction.requiresApproval == true` 或 `stopReason == "approval-required"`。
- `final-submit-review`：`stopReason == "final-submit"` 或 `riskTags` 包含 `final-submit-gate`。
- `blocked`：`stopReason` 是 `destructive`、`external` 或其他需要停下处理的非 `none` / `complete` 值。
- `ready-to-continue`：有可继续下一步且无需审批。
- `complete`：`selectedNextAction.kind == "none"` 或 `stopReason == "complete"`。

如果多个条件同时成立，建议优先级：`needs-evidence` > `final-submit-review` > `waiting-for-approval` > `blocked` > `complete` > `ready-to-continue`。

## 验收标准

- Gateway `agentTrace` artifact event metadata 包含 `handoffStatus`，且状态来自固定枚举。
- 手机端 Mission Run、Gateway session 和 result row 能展示 handoff 状态。

## 风险与禁止项

- 风险：`handoffStatus` 被误读为自动执行 readiness。文档和 UI 必须说明它是复核状态，不是执行授权。
- 风险：状态优先级不稳定导致测试脆弱。使用固定 helper 和固定枚举。
- 风险：metadata 键增加会破坏 smoke 的 allowed key 断言。必须同步 direct/WebSocket smoke。
- 禁止把项目方向改成法律/法务产品。
- 禁止自然语言直连 Shell、AppleScript 或桌面动作。
- 禁止绕过 allowlist、审批闸门、workspace 限制和 artifact 审计。
- 禁止把旧 artifact、旧 output 或本地输出冒充最新云端结果。
