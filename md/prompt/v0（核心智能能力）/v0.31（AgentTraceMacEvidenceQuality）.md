# v0.31 / AgentTrace Mac Evidence Quality

## 版本号

- 版本：v0.31
- 阶段：v0（核心智能能力）
- 任务名：AgentTrace Mac Evidence Quality

## 版本分配依据

v0.30 已为 `agentTrace` 增加固定 handoff 状态，并让手机端可复核智能体下一步交接。下一步应提升 mac Gateway 的证据判断质量：让 `runAgentLoop` 区分真实可用证据、降级证据和缺失证据，避免把 dry-run、window metadata、不可用或失败的 mac 观察结果误判成完整可继续证据。

## 背景

当前 `runAgentLoop` 的 readiness 主要基于 artifact count。只要 `screenObservation` 或 `accessibilityTree` 有 artifact，就可能进入 `satisfiedSignals`。但 mac Gateway 在默认 dry-run、Accessibility 未授权、非 macOS、窗口 metadata 或策略关闭时也会写审计 artifact；这些 artifact 对人工复核有价值，但不应等同于真实可操作 UI 证据。

## 目标

1. Gateway `runAgentLoop` 为每个 input source 派生 evidence quality：
   - `satisfied`：真实或可操作的结构化证据。
   - `degraded`：dry-run、window metadata、blocked、failed、unavailable、not-requested 等降级证据。
   - `missing`：完全没有该 source artifact。
2. `decisionChecklist[].status` 支持 `satisfied`、`degraded`、`missing`。
3. `readiness.satisfiedSignals` 只包含 satisfied source。
4. 新增 `readiness.degradedSignals`，并把 degraded source 计入风险标签。
5. `agentTrace` artifact metadata 增加安全字符串 `degradedSignals`，不包含 payload、路径、命令、URL、token 或 `toolArguments`。
6. direct/WebSocket smoke 覆盖 degraded signal、metadata key 和 trace JSON 一致性。
7. 同步 README、协议、flow/flowchart、测试说明和 update log。

## 非目标

- 不新增 schema/action/artifact/event kind。
- 不新增真实 Accessibility 点击、输入、Playwright/browser-use、桌面最终提交或 Shell 权限。
- 不让自然语言直接进入 Shell、AppleScript、浏览器或桌面自动化。
- 不改变 iOS UI，本轮只让现有 AgentTrace metadata 和 smoke 具备质量分层。
- 不读取 Gateway `file://` payload，不在 UI 或 metadata 中展示 raw URL/path/command/stdout/stderr/diff/token/header/workspace path/`toolArguments`。

## 当前架构依据

- `Tools/claw-gateway-server.mjs`：
  - `runAgentLoop` 调用 `summarizeArtifactContext`、`makeAgentDecisionChecklist`、`makeAgentReadiness`、`agentTraceMetadata`。
  - `rememberArtifact` 已保存 browser/screen/accessibility/file/shell/message artifact payload 到 session context。
  - `observeScreen` 和 `accessibilityTree` payload 已有 `mode`、policy、platform 等安全字段。
- `Tools/claw-gateway-direct-smoke.mjs` 和 `Tools/claw-gateway-smoke.mjs` 已断言 AgentTrace readiness/checklist/handoff metadata。
- 文档已声明 `agentTrace` 只解释已有证据和下一步建议，不能绕过 allowlist/审批。

## 实现步骤

1. 在 `Tools/claw-gateway-server.mjs` 中扩展 artifact context summary：
   - 保留现有 `*Count` 字段。
   - 新增内部安全质量摘要，例如 `signalQualities` 或 helper，按 source 判断 satisfied/degraded/missing。
   - `screenObservation`：`screen-capture` 可 satisfied；`dry-run`、`screen-capture-failed`、`screen-capture-unavailable`、window-only placeholder 等 degraded。
   - `accessibilityTree`：`accessibility-summary` 可 satisfied；`dry-run`、`window-metadata`、`not-requested`、`accessibility-unavailable`、`accessibility-failed` degraded。
   - `browserTrace`：`local-html`、`network-fetch` 可 satisfied；`dry-run`、`network-blocked` degraded。
   - `fileDiff`：workspace write 成功可 satisfied；blocked/failed audit 不应被记为 fileDiff count。
   - `commandOutput`：结构化 dry-run/allowlist 输出仍是 command evidence，可维持 satisfied；缺失为 missing。
   - `messageDraft`：存在 draft 可 satisfied；缺失为 missing。
2. 更新 `makeAgentDecisionChecklist`：
   - 使用 quality helper 返回 `status`、`count`、`reason`。
   - reason 只使用固定 source/status/count 文案，不含 payload。
3. 更新 `makeAgentReadiness`：
   - satisfied 只来自 `status === "satisfied"`。
   - 新增 degradedSignals。
   - score 按 satisfied/total 计算；degraded 不加分。
   - `canContinue` 继续要求 score 足够且存在 evidence row。
4. 更新 `makeAgentRiskTags`：
   - 对 degraded source 加 `degraded-<source>` 风险标签。
5. 更新 `makeAgentHandoffSummary`：
   - 用固定短语提到 degraded signals，保持 metadata-only。
6. 更新 `agentTraceMetadata`：
   - 新增 `degradedSignals`。
   - 保持 metadata key 白名单稳定，值来自 source enum join。
7. 更新 direct/WebSocket smoke：
   - direct smoke 的默认 dry-run screen/accessibility 应产生 degraded signals。
   - 断言 decisionChecklist 中 screen/accessibility 为 degraded，browser/file/command 为 satisfied，messageDraft 为 missing。
   - 断言 metadata `degradedSignals` 与 trace 一致且只包含安全枚举。
   - WebSocket smoke 断言 metadata key 和 degradedSignals 兼容；若该 flow 不包含 degraded source，允许空字段不出现或按实现精确断言。
8. 更新文档和 update log。

## 验收标准

- `agentTrace.readiness.degradedSignals` 存在并只含固定 source enum。
- dry-run/window metadata Accessibility 不再进入 `satisfiedSignals`，而进入 `degradedSignals`。
- direct/WebSocket smoke 断言 metadata 和 trace JSON 一致。
- `handoffStatus` 仍保持 v0.30 固定枚举行为。
- 所有新增 metadata 不包含 raw payload、路径、URL、命令、输出、diff、token、header 或 `toolArguments`。
- 本地轻量检查通过，最新 `origin/main` GitHub Actions artifact 由 Agent C 复判通过。

## 风险

- 如果 score 从 count-based 改为 quality-based，部分 smoke 的 readiness score 会下降；测试必须显式接受新语义。
- 旧手机端只解析已知 metadata，新增 `degradedSignals` 不应破坏现有 Swift 解析；本轮不要求 UI 展示它。
- evidence quality 是启发式，不是完整 macOS UI 可控性证明。

## 禁止项

- 禁止把 dry-run 或失败 artifact 伪装成真实可操作证据。
- 禁止绕过 allowlist、workspace 限制、审批闸门或最终提交闸门。
- 禁止把自然语言 instruction 变成可执行命令或 AppleScript。
- 禁止删除 smoke 断言来让测试通过。
- 禁止把旧 run、旧 artifact 或本地输出冒充云端验收。
