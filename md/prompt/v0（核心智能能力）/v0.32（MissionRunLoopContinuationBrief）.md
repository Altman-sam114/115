# v0.32 / Mission Run Loop Continuation Brief

## 版本号

- 版本：v0.32
- 阶段：v0（核心智能能力）
- 任务名：Mission Run Loop Continuation Brief

## 版本分配依据

v0.31 已让 Gateway `agentTrace` 区分 satisfied、degraded 和 missing 证据质量。下一步应把这些安全 metadata 带进 iPad/Mission Run 首屏，帮助操作者快速判断 agent loop 是可继续、需要证据、等待审批、最终提交复核、阻断还是已完成。

## 背景

当前 Mission Run 已有 Operator Strip、Artifact 证据索引、复核态势摘要、下一步复核行动和 AgentTrace 详情 row，但用户要判断“这个 loop 能不能继续”仍要在多个区域之间跳转。需要一个紧凑的 continuation brief，把 AgentTrace handoff、readiness、degraded/missing 证据、下一步动作和审批要求汇总为固定安全字段。

## 目标

1. Swift 端 `ClawAgentTraceReviewSummary` 解析 v0.31 metadata `degradedSignals`，并继续使用安全 list 脱敏。
2. 新增 `ClawMissionRunLoopContinuationSummary`，由 `ClawMissionRunSummary` 从现有 AgentTrace review 和 Mission Run 状态派生。
3. 新增 `ClawMissionRunLoopContinuationBriefView`，在 Mission Run 面板中显示：
   - handoff 状态或 metadata 待同步。
   - readiness score 或证据待同步。
   - satisfied/degraded/missing 计数。
   - selected next action kind。
   - 是否需要审批或人工复核。
   - 仅允许“聚焦 AgentTrace”按钮，不执行 Gateway 动作。
4. XCTest 和 Swift logic smoke 覆盖 idle、final-submit-review、ready-to-continue、metadata 缺失和敏感字符串不外显。
5. 同步 README、协议、flow/flowchart、测试说明和 update_log。

## 非目标

- 不新增 schema/action/artifact/event kind。
- 不改 Gateway JS，不改真实 macOS Accessibility、浏览器、Shell、桌面 App 权限。
- 不让 continuation brief 发送、审批、重试、执行或自动继续 loop。
- 不读取 Gateway `file://` payload。
- 不展示 raw URL/path/command/stdout/stderr/diff/token/header/workspace path/`toolArguments`。

## 当前架构依据

- `ClawAgentTraceReviewSummary` 已从 `agentTrace` artifact metadata 解析 readiness、missingSignals、selectedNextAction、riskTags、stopReason、handoffStatus 和 handoffSummary。
- `ClawMissionRunSummary` 已有 computed helpers：`artifactEvidenceIndex(focusedOn:)`、`reviewReadinessSummary(focusedOn:)`、`nextReviewAction(focusedOn:)`、`operatorStrip(focusedOn:)`。
- `ClawMissionRunPanel` 已用 `focusedReviewKind` 支持复核聚焦，AgentTrace 的 detail review kind 是 `agent-trace`。
- `Tools/LogicSmoke.swift` 和 `ClawTests/ClawTests.swift` 已覆盖 AgentTrace handoff 状态和敏感 metadata 脱敏。

## 实现步骤

1. 在 `Claw/Core/ClawModels.swift`：
   - 给 `ClawAgentTraceReviewSummary` 增加 `degradedSignals: [String]`。
   - 从 metadata `degradedSignals` 解析安全 list。
   - 新增 `ClawMissionRunLoopContinuationSummary`。
   - 增加 `loopContinuationSummary` / `loopContinuationSummary(focusedOn:)` helper。
   - 逻辑建议：
     - 无 AgentTrace：不可 review，不能继续，等待 Gateway evidence。
     - metadata 缺失：metadata gap，需要人工复核，可聚焦 AgentTrace。
     - `handoffStatus == ready-to-continue` 且 `readinessCanContinue == true` 且无审批要求：`canContinueLoop == true`，不要求人工动作。
     - `needs-evidence`、`waiting-for-approval`、`final-submit-review`、`blocked`、metadata 缺失、missing/degraded signal 非空、selected action requires approval：`requiresHumanAction == true`。
     - `complete`：不可继续，但可 review。
2. 在 `Claw/Views/ContentView.swift`：
   - 在 `ClawMissionRunPanel` 计算并展示 continuation brief，建议放在 Operator Strip 后、metrics 前。
   - 卡片使用固定安全文案、`Label`、44pt 按钮、VoiceOver label/hint。
   - 按钮只能调用已有 `focusReviewKind("agent-trace")`。
3. 在 `Claw/Services/ClawStore.swift`：
   - 如有必要，更新 simulator `agentTraceMetadata()`，加入安全 `degradedSignals` fixture。
4. 在 `ClawTests/ClawTests.swift`：
   - 在 AgentTrace/Mission Run 测试中断言 degradedSignals。
   - 新增或扩展 Mission Run continuation summary 测试。
   - 敏感 metadata 测试加入恶意 degradedSignals 并断言不外显。
5. 在 `Tools/LogicSmoke.swift`：
   - 镜像关键 continuation summary 和 degradedSignals 断言。
6. 同步文档。

## 验收标准

- Mission Run 首屏出现 Loop 继续态势卡片。
- 无 AgentTrace 时显示待生成且不执行动作。
- final-submit-review fixture 显示需要人工复核、可聚焦 AgentTrace、不能自动继续。
- ready-to-continue fixture 显示可继续，但仍不执行 Gateway 动作。
- degraded/missing signal count 从 AgentTrace metadata 安全派生。
- 恶意 degradedSignals / handoff / summary 不泄露 token、Authorization、`file://`、本地路径或 `toolArguments`。
- 本地轻量检查通过，最新 `origin/main` GitHub Actions artifact 由 Agent C 复判通过。

## 风险

- 这是 presentation layer 状态，不代表 Gateway readiness 或自动执行授权。
- v0.31 之前的旧 artifact 没有 degradedSignals，必须兼容空值。
- UI 文案必须保持短小、可换行，不能遮挡 Mission Run 既有内容。

## 禁止项

- 禁止把 continuation brief 做成自动继续、自动审批或自动执行按钮。
- 禁止读取 artifact payload 或展示 raw URL/path/command/output/diff/token/header/workspace path/`toolArguments`。
- 禁止删除现有 Mission Run、AgentTrace、smoke 或脱敏断言来让测试通过。
- 禁止改变 Gateway 执行权限或 schema。
