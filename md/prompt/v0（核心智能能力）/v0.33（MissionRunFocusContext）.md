# v0.33（MissionRunFocusContext）

## 版本号

v0.33

## 版本分配依据

v0.32 已完成 Mission Run Loop 继续态势。v0.33 继续沿用手机端 presentation-layer 路线，在不改 Gateway 协议、不新增 action/artifact/event/schema 的前提下，补齐 iPad/compact Mission Run 的“当前聚焦上下文”。

## 背景

当前 Mission Run 已有 Operator Strip、Loop continuation brief、Artifact 证据索引、复核态势摘要、下一步复核行动和复核优先队列。用户点击聚焦后，详细 row 会被过滤，但首屏缺少一个稳定上下文说明：当前聚焦的是哪类复核、是否有详细 row、证据/metadata 是否覆盖、下一步应该只查看还是先处理审批/阻断。

同时，Loop continuation 卡片内部有按钮，但父容器使用 `.accessibilityElement(children: .combine)`，可能让 VoiceOver 合并内部按钮，降低可达性。

## 目标

- 新增 `ClawMissionRunFocusContextSummary`，由 `ClawMissionRunSummary` 从当前有效 `focusedReviewKind`、复核优先队列、Artifact 证据索引、复核态势和下一步复核行动派生。
- 在 Mission Run 中展示聚焦上下文卡，位置靠近复核优先队列和详细 row。
- 对无聚焦、有效 detail 聚焦、状态级聚焦和过期聚焦提供稳定 fallback。
- 聚焦上下文只提供清除聚焦或聚焦 detail 的 UI 行为，不执行 Gateway 动作，不审批、不发送、不重试、不自动继续。
- 修复至少 `ClawMissionRunLoopContinuationBriefView` 的父级 accessibility combine 问题，确保内部“聚焦 AgentTrace”按钮对 VoiceOver 保持可达。

## 非目标

- 不新增或修改 `claw.computer.control.v1` schema。
- 不新增 Gateway action kind、artifact kind 或 event kind。
- 不修改 `Tools/claw-gateway-server.mjs`。
- 不读取 artifact `reference` 或 Gateway `file://` payload。
- 不展示 raw URL/path/command/stdout/stderr/diff/token/header/workspace path/`toolArguments`。
- 不实现自动安全裁决、自动继续、后台保活、配对或真实 macOS Accessibility bridge。

## 当前架构依据

- `Claw/Core/ClawModels.swift` 已有 `ClawMissionRunSummary`、`reviewPriorityQueue`、`artifactEvidenceIndex(focusedOn:)`、`reviewReadinessSummary(focusedOn:)`、`nextReviewAction(focusedOn:)`、`focusUsesDetailReview(_:)`、`reviewPriorityItem(focusedOn:)` 和 `availableDetailReviewKinds`。
- `Claw/Views/ContentView.swift` 的 `ClawMissionRunPanel` 持有本地 `focusedReviewKind`，并把有效聚焦传入各派生摘要。
- `ClawTests/ClawTests.swift` 和 `Tools/LogicSmoke.swift` 已覆盖 Mission Run presentation-layer 派生模型和敏感字符串不外显。

## 实现步骤

1. 在 `Claw/Core/ClawModels.swift` 增加 `ClawMissionRunFocusContextSummary`。
2. 为 `ClawMissionRunSummary` 增加 `focusContextSummary(focusedOn:)` 和无参便捷属性。
3. 聚焦上下文应包含：title、status、guidance、icon、focusedReviewKind、focusedReviewTitle、primaryReviewKind、primaryButtonTitle、canFocusDetailReview、canClearFocus、hasEvidence、hasMetadataGap、requiresHumanAction、isReviewable。
4. 逻辑优先级：
   - 无聚焦：提示可从优先队列选择复核项；如果已有下一步行动，可把 primary 指向下一步 detail。
   - 有队列项聚焦：使用该队列项 title/status/actionHint/requiresHumanAction。
   - 有 detail-only 聚焦：使用 detail review title，提示查看安全摘要。
   - 状态级或过期聚焦：不收窄详情，提示保持全量复核。
5. 在 `ContentView.swift` 增加 `ClawMissionRunFocusContextView` 并接入 `ClawMissionRunPanel`。
6. 修复带内部按钮卡片的 `.accessibilityElement(children: .combine)`，至少 Loop continuation 改为 `.contain`。
7. 扩展 XCTest 和 Swift logic smoke，覆盖 idle、delivery 聚焦、status-only 聚焦、shell 聚焦、敏感字符串不外显。
8. 同步 README、协议、flow、flowchart、test 和 update_log。

## 验收标准

- Mission Run 能在无聚焦、detail 聚焦、状态级聚焦和 shell 高风险聚焦时给出一致的聚焦上下文。
- 聚焦上下文按钮只改变本地聚焦状态，不触发 Gateway、approval、send、retry 或 continue。
- 所有新增可见字符串不包含 raw URL/path/command/stdout/stderr/diff/token/header/workspace path/`toolArguments`。
- Swift logic smoke 和本地静态检查通过。
- 最新 `origin/main` GitHub Actions run 通过，Agent C artifact 复判通过。

## 风险

- UI 文案过多可能压缩 compact 布局；应使用动态字体、可换行文本和 44pt 触控区。
- 与既有复核态势、下一步行动文案重复；应让 Focus Context 只回答“当前聚焦是什么、怎么退出或看详情”。
- 误把状态级聚焦当成 detail 聚焦会隐藏必要详情；必须保留全量详情。
