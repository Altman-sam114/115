# v0.34（iPadMissionReviewDetailDock）

## 版本号

v0.34

## 版本分配依据

v0.33 已完成 Mission Run Focus Context 聚焦上下文。v0.34 继续沿用手机端 presentation-layer 路线，在不改 Gateway 协议、不新增 action/artifact/event/schema 的前提下，把 iPad/mac regular 多栏工作台中的 Mission 复核详情从左侧窄栏解耦出来，让宽屏用户能在右侧工作区顶部直接查看当前聚焦或下一步复核详情。

## 背景

当前 iPad/regular 布局左侧放命令输入和完整 Mission Run 面板，右侧放计划、Gateway 任务、会话、权限和日志。Mission Run 内部已有 Operator Strip、Loop 继续态势、Focus Context、Artifact 证据索引、复核态势、下一步复核行动、复核优先队列和详细复核 row，但聚焦状态只存在于 `ClawMissionRunPanel` 内部 `@State`。因此宽屏右侧不能感知用户在左侧选择的复核项，也不能把当前重点复核内容提升到更宽的操作区。

## 目标

- 将 Mission Run 的 `focusedReviewKind` 状态提升到 compact/workbench 布局层，compact iPhone 保持单栏行为，regular iPad/mac 左右栏共享同一聚焦状态。
- 抽出可复用的 Mission 详细复核 rows 视图，左侧 Mission Run 和右侧 Dock 共用同一套安全 metadata summary，不读取 artifact payload。
- 在 regular 右侧 `PhoneAgentReviewColumn` 顶部新增 “Mission 复核详情” Dock，展示当前聚焦、下一步复核或全量详情 fallback。
- 新增轻量 presentation summary `reviewDetailDockSummary(focusedOn:)`，用于描述右侧 Dock 当前展示模式：idle、全量详情、单项详情、状态级聚焦全量回退、过期聚焦回退。
- Dock 只提供本地聚焦或清除聚焦，不执行 Gateway 动作，不审批、不发送、不重试、不自动继续。

## 非目标

- 不新增或修改 `claw.computer.control.v1` schema。
- 不新增 Gateway action kind、artifact kind 或 event kind。
- 不修改 `Tools/claw-gateway-server.mjs`。
- 不读取 artifact `reference` 或 Gateway `file://` payload。
- 不展示 raw URL/path/command/stdout/stderr/diff/token/header/workspace path/`toolArguments`。
- 不实现自动安全裁决、自动继续、后台保活、配对、真实 macOS Accessibility bridge 或完整 artifact payload viewer。

## 当前架构依据

- `Claw/Views/ContentView.swift` 的 `PhoneAgentWorkbenchLayout` 已按 regular horizontal size class 使用左右两栏。
- `ClawMissionRunPanel` 已有本地 `focusedReviewKind`、`activeFocusedReviewKind(for:)`、`focusReviewKind(_:)` 和 `clearFocusedReviewKind()`。
- `ClawMissionRunSummary` 已有 `availableDetailReviewKinds`、`detailReviewKinds(focusedOn:)`、`shouldShowDetailReview(_:focusedOn:)`、`focusContextSummary(focusedOn:)`、`reviewReadinessSummary(focusedOn:)` 和 `nextReviewAction(focusedOn:)`。
- `ClawTests/ClawTests.swift` 和 `Tools/LogicSmoke.swift` 已覆盖 Mission Run presentation-layer 派生模型和敏感字符串不外显。

## 实现步骤

1. 在 `Claw/Core/ClawModels.swift` 增加 `ClawMissionRunReviewDetailDockSummary`。
2. 为 `ClawMissionRunSummary` 增加：
   - `activeReviewFocus(from:)`：把 raw focus 收敛为当前队列项或 detail kind，未知/过期返回 nil。
   - `reviewDetailDockSummary(focusedOn:)`：派生 Dock title/status/guidance/icon、active/focused kind、展示的 detail kinds、是否单项聚焦、是否可清除、是否过期聚焦、是否 reviewable。
3. 修改 `ContentView.swift`：
   - `PhoneAgentCompactLayout` 持有 `@State focusedReviewKind` 并传给 `ClawMissionRunPanel`。
   - `PhoneAgentWorkbenchLayout` 持有同一 `@State focusedReviewKind`，左侧 `ClawMissionRunPanel` 和右侧 `PhoneAgentReviewColumn` 共享。
   - `ClawMissionRunPanel` 改用 `@Binding focusedReviewKind`，并使用 `summary.activeReviewFocus(from:)`。
   - 抽出 `ClawMissionDetailReviewRows`，复用既有 detail row 条件。
   - 新增 `ClawMissionReviewDetailDockView`，只在 regular 右侧展示 Focus Context + detail rows + 清除聚焦入口；无证据时显示 idle/fallback 说明。
4. 扩展 XCTest 和 Swift logic smoke，覆盖 idle Dock、delivery 聚焦单项详情、status-only 聚焦全量回退、stale 聚焦回退、shell 聚焦和敏感字符串不外显。
5. 同步 README、协议、flow、flowchart、test 和 update_log。

## 验收标准

- compact iPhone 仍为单栏 Mission Run；regular iPad/mac 左侧 Mission Run 的聚焦选择能同步到右侧 Mission 复核详情 Dock。
- Dock 在无证据、无聚焦、detail 聚焦、状态级聚焦、过期聚焦和 shell 高风险聚焦时都有稳定 fallback。
- Dock 只显示既有安全 metadata summary 和固定文案，不读取 Gateway `file://` payload。
- Dock 按钮只改变本地聚焦状态，不触发 Gateway、approval、send、retry 或 continue。
- 所有新增可见字符串不包含 raw URL/path/command/stdout/stderr/diff/token/header/workspace path/`toolArguments`。
- Swift logic smoke、本地静态检查和本地无签名 iOS build 通过。
- 最新 `origin/main` GitHub Actions run 通过，Agent C artifact 复判通过。

## 风险

- 状态提升可能影响 compact 单栏聚焦行为；必须保留 `@State` 在 compact layout 中。
- 左侧 Mission Run 与右侧 Dock 可能重复显示 detail row；regular 下这是刻意的宽屏复核增强，但 Dock 应放在右侧顶部并用简洁说明避免拥挤。
- 状态级聚焦或过期聚焦不能隐藏 detail rows；必须回退全量详情。
