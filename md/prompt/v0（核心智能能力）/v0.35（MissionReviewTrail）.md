# v0.35 / Mission Review Trail

## 版本号与分配依据

- 版本号：v0.35。
- 分配依据：v0.34 已完成 iPad/mac 右侧 Mission Review Detail Dock；下一轮应补齐 Dock 与 Mission Run 首屏之间的复核路径说明，让用户在 mac/iPad 宽屏和 iPhone compact 上都能快速看清证据覆盖、metadata 状态、最高优先复核和下一步动作。

## 背景

Claw 当前方向是 OpenClaw 式电脑智能体：手机端负责计划、风险展示、审批、envelope 发送和事件查看，真实电脑操作只发生在用户授权的桌面 Gateway。现有 Mission Run 已有 Operator Strip、Loop 继续态势、Focus Context、Review Detail Dock、Artifact 证据索引、复核态势摘要、下一步复核行动和详细 metadata-only row，但缺少一条把“证据覆盖 -> metadata 状态 -> 优先复核 -> 下一步”串起来的短复核路径。

## 目标

1. 新增手机端 presentation-layer 的 Mission Review Trail 派生摘要。
2. Trail 只复用现有安全摘要：Artifact 证据索引、复核态势摘要、下一步复核行动、复核优先队列、有效聚焦和 detail review helper。
3. Mission Run 面板在 Artifact 证据索引后展示 Trail；iPad/mac 右侧 Review Detail Dock 也展示同一 Trail，跟随左侧聚焦变化。
4. Trail 中的交互只能聚焦已有复核项或保持状态展示，不执行 Gateway 动作，不审批、不发送、不重试、不自动继续。
5. 补 XCTest、Swift logic smoke、README、协议、flow/flowchart、测试说明和 update_log。

## 非目标

- 不新增或修改 `claw.computer.control.v1` schema。
- 不新增 action kind、artifact kind、Gateway event kind。
- 不修改 Gateway JS 执行器、allowlist、Shell、AppleScript、浏览器或桌面控制能力。
- 不做完整审计链、forensic timeline、artifact payload viewer、自动安全裁决或自动继续机制。
- 不读取 artifact `reference`，不打开 Gateway `file://` payload。

## 验收标准

- Trail 在 idle 时显示待生成，不产生可执行动作。
- 有 Gateway 证据时 Trail 能稳定展示证据覆盖、metadata 状态、最高优先复核和下一步复核。
- 聚焦 detail review、状态级 review、过期 review 时表现稳定并只改变本地聚焦。
- UI 不展示 raw token、Authorization/header、`toolArguments`、`file://`、本地路径、URL/path/command/stdout/stderr/diff、artifact reference 或 payload。
- 本地轻量检查通过，最新 `origin/main` CI artifact 复判通过。

## 风险与禁止项

- 禁止把 Trail 描述为完整证据链、审计链、forensic timeline 或自动安全裁决。
- 禁止读取 `auditTrail` 原文或 artifact `reference` 来拼 UI。
- 禁止为了补 Trail 扩大 Gateway 权限或执行面。
- 禁止把自然语言直接交给 Shell、AppleScript 或桌面自动化。
- 禁止把无关改动混入 v0.35。
