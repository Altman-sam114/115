# v0.36 / Mission Approval Queue

## 版本号

- 版本：v0.36
- 阶段：v0（核心智能能力）
- 任务名：Mission Approval Queue

## 版本分配依据

v0.35 已完成 Mission Review Trail，把证据、metadata、优先复核和下一步串成短路径。当前 iPad/mac 工作台仍缺少一个专门回答“哪些确认点要先处理”的展示层队列。v0.36 继续沿用 presentation-layer 路线，只把已有计划审批、Gateway 等待确认、最终提交闸门和 AgentTrace 等待审批整理为手机端复核队列，不新增协议或执行能力。

## 背景

Claw 的方向是 OpenClaw 式电脑智能体：iOS 端负责输入、计划、风险展示、审批、envelope 发送和事件查看；真实电脑操作只发生在授权桌面 Gateway。当前已有 Mission Run、Operator Strip、Loop 继续态势、Focus Context、Review Detail Dock、Review Trail、Artifact 证据索引、复核态势、下一步复核行动和详细 metadata-only 安全摘要。审批相关信号目前分散在 `ClawMobileTask.approvalCount/actions`、`ClawGatewaySession.results.waitingForApproval`、Delivery Safety metadata、AgentTrace handoff 和复核优先队列中。

## 目标

1. 新增 Mission Approval Queue 展示层模型，用固定、脱敏、安全的字段列出当前待确认/应复核的审批点。
2. 队列来源只能是已有 task action、session action result、Delivery Safety review、AgentTrace review、复核优先队列和 Mission Run 状态。
3. 在 compact Mission Run 和 iPad/mac regular 右侧 Dock 中展示该队列，帮助用户按顺序处理手机审批、Gateway 等待确认、最终提交闸门和 AgentTrace 审批等待。
4. 队列按钮只能聚焦已有 detail review 或状态项；不得执行 Gateway 动作、审批、发送、重试、继续或自动通过。
5. XCTest 和 Swift logic smoke 覆盖 idle、待发送审批、发送后 Gateway 等待确认、最终提交、AgentTrace 等待审批、排序稳定、聚焦状态和敏感字符串不外显。

## 非目标

- 不新增 `claw.computer.control.v1` schema 字段。
- 不新增 action kind、artifact kind、Gateway event kind。
- 不修改 Gateway JS、执行权限、allowlist、Shell/AppleScript/桌面控制策略。
- 不读取 Gateway `file://` artifact payload，不读取 `auditTrail` 原文。
- 不实现自动审批、自动发送、自动重试、自动继续或自动安全裁决。
- 不做完整 artifact payload viewer、forensic timeline 或审计链。

## 当前架构依据

- `Claw/Core/ClawModels.swift` 定义 `ClawMissionRunSummary`、复核优先队列、Review Trail、Loop 继续态势、Delivery/AgentTrace review summary。
- `Claw/Services/ClawStore.swift` 负责从当前 task/session/review metadata 派生 `missionRunSummary`。
- `Claw/Views/ContentView.swift` 中 `ClawMissionRunPanel` 和 `ClawMissionReviewDetailDockView` 共享 `focusedReviewKind`。
- `ClawTests/ClawTests.swift` 和 `Tools/LogicSmoke.swift` 已覆盖 Mission Run 派生、聚焦和敏感字符串不外显。

## 实现步骤

1. 在 `ClawModels.swift` 增加：
   - `ClawMissionRunApprovalQueueItem`
   - `ClawMissionRunApprovalQueueSummary`
   - 必要的固定 source/severity/tone 字段或复用既有 `ClawMissionRunReviewPrioritySeverity`、`ClawMissionRunOperatorLaneTone`。
2. 在 `ClawMissionRunSummary` 增加 `approvalQueue` 字段，并提供 `approvalQueueSummary(focusedOn:)` helper。
3. 在 `ClawStore.missionRunSummary` 派生 `approvalQueue`：
   - task action 中 `userConfirmation`、`gatewayApproval`、`blocked` 生成计划审批/策略阻断项。
   - session result 中 `waitingForApproval`、`failed`、`isRetryable` 生成 Gateway 结果项。
   - Delivery Safety review 的 final submit/user approval 生成最终提交项。
   - AgentTrace review 的 `waiting-for-approval`、`final-submit-review`、`selectedNextActionRequiresApproval` 生成 loop handoff 项。
   - 队列只使用 action title、kind title、approval title、result status title、summary compact status 等安全字段；不得使用 raw `toolArguments`、reference、路径、URL、stdout/stderr、diff、token/header。
4. 在 `ContentView.swift` 增加 `ClawMissionApprovalQueueView` 和 row view，插入：
   - `ClawMissionRunPanel` 中 Review Trail 之后、Review Readiness 之前。
   - `ClawMissionReviewDetailDockView` 中 Review Trail 之后、Focus Context 之前。
5. 同步测试：
   - XCTest 更新 Mission Run 派生测试和手写 `ClawMissionRunSummary` 初始化。
   - Swift logic smoke 更新同类断言。
   - 增加 visible text 敏感字符串断言覆盖 Approval Queue。
6. 同步文档：`README.md`、`Docs/claw-mobile-gateway-protocol.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`update_log.md`。

## 验收标准

- Mission Run summary 可在 idle、待审批、发送后复核、状态级/过期聚焦下稳定派生 approval queue。
- iPad/mac 右侧 Dock 与左侧 Mission Run 使用同一 `focusedReviewKind` 展示选中状态。
- 所有 queue 可见字符串不包含 raw token、Authorization/header、`toolArguments`、`file://`、本地路径、URL、stdout/stderr、diff、patch、raw command 或 workspace path。
- 本地轻量检查通过。
- 最新 `origin/main` GitHub Actions run 成功，Agent C artifact 复判通过。

## 风险和禁止项

- 禁止把 Approval Queue 描述成自动审批系统。
- 禁止新增审批绕过按钮。
- 禁止读取或展示 raw `toolArguments`、artifact reference 或 payload。
- 禁止把自然语言交给 Shell、AppleScript 或桌面自动化。
- 禁止修改 Gateway 权限、allowlist 或真实执行策略。
- 禁止混入无关重构或删除旧测试断言。
