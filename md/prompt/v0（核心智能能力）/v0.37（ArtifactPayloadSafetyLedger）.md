# v0.37 / Artifact Payload Safety Ledger

## 版本号

- 版本：v0.37
- 阶段：v0（核心智能能力）
- 任务名：Artifact Payload Safety Ledger

## 版本分配依据

v0.36 已完成 Mission Approval Queue，把审批和确认点收敛成可聚焦队列。当前 Mission Run 已能展示 Artifact Evidence Index、Review Trail、Approval Queue 和 metadata-only 详细复核，但用户还不能一眼判断“哪些 artifact payload 没有被手机端读取、哪些敏感内容被省略、metadata 覆盖是否足够”。v0.37 继续沿用 presentation-layer 路线，只从已有安全 metadata 和 `safetyFlags` 派生 Payload Safety Ledger，不新增协议或执行能力。

## 背景

Claw 的方向是 OpenClaw 式电脑智能体：iOS 端负责输入、计划、风险展示、审批、envelope 发送和事件查看；真实电脑操作只发生在授权桌面 Gateway。当前 Gateway artifact 只把 `reference`、kind、redaction 和 metadata 回传给手机端，SwiftUI 不打开 `file://` reference，也不读取 artifact payload。各类 review summary 已经解析 `metadata-only`、`artifact-payload-not-read`、`tool-arguments-omitted`、`stdout-omitted`、`diff-content-omitted` 等安全标志，但这些边界分散在各个 detail row 中。

## 目标

1. 新增 Mission Payload Safety Ledger 展示层模型，用固定、脱敏、安全字段汇总 payload 访问边界。
2. Ledger 只能从已有 review summary 的 `hasMetadata` 和 `safetyFlags` 派生；不得读取 artifact `reference`、payload、文件内容、命令输出、diff、网页正文、草稿正文或 `toolArguments`。
3. 在 compact Mission Run 和 iPad/mac regular 右侧 Dock 展示 Ledger，说明 payload 未读取、metadata-only 覆盖、敏感内容省略和 metadata 缺口。
4. Ledger row 只能聚焦已有 detail review 或状态项；不得执行 Gateway 动作、审批、发送、重试、继续或自动通过。
5. XCTest 和 Swift logic smoke 覆盖 idle、发送后正常摘要、detail 聚焦、状态级聚焦、过期聚焦、Shell 高风险聚焦和敏感字符串不外显。

## 非目标

- 不新增 `claw.computer.control.v1` schema 字段。
- 不新增 action kind、artifact kind、Gateway event kind。
- 不修改 Gateway JS、执行权限、allowlist、Shell/AppleScript/桌面控制策略。
- 不实现完整 artifact payload viewer，不打开 `file://`，不读取 workspace 文件。
- 不展示 raw path、URL、stdout/stderr、diff、patch、草稿正文、网页正文、token/header 或 `toolArguments`。
- 不做自动安全裁决、自动审批、自动执行或自动继续。

## 当前架构依据

- `Claw/Core/ClawModels.swift` 定义 `ClawMissionRunSummary`、Artifact Evidence Index、Review Trail、Approval Queue 和各类 Gateway review summary。
- 各类 review summary 已白名单解析 `safetyFlags`，并将 raw metadata 通过 `ClawArtifactMetadataDisplaySanitizer` 脱敏。
- `Claw/Views/ContentView.swift` 中 `ClawMissionRunPanel` 和 `ClawMissionReviewDetailDockView` 共享 `focusedReviewKind`。
- `ClawTests/ClawTests.swift` 和 `Tools/LogicSmoke.swift` 已覆盖 Mission Run 派生、聚焦、计数一致性和敏感字符串不外显。

## 实现步骤

1. 在 `ClawModels.swift` 增加：
   - `ClawMissionRunPayloadSafetyLedgerItem`
   - `ClawMissionRunPayloadSafetyLedgerSummary`
2. 在 `ClawMissionRunSummary` 增加 `payloadSafetyLedger` 和 `payloadSafetyLedger(focusedOn:)` helper：
   - 从 `availableDetailReviewKinds` 生成 item。
   - 每个 item 使用固定 review title/icon、`hasMetadata`、白名单 `safetyFlags`。
   - 统计 `payloadNotReadCount`、`metadataOnlyCount`、`omissionSignalCount`、`metadataPendingCount`。
   - 聚焦只接受 `activeReviewFocus(from:)` 认可的 review kind；未知聚焦回到全量摘要。
3. 在 `ContentView.swift` 增加 `ClawMissionPayloadSafetyLedgerView` 和 row view，插入：
   - `ClawMissionRunPanel` 中 Artifact Evidence Index 之后、Review Trail 之前。
   - `ClawMissionReviewDetailDockView` 中 Review Trail 之后、Approval Queue 之前。
4. 同步测试：
   - XCTest 更新 Mission Run 派生测试和手写 `ClawMissionRunSummary` 初始化。
   - Swift logic smoke 更新同类断言。
   - visible text denylist 必须覆盖 Ledger 的 title/status/guidance/item fields。
5. 同步文档：`README.md`、`Docs/claw-mobile-gateway-protocol.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`update_log.md`。

## 验收标准

- Mission Run summary 可在 idle、发送后复核、detail 聚焦、状态级/过期聚焦和 Shell 高风险聚焦下稳定派生 Payload Safety Ledger。
- iPad/mac 右侧 Dock 与左侧 Mission Run 使用同一 `focusedReviewKind` 展示选中状态。
- Ledger 只展示固定安全摘要和白名单 flag 计数，不读取或展示 raw artifact payload。
- 所有 Ledger 可见字符串不包含 raw token、Authorization/header、`toolArguments`、`file://`、本地路径、URL、stdout/stderr、diff、patch、raw command 或 workspace path。
- 本地轻量检查通过。
- 最新 `origin/main` GitHub Actions run 成功，Agent C artifact 复判通过。

## 风险和禁止项

- 禁止把 Ledger 描述成完整 artifact viewer。
- 禁止打开、读取或展示 artifact `reference` 或 payload。
- 禁止新增审批绕过按钮。
- 禁止读取或展示 raw `toolArguments`、路径、URL、命令输出、diff、网页正文或草稿正文。
- 禁止把自然语言交给 Shell、AppleScript 或桌面自动化。
- 禁止修改 Gateway 权限、allowlist 或真实执行策略。
- 禁止混入无关重构或删除旧测试断言。
