# v0.38 / Mac Agent Readiness Board

## 版本号

- 版本：v0.38
- 阶段：v0（核心智能能力）
- 任务名：Mac Agent Readiness Board

## 版本分配依据

v0.37 已完成 Artifact Payload Safety Ledger，让手机端能确认 artifact payload 边界。当前 iPad/mac 工作台已经能展示 Operator Strip、Loop 继续态势、Review Trail、Approval Queue、Payload Safety Ledger 和详细复核 row，但用户仍需要在 Capability、Accessibility、AgentTrace、Live Gateway 健康之间来回查找，才能判断桌面 Gateway 是否具备继续执行下一轮电脑任务的条件。v0.38 继续走 presentation-layer 路线，只把已有安全 metadata 派生成 Mac Agent Readiness Board，不新增协议、action、artifact 或 Gateway 权限。

## 背景

Claw 是 OpenClaw 式电脑智能体：iOS 端负责输入、计划、风险展示、审批、envelope 发送和事件查看；真实电脑操作只发生在用户授权的桌面/自托管 Gateway。当前 Gateway 已产出 capability snapshot、accessibility summary、AgentTrace readiness/handoff 和 live health 摘要，SwiftUI 也有对应 detail row。缺口在于这些 readiness 信号分散，用户无法一眼看到 mac/Gateway 执行面是否 connected、capable、observable、loop-ready，以及下一步该聚焦哪类复核。

## 目标

1. 新增 Mission Mac Agent Readiness Board 展示层模型，从现有 `ClawMissionRunSummary` 的 Gateway capability、Accessibility、AgentTrace、Live health、review readiness 和 loop continuation 派生。
2. Board 用固定、脱敏、安全字段汇总连接、能力、观察、Loop 继续和人工闸门状态。
3. 在 compact Mission Run 和 iPad/mac regular 右侧 Mission Review Detail Dock 展示同一 Board。
4. Board row 只能聚焦已有 detail review 或状态项；不得执行 Gateway 动作、审批、发送、重试、继续或自动通过。
5. XCTest 和 Swift logic smoke 覆盖 idle、发送后 readiness、detail 聚焦、状态级聚焦、过期聚焦、Shell 高风险聚焦和敏感字符串不外显。

## 非目标

- 不新增 `claw.computer.control.v1` schema 字段。
- 不新增 action kind、artifact kind、Gateway event kind。
- 不修改 Gateway JS、执行权限、allowlist、Shell/AppleScript/桌面控制策略。
- 不实现真实 macOS Accessibility bridge、Playwright/browser-use 控制器、后台保活或配对协议。
- 不打开 artifact `reference`，不读取 payload、文件内容、命令输出、diff、网页正文、草稿正文或 `toolArguments`。
- 不做自动安全裁决、自动审批、自动执行或自动继续。

## 当前架构依据

- `Claw/Core/ClawModels.swift` 已有 `ClawGatewayCapabilityReviewSummary`、`ClawGatewayAccessibilityReviewSummary`、`ClawAgentTraceReviewSummary`、`ClawGatewayLiveHealthSummary`、`ClawMissionRunLoopContinuationSummary` 和 `ClawMissionRunReviewReadinessSummary`。
- `Claw/Services/ClawStore.swift` 已在 `missionRunSummary` 汇入 capability、accessibility、AgentTrace 和 review priority queue。
- `Claw/Views/ContentView.swift` 的 `ClawMissionRunPanel` 和 `ClawMissionReviewDetailDockView` 共享 `focusedReviewKind`，既有 board row 只做本地聚焦。
- `ClawTests/ClawTests.swift` 和 `Tools/LogicSmoke.swift` 已覆盖 v0.34-v0.37 presentation-layer 派生、聚焦和敏感字符串 denylist。

## 实现步骤

1. 在 `ClawModels.swift` 增加：
   - `ClawMissionRunMacAgentReadinessItem`
   - `ClawMissionRunMacAgentReadinessBoard`
2. 在 `ClawMissionRunSummary` 增加 `macAgentReadinessBoard` 和 `macAgentReadinessBoard(focusedOn:)`：
   - 固定生成 `connection`、`capability`、`observation`、`loop`、`human-gate` 五类 row。
   - `connection` 从是否有 Gateway/session artifact 与结果状态派生，必要时只显示“等待 Gateway 事件”。
   - `capability` 从 `gatewayCapabilityReview` 派生，可聚焦 `gateway-capability`。
   - `observation` 从 `gatewayAccessibilityReview` 派生，可聚焦 `accessibility`。
   - `loop` 从 `loopContinuationSummary` / `agentTraceReview` 派生，可聚焦 `agent-trace`。
   - `human-gate` 从 `reviewReadinessSummary` / `approvalQueueSummary` 派生，优先指向可聚焦的下一复核项。
   - 统计 ready、blocked、metadata pending、human action、focused item、primary item。
   - 未知或过期聚焦必须回退到全量 Board。
3. 在 `ContentView.swift` 增加 `ClawMissionMacAgentReadinessBoardView` 和 row view：
   - 插入 `ClawMissionRunPanel` 中 Loop 继续态势后、基础指标前。
   - 插入 `ClawMissionReviewDetailDockView` 中 Review Trail 前或后，保持右侧 Dock 与左侧同一聚焦状态。
   - Row 按 `canFocusReview` 启用本地 focus，accessibility hint 必须说明不执行 Gateway 动作。
4. 同步测试：
   - XCTest 更新 Mission Run presentation 测试和手写 `ClawMissionRunSummary` 初始化。
   - Swift logic smoke 更新同类断言；大字符串组合必须拆成中间数组，避免 Swift type-check 压力。
   - visible text denylist 必须覆盖 Board title/status/guidance/item fields。
5. 同步文档：`README.md`、`Docs/claw-mobile-gateway-protocol.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`update_log.md`。

## 验收标准

- Mission Run summary 可在 idle、发送后复核、detail 聚焦、状态级/过期聚焦和 Shell 高风险聚焦下稳定派生 Mac Agent Readiness Board。
- iPad/mac 右侧 Dock 与左侧 Mission Run 使用同一 `focusedReviewKind` 展示选中状态。
- Board 只展示固定安全摘要、计数和状态，不读取或展示 raw artifact payload。
- 所有 Board 可见字符串不包含 raw token、Authorization/header、`toolArguments`、`file://`、本地路径、URL、stdout/stderr、diff、patch、raw command 或 workspace path。
- 本地轻量检查通过。
- 最新 `origin/main` GitHub Actions run 成功，Agent C artifact 复判通过。

## 风险和禁止项

- 禁止把 Board 描述成真实 macOS 权限授予、自动执行器或完整 Gateway readiness。
- 禁止新增审批绕过按钮或自动继续按钮。
- 禁止读取或展示 raw `toolArguments`、路径、URL、命令输出、diff、网页正文或草稿正文。
- 禁止把自然语言交给 Shell、AppleScript 或桌面自动化。
- 禁止修改 Gateway 权限、allowlist 或真实执行策略。
- 禁止混入无关重构或删除旧测试断言。
