# v0.39 / Mac Gateway Action Preflight Matrix

## 版本号

v0.39（MacGatewayActionPreflightMatrix）

## 版本分配依据

v0.38 已完成 Mac Agent Readiness Board，把连接、能力、观察、Loop 和人工闸门汇总为 iPad/mac 就绪看板。下一步需要把“每个将要或已经交给 Gateway 的 action 是否可派发、是否被阻断、是否需要人工、是否已有 result/metadata”压缩成可扫描矩阵，继续提升 mac/iPad 工作台的电脑智能体可解释性。

## 背景

Claw 当前主链路是 iPhone 生成可审批的 `ClawMobileTask` 和 envelope，桌面 Gateway 执行结构化 action 并回传事件和 artifact。手机端已有 Operator Strip、Loop continuation、Mac Agent Readiness Board、Focus Context、Review Detail Dock、Review Trail、Approval Queue、Payload Safety Ledger 和 Artifact Evidence Index。

现有 detail review 多数是按复核域或 artifact kind 汇总，不是严格 per-action payload 证明。因此本轮矩阵必须以 task action 快照和 actionID 关联的 Gateway result 为主，kind 级 detail review 只作为聚焦入口，不得声称每个 action 都已有独立 payload 复核证明。

## 目标

- 新增手机端 presentation-layer 的 Mac Gateway Action Preflight Matrix。
- 从当前 task action、actionID 关联的 Gateway result、审批级别、结构化参数 presence 和既有复核域派生 action 级 row。
- 展示每个 action 的 kind、审批状态、可派发、阻断、metadata 待同步、降级、人工确认和可重试状态。
- 发送前手机审批聚焦 `approval`；发送后 Gateway result 按 action kind 聚焦 Browser/File/Shell/Delivery/AgentTrace 等复核域。
- 在 compact Mission Run 和 iPad/mac 右侧 Mission Review Detail Dock 中展示同一 Matrix，并复用共享 `focusedReviewKind`。
- 补 Swift logic smoke、XCTest、README、协议、flow/flowchart、测试规范和 update log。

## 非目标

- 不新增 schema、event、action kind、artifact kind 或 Gateway JS handler。
- 不改变 envelope、Gateway 权限、allowlist、审批闸门或真实执行能力。
- 不读取或展示 action `toolArguments` 原文、artifact `reference`、Gateway `file://` payload、文件内容、命令输出、diff、网页正文或草稿正文。
- 不把 Matrix 作为自动执行授权、自动审批、自动安全裁决、完整 Gateway readiness 或 per-action payload viewer。
- 不引入第三方依赖，不做无关 UI 重构。

## 当前架构依据

- `Claw/Core/ClawModels.swift`：已有 `ClawMissionRunSummary`、`ClawMissionRunMacAgentReadinessBoard`、`approvalQueueSummary`、`activeReviewFocus(from:)` 和 detail review helpers。
- `Claw/Services/ClawStore.swift`：已有 `missionRunSummary` 汇总 task/session/review summaries，已有 `reviewKind(forActionKind:)` 和 `missionRunApprovalQueue`。
- `Claw/Views/ContentView.swift`：compact Mission Run 和 regular Dock 已共享 `focusedReviewKind`，已有 Board/Trail/Ledger/Queue row 的 focus-only 模式。
- `Tools/LogicSmoke.swift` 和 `ClawTests/ClawTests.swift`：已有 v0.38 展示层断言和敏感字符串不外显扫描。

## 实现步骤

1. 在 `Claw/Core/ClawModels.swift` 新增 `ClawMissionRunActionPreflightItem` 和 `ClawMissionRunActionPreflightMatrix`。
2. 在 `ClawMissionRunSummary` 增加 `actionPreflightItems`，并新增 `macGatewayActionPreflightMatrix(focusedOn:)` 派生 summary。
3. 在 `ClawStore.missionRunSummary` 中从 task actions 和 session results 生成 action preflight rows：
   - 用 actionID 关联 result。
   - 未发送前的 `userConfirmation`/`gatewayApproval` 聚焦 `approval`。
   - 已有 result 后按 action kind 映射到 browser/file/shell/extraction/delivery/accessibility/agent-trace 等复核域。
   - `toolArguments` 只作为 presence 布尔值，不展示原文。
4. 在 `ContentView.swift` 增加 `ClawMissionMacGatewayActionPreflightMatrixView` 和 item row。
5. compact Mission Run 将 Matrix 放在 Mac Agent Readiness Board 后；regular Dock 将 Matrix 放在 Board 后、Trail 前。
6. row button 只能调用本地 focus callback，accessibility hint 明确“不执行 Gateway 动作、不审批、不发送”。
7. 补 LogicSmoke/XCTest：
   - idle 无 row、无 primary focus。
   - 发送前审批聚焦。
   - 发送后 action 顺序、ready/human/metadata count、browser/agentTrace/delivery kind 聚焦。
   - 状态级聚焦、过期聚焦、Shell 高风险聚焦、可重试状态。
   - 可见文本不含 token、Authorization、`toolArguments`、`file://`、路径、stdout/stderr、diff。
8. 同步 README、协议、flow/flowchart、测试规范和 update_log。

## 验收标准

- Matrix 在 idle 状态不虚构 row。
- Matrix 在发送前能显示 action 预检和审批聚焦。
- Matrix 在发送后按 action kind 映射复核域，且可以聚焦 delivery/browser/agentTrace/shell 等已有复核项。
- 状态级聚焦和过期聚焦符合既有 `activeReviewFocus(from:)` 规则。
- UI row 只聚焦，不执行 Gateway 动作、不审批、不发送、不重试、不自动继续。
- 可见文本不泄漏 token、Authorization、`toolArguments`、`file://`、完整路径、stdout/stderr、diff 或 payload 内容。
- 本地轻量检查通过，云端最新 `origin/main` run artifact 复判通过。

## 风险和禁止项

- 不得把 kind 级 metadata summary 伪装成严格 per-action payload 证明。
- 不得展示 action `toolArguments` 原文；只能展示 presence 或固定 title。
- 不得新增 Gateway 权限或绕过 allowlist/审批闸门。
- 不得为了通过测试删除断言。
- 不得混入无关重构。
