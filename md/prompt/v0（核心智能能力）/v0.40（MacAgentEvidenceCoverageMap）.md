# v0.40 / Mac Agent Evidence Coverage Map

## 版本号和分配依据

- 版本号：v0.40。
- 分配依据：v0.39 已完成 Mac Gateway Action Preflight Matrix，下一步应把 action 预检与 artifact / metadata / payload 安全边界串成更直接的证据覆盖视图，继续增强 mac/iPad 工作台的复核智能。

## 背景

Claw 当前已经能在 Mission Run 中展示 Operator Strip、Loop 继续态势、Mac Agent Readiness Board、Action Preflight Matrix、Artifact Evidence Index、Payload Safety Ledger、Review Trail、Approval Queue 和 Review Detail Dock。现有视图分别回答“action 是否可派发”“有哪些 artifact”“payload 是否被读取”，但用户仍需要跨多个组件判断每个复核域是否同时具备 action 来源、artifact 证据、metadata 和 payload 边界。

## 当前轮次目标

新增手机端 presentation-layer 的 Mac Agent Evidence Coverage Map：

- 从既有 `ClawMissionRunSummary` 派生，不进入 envelope，不回传 Gateway。
- 按复核域汇总 action 支撑、artifact 证据、metadata ready、payload 边界和人工复核状态。
- 在 compact Mission Run 和 iPad/mac regular Review Detail Dock 中展示。
- 行点击只改变本地聚焦，不执行 Gateway 动作、不审批、不发送、不重试、不自动继续。
- 不读取或展示 action `toolArguments` 原文、artifact `reference`、Gateway `file://` payload、文件内容、命令输出、diff、网页正文、草稿正文、token 或 header。

## 非目标

- 不新增 Gateway action、artifact kind、schema 字段或 WebSocket 协议。
- 不实现真实 macOS Accessibility bridge、Playwright/browser-use 控制器、后台保活或真实多轮 agent loop。
- 不做自动审批、自动安全裁决、自动执行授权、完整 payload viewer 或 forensic timeline。
- 不改变现有 action allowlist、审批闸门、workspace 限制或 replay guard 行为。

## 当前架构依据

- `Claw/Core/ClawModels.swift`
  - `ClawMissionRunSummary` 是 presentation-layer 聚合入口。
  - `actionPreflightItems` 已提供 action 级状态和 `reviewKind`。

  - `reviewReadinessSummary(focusedOn:)` 和 `reviewPriorityQueue` 已提供人工复核、metadata 缺口和优先级。
- `Claw/Views/ContentView.swift`
  - compact Mission Run 可在 `ClawMissionMacGatewayActionPreflightMatrixView` 后插入 Coverage Map。
  - regular Review Detail Dock 可在 Matrix 后、Trail 前插入 Coverage Map。
- `Tools/LogicSmoke.swift` 和 `ClawTests/ClawTests.swift`
  - mirror v0.38/v0.39 的 idle、pre-send、sent、focus、stale focus、shell risk 和敏感字符串不外显断言。

## 验收标准

- Coverage Map idle 时不可复核且不虚构 row。
- pre-send 时能基于 action preflight 暴露 action 支撑和人工确认状态，但 metadata/evidence 不伪装已覆盖。
- sent 后能按复核域汇总 action 支撑、artifact evidence、metadata、payload boundary 和人工复核状态。
- 聚焦 `delivery-safety`、`shell-safety` 等 detail review 时能标记对应 row；未知聚焦回退 nil。
- Shell 高风险/可重试路径能显示人工复核和 evidence coverage，而不外显命令、stdout/stderr、cwd 或 `toolArguments`。
- UI row 点击只聚焦复核项，不触发 Gateway 动作、审批、发送、重试或自动继续。
- 所有新增可见字符串不包含 raw URL/path/command/stdout/stderr/diff/token/header/草稿正文/`toolArguments`/`file://`。
- 本地非编译静态检查和云端 CI artifact 复判通过；不得用本地编译或本地 smoke 替代云端结果包。

## 风险和禁止项

- 禁止把 Coverage Map 描述为自动安全裁决、自动执行授权或完整证据链。
- 禁止打开或解析 artifact payload、`reference`、`file://`、文件内容、命令输出、diff、网页正文或草稿正文。
- 禁止把 `toolArguments` 原文放入 UI、日志、metadata 或测试可见字符串。
- 禁止改变 Gateway 执行能力、allowlist、审批闸门、workspace 限制、replay guard 或 token 处理。
- 禁止删除现有测试断言来换取通过。
