# v0.42 / Mac Agent Run Timeline

## 版本号和分配依据

- 版本号：v0.42。
- 分配依据：v0.41 已完成 Mac Agent Next Step Deck，下一步应把 action 预检、Gateway 结果、证据覆盖和人工交接串成可扫视的执行时间线，继续增强 mac/iPad 工作台对“电脑智能体正在做什么、卡在哪里、下一步复核什么”的理解。

## 背景

Claw 当前 Mission Run 已有 Operator Strip、Loop 继续态势、Mac Agent Readiness Board、Action Preflight Matrix、Evidence Coverage Map、Artifact Evidence Index、Payload Safety Ledger、Review Trail、Approval Queue 和 Review Detail Dock。它们能回答就绪、预检、证据覆盖、payload 边界和审批状态，但用户仍需要跨多个卡片还原 action 执行顺序和当前卡点。

## 当前轮次目标

新增手机端 presentation-layer 的 Mac Agent Run Timeline：

- 从既有 `ClawMissionRunSummary` 派生，不进入 envelope，不回传 Gateway。
- 按 action 顺序展示计划/审批、Gateway 结果、metadata、证据覆盖、阻断/降级/人工确认状态。
- 在 action 行后追加安全证据同步和人工交接摘要行，帮助 iPad/mac 右侧 Dock 快速判断本轮执行进度。
- compact Mission Run 和 iPad/mac regular Review Detail Dock 复用同一 timeline 与共享聚焦状态。
- row 点击只改变本地聚焦，不执行 Gateway 动作、不审批、不发送、不重试、不自动继续。
- 不读取或展示 action `toolArguments` 原文、artifact `reference`、Gateway `file://` payload、`auditTrail` 原文、文件内容、命令输出、diff、网页正文、草稿正文、token 或 header。

## 非目标

- 不新增 Gateway action、artifact kind、schema 字段或 WebSocket 协议。
- 不实现真实 macOS Accessibility bridge、Playwright/browser-use 控制器、后台保活或真实多轮 agent loop。
- 不做自动审批、自动安全裁决、自动执行授权、完整 payload viewer 或 forensic timeline。
- 不改变现有 action allowlist、审批闸门、workspace 限制或 replay guard 行为。

## 验收标准

- Timeline idle 时不可复核且不虚构 step。
- pre-send 时按 action 顺序展示待审批/可派发状态，能聚焦 `approval`，但不伪装 Gateway 已执行。
- sent 后能展示 action 执行状态、metadata/evidence 状态、安全证据同步 step 和人工交接 step。
- 聚焦 `delivery-safety`、`shell-safety`、`approval` 等 review kind 时能标记对应 step；未知聚焦回退 nil。
- Shell 高风险/可重试路径能显示人工复核、阻断/失败和证据状态，而不外显命令、stdout/stderr、cwd 或 `toolArguments`。
- UI row 点击只聚焦复核项，不触发 Gateway 动作、审批、发送、重试或自动继续。
- 所有新增可见字符串不包含 raw URL/path/command/stdout/stderr/diff/token/header/草稿正文/`toolArguments`/`file://`/`auditTrail` 原文。
- 本地非编译静态检查和云端 CI artifact 复判通过；不得用本地编译或本地 smoke 替代云端结果包。

## 风险和禁止项

- 禁止把 Run Timeline 描述为自动安全裁决、自动执行授权或完整 forensic timeline。
- 禁止打开或解析 artifact payload、`reference`、`file://`、`auditTrail` 原文、文件内容、命令输出、diff、网页正文或草稿正文。
- 禁止把 `toolArguments` 原文放入 UI、日志、metadata 或测试可见字符串。
- 禁止改变 Gateway 执行能力、allowlist、审批闸门、workspace 限制、replay guard 或 token 处理。
- 禁止删除现有测试断言来换取通过。
