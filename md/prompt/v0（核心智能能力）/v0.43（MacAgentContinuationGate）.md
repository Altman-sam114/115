# v0.43 / Mac Agent Continuation Gate 继续闸门

## 版本号

v0.43

提交信息：`v0.43: Mac Agent Continuation Gate 继续闸门`

## 版本分配依据

v0.42 已完成 Mac Agent Run Timeline。下一步需要把 Next Step Deck、Run Timeline、Approval Queue、Review Readiness、Loop 继续态势和 Mac Agent Readiness Board 压缩成一个可扫视的继续闸门，回答“现在能否由用户显式继续下一轮，或必须先复核什么”。

## 背景

Claw 当前方向是 OpenClaw 式电脑智能体：iPhone/iPad 负责计划、风险、审批、envelope 发送和事件复核；真实电脑操作只发生在用户授权的桌面 Gateway。当前展示层已有 Operator Strip、Loop 继续态势、Mac Agent Readiness Board、Action Preflight Matrix、Evidence Coverage Map、Next Step Deck 和 Run Timeline，但还缺少一个面向 Mac/iPad 工作台的 continuation gate，把继续前的阻断、人工确认、metadata 缺口和 Loop 可继续状态放在同一处。

## 目标

- 新增 `ClawMissionRunContinuationGateItem` 和 `ClawMissionRunContinuationGateSummary`。
- 在 `ClawMissionRunSummary` 上新增 `macAgentContinuationGate` 和 `macAgentContinuationGate(focusedOn:)`。
- Gate 从既有安全 summary 派生：Next Step Deck、Run Timeline、Approval Queue、Review Readiness、Loop continuation 和 Mac Agent Readiness Board。
- Gate 固定展示阻断/可重试、人工确认、metadata/证据缺口、Loop 可继续和抽查入口。
- compact Mission Run 在 Run Timeline 后展示 Gate；iPad/mac 右侧 Review Detail Dock 在 Run Timeline 后、Review Trail 前展示 Gate。
- Gate row 只能改变本地复核聚焦，不执行 Gateway 动作、不审批、不发送、不重试、不自动继续。

## 非目标

- 不新增 schema、event kind、action kind、artifact kind 或 envelope 字段。
- 不修改 Gateway JS 执行器、allowlist、workspace、Shell、browser 或桌面 App 权限。
- 不实现自动继续、自动审批、自动重试、自动安全裁决或完整 payload viewer。
- 不读取或展示 action `toolArguments` 原文、artifact `reference`、Gateway `file://` payload、`auditTrail` 原文、文件内容、命令输出、stdout/stderr、diff/patch、网页正文、草稿正文、token、header、cookie 或 secret。

## 当前架构依据

- `Claw/Core/ClawModels.swift` 已集中定义 Mission Run presentation-layer summary。
- `Claw/Views/ContentView.swift` 现有 compact Mission Run 和 iPad/mac Dock 共用同一聚焦状态。
- `ClawTests/ClawTests.swift` 与 `Tools/LogicSmoke.swift` 已覆盖 Next Step Deck 和 Run Timeline 的 idle、发送前、发送后、状态级聚焦、过期聚焦、Shell 高风险和敏感字符串不外显。

## 实现步骤

1. 在 timeline summary 后新增 Gate item/summary model。
2. 在 Run Timeline helper 后新增 Gate derivation helper，优先级为 focused -> blocked -> retryable -> human action -> metadata gap -> loop continuation -> first。
3. Gate title/status/guidance 必须清楚区分待生成、阻断、等待人工、metadata 待同步、可继续和可抽查。
4. 在 compact Mission Run 和 iPad/mac Review Detail Dock 插入 Gate view。
5. 新增 SwiftUI Gate view 和 row view，遵循现有 8px card/row 风格、Dynamic Type、VoiceOver label/hint/input labels。
6. 更新 XCTest 与 Swift logic smoke 的同类断言。
7. 更新 README、协议、flow、flowchart、test、update_log。

## 验收标准

- Gate idle 不虚构 row 或可继续状态。
- 发送前 Gate 聚焦手机审批，标记人工确认。
- 发送后 Gate 汇总阻断/可重试、人工确认、metadata/证据缺口、Loop 可继续和抽查入口。
- detail 聚焦、状态级聚焦和过期聚焦行为与既有展示层一致。
- Shell 高风险/可重试路径可聚焦 Shell safety，且不泄漏 stdout/stderr。
- UI 只提供本地聚焦按钮，不执行任何 Gateway 或审批动作。
- 最新 `origin/main` GitHub Actions artifact 由 Agent C 复判通过。
