# v0.44 / Mac Agent Review Radar 复核雷达

## 版本号

v0.44

提交信息：`v0.44: Mac Agent Review Radar 复核雷达`

## 版本分配依据

v0.43 已完成 Continuation Gate，能判断下一轮是否可继续或被什么阻断。下一步需要一个更适合 iPad/mac 工作台横向扫视的 Review Radar，把安全、证据、执行、交接和 Loop 复核域压缩成少量 sector，帮助用户快速决定先看哪一类风险。

## 背景

Claw 当前是 OpenClaw 式电脑智能体控制台。iPhone/iPad 只负责计划、风险展示、审批、发送 envelope 和事件复核；真实电脑操作只在用户授权的桌面 Gateway 中发生。已有 Operator Strip、Readiness Board、Action Preflight Matrix、Evidence Coverage Map、Next Step Deck、Run Timeline 和 Continuation Gate，但 iPad/mac 右侧 Dock 缺少一个跨复核域的“雷达视图”，用少量 sector 显示高风险/metadata/人工/Loop/证据状态。

## 目标

- 新增 `ClawMissionRunReviewRadarSector` 和 `ClawMissionRunReviewRadarSummary`。
- 在 `ClawMissionRunSummary` 上新增 `macAgentReviewRadar` 和 `macAgentReviewRadar(focusedOn:)`。
- Radar 从既有 presentation-layer summary 派生，不读取 payload：Review Readiness、Evidence Coverage Map、Action Preflight Matrix、Approval Queue、Continuation Gate、Loop continuation、Mac Agent Readiness Board。
- 固定 sector：安全复核、证据覆盖、执行状态、人工交接、Loop 继续。
- compact Mission Run 在 Continuation Gate 后展示 Radar；iPad/mac Review Detail Dock 也在 Continuation Gate 后、Review Trail 前展示 Radar。
- Radar row 只能改变本地复核聚焦，不执行 Gateway 动作、不审批、不发送、不重试、不自动继续。

## 非目标

- 不新增 schema、event kind、action kind、artifact kind、envelope 字段或 Gateway metadata 字段。
- 不修改 Gateway JS、planner、bridge、simulator 或真实执行权限。
- 不实现自动安全裁决、自动继续、自动重试、自动审批、自动修复或完整 payload viewer。
- 不读取或展示 action `toolArguments` 原文、artifact `reference`、Gateway `file://` payload、`auditTrail` 原文、文件内容、命令输出、stdout/stderr、diff/patch、网页正文、草稿正文、token、header、cookie 或 secret。

## 当前架构依据

- `Claw/Core/ClawModels.swift` 已集中承载 Mission Run presentation-layer 派生模型。
- `Claw/Views/ContentView.swift` 现有 compact Mission Run 和 iPad/mac Dock 复用相同聚焦状态。
- `ClawTests/ClawTests.swift` 与 `Tools/LogicSmoke.swift` 已覆盖 idle、发送前、发送后、状态级聚焦、过期聚焦、Shell 高风险和敏感字符串不外显。

## 实现步骤

1. 在 v0.43 Gate summary 后新增 Radar sector/summary model。
2. 在 `ClawMissionRunSummary` extension 中新增 Radar helper，派生固定五个 sector。
3. sector 计数包括 `priorityCount`、`metadataPendingCount`、`humanActionCount`、`blockedCount`、`readyCount`，并带 `reviewKind`/`reviewTitle`/`canFocusReview`/`isFocused`。
4. summary 选择 primary sector 优先级：focused -> blocked -> human action -> metadata gap -> loop continue -> highest priority -> first。
5. 新增 SwiftUI Radar view/row，遵循现有 8px card/row、Dynamic Type、VoiceOver label/hint/input labels。
6. 更新 XCTest 和 Swift logic smoke 覆盖 idle、发送前、发送后、状态级聚焦、过期聚焦、Shell 高风险和敏感字符串不外显。
7. 更新 README、协议、flow、flowchart、test、update_log。

## 验收标准

- Radar idle 不虚构 reviewable 状态或 sector 风险。
- 发送前 Radar 可显示手机审批/执行支撑，但不伪装 Gateway 已执行。
- 发送后 Radar 固定五个 sector，计数和 primary sector 稳定。
- detail 聚焦、状态级聚焦、过期聚焦与既有展示层一致。
- Shell 高风险路径可聚焦 Shell safety，且不泄漏 stdout/stderr。
- UI 只提供本地聚焦按钮，不执行任何 Gateway 或审批动作。
- 最新 `origin/main` GitHub Actions artifact 由 Agent C 复判通过。
