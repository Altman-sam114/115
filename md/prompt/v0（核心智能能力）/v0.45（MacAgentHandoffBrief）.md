# v0.45 / Mac Agent Handoff Brief 人工交接简报

## 版本号

v0.45

提交信息：`v0.45: Mac Agent Handoff Brief 人工交接简报`

## 版本分配依据

v0.44 已完成 Review Radar，把安全、证据、执行、人工交接和 Loop 继续压缩成固定 sector。下一步需要一个更适合 iPad/mac 操作员接手的一屏交接简报，把 Radar、Continuation Gate、Next Step Deck、Run Timeline、Approval Queue 和 Focus Context 的信号转成 3-5 条“现在该人工看什么/为什么/下一步是否可继续”的短摘要。

## 背景

Claw 当前是 OpenClaw 式电脑智能体控制台。iPhone/iPad 只负责计划、风险展示、审批、发送 envelope 和事件复核；真实电脑操作只在用户授权的桌面 Gateway 中发生。已有 Operator Strip、Readiness Board、Action Preflight Matrix、Evidence Coverage Map、Next Step Deck、Run Timeline、Continuation Gate 和 Review Radar，但用户仍需要一个“交接给人工操作员时先看哪里”的固定短简报。

## 目标

- 新增 `ClawMissionRunHandoffBriefItem` 和 `ClawMissionRunHandoffBriefSummary`。
- 在 `ClawMissionRunSummary` 上新增 `macAgentHandoffBrief` 和 `macAgentHandoffBrief(focusedOn:)`。
- Brief 从既有 presentation-layer summary 派生，不读取 payload：Review Radar、Continuation Gate、Next Step Deck、Run Timeline、Approval Queue、Focus Context、Loop continuation。
- 固定 brief item：阻断/可重试、人工确认、metadata/证据缺口、Loop 下一轮、抽查复核。
- compact Mission Run 在 Review Radar 后展示 Brief；iPad/mac Review Detail Dock 也在 Review Radar 后、Review Trail 前展示 Brief。
- Brief row 只能改变本地复核聚焦，不执行 Gateway 动作、不审批、不发送、不重试、不自动继续。

## 非目标

- 不新增 schema、event kind、action kind、artifact kind、envelope 字段或 Gateway metadata 字段。
- 不修改 Gateway JS、planner、bridge、simulator 或真实执行权限。
- 不实现自动安全裁决、自动继续、自动重试、自动审批、自动修复、批量操作或 payload viewer。
- 不读取或展示 action `toolArguments` 原文、artifact `reference`、Gateway `file://` payload、`auditTrail` 原文、文件内容、命令输出、stdout/stderr、diff/patch、网页正文、草稿正文、token、header、cookie 或 secret。

## 当前架构依据

- `Claw/Core/ClawModels.swift` 已集中承载 Mission Run presentation-layer 派生模型。
- `Claw/Views/ContentView.swift` 现有 compact Mission Run 和 iPad/mac Dock 复用相同聚焦状态。
- `ClawTests/ClawTests.swift` 与 `Tools/LogicSmoke.swift` 已覆盖 idle、发送前、发送后、状态级聚焦、过期聚焦、Shell 高风险和敏感字符串不外显。

## 实现步骤

1. 在 v0.44 Radar summary 后新增 Handoff Brief item/summary model。
2. 在 `ClawMissionRunSummary` extension 中新增 Brief helper，派生固定五个 item。
3. item 字段包括 `rank`、`title`、`status`、`guidance`、`icon`、`tone`、`reviewKind`、`reviewTitle`、`canFocusReview`、`isFocused`、`isDone`、`requiresHumanAction`、`isBlocked`、`isRetryable`、`hasMetadataGap`、`canContinueLoop`、`isOptionalReview`。
4. summary 选择 primary item 优先级：focused -> blocked -> human action -> retryable -> metadata gap -> loop continue -> optional review -> first。
5. 新增 SwiftUI Brief view/row，遵循现有 8px card/row、Dynamic Type、VoiceOver label/hint/input labels，窄屏标签可换行。
6. 更新 XCTest 和 Swift logic smoke 覆盖 idle、发送前、发送后、状态级聚焦、过期聚焦、Shell 高风险和敏感字符串不外显。

## 验收标准

- Brief idle 不虚构 reviewable 状态或交接项。
- 发送前 Brief 可显示手机审批/action 支撑，但不伪装 Gateway 已执行。
- 发送后 Brief 固定五个 item，计数和 primary item 稳定。
- primary item 排序稳定：阻断 > 人工 > 可重试 > metadata > Loop 可继续 > 抽查。
- detail 聚焦、状态级聚焦、过期聚焦与既有展示层一致。
- Shell 高风险路径可聚焦 Shell safety，且不泄漏 stdout/stderr/raw-token/header/cookie/secret。
- UI 只提供本地聚焦按钮，不执行任何 Gateway 或审批动作。
- 最新 `origin/main` GitHub Actions artifact 由 Agent C 复判通过。
