# v0.46 / Mac Agent Control Snapshot 控制态势快照

日期：2026-07-07

提交信息：`v0.46: Mac Agent Control Snapshot 控制态势快照`

## 版本分配依据

v0.38-v0.45 已连续补齐 Mac Agent Readiness Board、Action Preflight Matrix、Evidence Coverage Map、Next Step Deck、Run Timeline、Continuation Gate、Review Radar 和 Handoff Brief。下一步需要一个更靠近 mac/iPad 控制台首屏的一眼态势层，把已有复核面板压缩成“当前 Mac 代理能否继续干活、先看哪里、是否只能由用户显式继续”的控制快照。

## 背景

当前 Mission Run 信息完整但分散。操作员需要扫多个面板才能判断是否阻断、是否等待人工、是否缺 metadata、是否可重试或是否可由用户显式开启下一轮。Control Snapshot 应位于 Mission Run 顶部，作为只读 presentation-layer 摘要，不成为新的执行授权。

## 目标

- 新增 `ClawMissionRunControlSnapshotSummary`。
- 在 `ClawMissionRunSummary` 上新增 `controlSnapshot` 和 `controlSnapshot(focusedOn:)`。
- Snapshot 只从既有 presentation summary 派生：Operator Strip、Loop Continuation、Continuation Gate、Review Radar、Handoff Brief、Approval Queue、Live Gateway health 和 Focus Context。
- compact Mission Run 在 Operator Strip 前展示 Snapshot。
- iPad/mac Mission Review Detail Dock 顶部复用 Snapshot。
- Snapshot 按优先级稳定归类：idle、聚焦中、阻断、等待人工、可重试、metadata 待同步、可显式继续、可抽查。
- Snapshot 的按钮只改变本地 `focusedReviewKind`，不执行 Gateway 动作、不审批、不发送、不重试、不自动继续。

## 非目标

- 不新增 schema 字段、Gateway event、action kind、artifact kind 或 envelope 字段。
- 不改 Gateway JS、planner、bridge、simulator 或 live transport。
- 不做真实 macOS Accessibility bridge、Playwright/browser-use 控制器或 AgentLoop Action Contract Preview；这些留给后续版本。
- 不把 Snapshot 当自动执行授权、自动安全裁决、自动继续、自动审批或 payload viewer。

## 当前架构依据

- `Claw/Core/ClawModels.swift` 已有 `macAgentHandoffBrief(focusedOn:)`、`macAgentReviewRadar(focusedOn:)`、`macAgentContinuationGate(focusedOn:)`、`approvalQueueSummary(focusedOn:)`、`loopContinuationSummary(focusedOn:)`、`operatorStrip(focusedOn:)` 和 `focusContextSummary(focusedOn:)`。
- `Claw/Views/ContentView.swift` compact Mission Run 与 iPad/mac Detail Dock 都已共享同一 `focusedReviewKind`。
- XCTest 与 Swift logic smoke 已覆盖同类 presentation-layer summary 的 idle、发送前、发送后、聚焦、过期聚焦、Shell 高风险和敏感字符串不外显。

## 实现步骤

1. 在 `ClawModels.swift` 新增 `ClawMissionRunControlSnapshotSummary`。
2. 在 `ClawMissionRunSummary` extension 中新增 `controlSnapshot` / `controlSnapshot(focusedOn:)`，复用已有 summary 的 counts 和 focus，不读取 raw payload。
3. 在 `ContentView.swift` 新增 `ClawMissionRunControlSnapshotView`，用紧凑 header/chip/button 呈现，不新增长列表。
4. 在 compact Mission Run 的 Operator Strip 前插入 Snapshot；在 iPad/mac Detail Dock 标题区之后插入 Snapshot。
5. 在 `ClawTests/ClawTests.swift` 与 `Tools/LogicSmoke.swift` 补覆盖：idle、pre-send、post-send、聚焦、过期聚焦、Shell 高风险和敏感字符串不外显。
6. 同步 README、协议、flow、flowchart、测试说明和 update_log。

## 验收标准

- idle 不虚构可控制、可继续或 Gateway 已执行状态。
- pre-send 只显示计划/审批待处理，不伪装 Gateway 执行。
- post-send 能稳定区分阻断、等待人工、可重试、metadata 待同步、可显式继续、可抽查。
- 聚焦状态只来自现有 review kind；过期聚焦会回退。
- 可点击入口只改变本地聚焦，不执行 Gateway 动作、不审批、不发送、不重试、不自动继续。
- 可见文本不包含 `toolArguments`、artifact `reference`、Gateway `file://` payload、`auditTrail` 原文、文件内容、命令输出、stdout/stderr、diff/patch、网页正文、草稿正文、token、Authorization/header、Bearer、cookie 或 secret。

## 风险

- Snapshot 容易和 Operator Strip / Handoff Brief 语义重叠；实现必须定位为顶层控制态势，不新增完整复核域或长列表。
- UI 首屏已有多个面板，Snapshot 必须紧凑，避免进一步挤压主要任务信息。
- 任何云端失败都必须在 `main` 追加修复 commit 后重新 push，不做本地编译替代。
