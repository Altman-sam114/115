# v0.47 / Approval Fast Lane 审批快车道

提交信息：`v0.47: Approval Fast Lane 审批快车道`

## 版本分配依据

v0.46 已完成 Mac Agent Control Snapshot，把当前控制态势压缩到 Mission Run 顶部。下一步应提高人工审批效率：用户在手机/iPad 上需要快速知道“现在最该确认哪一项、为什么、确认前看哪几个安全点”。本轮只做 presentation-layer 审批快车道，不新增真实执行权限。

## 背景

Claw 当前方向是 OpenClaw 式电脑智能体控制台。iPhone/iPad 负责计划、风险展示、审批、发送 envelope 和事件复核；真实电脑操作只在用户授权的桌面 Gateway 中发生。已有 Approval Queue，但它偏列表式，操作员仍需要扫多行才能找到最该处理的一项。

## 目标

- 新增 `ClawMissionRunApprovalFastLaneSummary`，从既有 `approvalQueueSummary` 派生首要审批/确认入口。
- 快车道突出：当前 lane 状态、首要 review kind/title、动作标题、审批标题、是否需人工、是否 metadata 缺口、是否可聚焦。
- 在 `ClawMissionApprovalQueueView` 顶部展示快车道，按钮只聚焦对应复核项。
- XCTest 和 Swift logic smoke 覆盖 idle、发送前审批、发送后 high priority、状态级聚焦、过期聚焦、Shell 高风险和敏感字符串不外显。

## 非目标

- 不做批量批准、一键批准、自动审批、自动发送、自动重试或自动继续。
- 不改变 `approveAndContinueAutonomousLoop()`、envelope、schema、Gateway action/event/artifact kind。
- 不读取或展示 `toolArguments` 原文、artifact `reference`、Gateway `file://` payload、`auditTrail` 原文、文件内容、命令输出、stdout/stderr、diff、网页正文、草稿正文、token、header、cookie 或 secret。
- 不实现真实 macOS Accessibility bridge、Playwright/browser-use 控制器、后台保活或真实多轮 agent loop。

## 当前架构依据

- `ClawMissionRunApprovalQueueSummary` 已从复核优先队列、审批和 Gateway handoff 事件派生审批队列。
- `ClawMissionApprovalQueueView` 已展示审批队列并支持本地聚焦。
- `ClawMissionRunSummary.activeReviewFocus(from:)` 可识别 detail review 和状态级 review。
- 本轮可复用既有 summary，不新增 source of truth。

## 实现步骤

1. 在 `Claw/Core/ClawModels.swift` 新增 `ClawMissionRunApprovalFastLaneSummary`。
2. 为 `ClawMissionRunSummary` 新增 `approvalFastLane` 和 `approvalFastLane(focusedOn:)`。
3. 快车道从 `approvalQueueSummary(focusedOn:)` 取 focused item 或首个 item，派生 fixed checklist，不读取 payload。
4. 在 `ClawMissionApprovalQueueView` 顶部插入 `ClawMissionApprovalFastLaneView`。
5. 补 XCTest 和 Swift logic smoke。
6. 同步 README、协议、flow/flowchart、测试说明、update_log。

- 禁止本地编译或用本地 smoke 替代云端验收。
