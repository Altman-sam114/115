# v0.48 / Browser Control Policy Diagnostics 浏览器策略诊断

提交信息：`v0.48: Browser Control Policy Diagnostics 浏览器策略诊断`

## 版本分配依据

v0.47 已完成 Approval Fast Lane，让人工审批入口更快。下一步应回到 macOS Gateway 能力本身：Browser Control 目前已有 metadata-only 计划复核，但操作员仍不能一眼判断策略诊断结果是 dry-run、app allowlist 阻断、host allowlist 阻断、平台不可用、执行失败还是可重试配置问题。本轮补齐可审计的 policy diagnostics，不扩大浏览器控制权限。

## 背景

Claw 当前方向是 OpenClaw 式电脑智能体控制台。iPhone/iPad 只负责输入、计划、风险展示、审批、envelope 发送和事件查看；真实电脑操作只发生在用户授权的桌面 Gateway。`controlBrowser` 已能生成 browser trace 和 `browser-control-*.json` artifact，并在 metadata 中表达是否请求打开浏览器、URL/search presence、allowlist 状态和执行结果。缺口是没有固定的 policy diagnostic 枚举和 retryable reason，导致 UI 只能显示零散 chips。

## 目标

- 在 Gateway `controlBrowser` metadata 中新增安全、固定枚举的策略诊断字段：
  - `policyDiagnostic`
  - `retryableReason`
  - `openAttempted`
  - `hostPolicyChecked`
  - `appPolicyChecked`
- Swift `ClawGatewayBrowserControlReviewSummary` 解析并 allowlist 这些字段。
- Browser Control review row 展示诊断 chips 和 compact status，仍只显示固定枚举和布尔状态。
- XCTest、Swift logic smoke、Gateway direct/WebSocket smoke 覆盖 dry-run、未请求、app 阻断和敏感字符串不外显。
- 同步 README、协议、flow/flowchart、测试说明和 update_log。

## 非目标

- 不新增 Playwright/browser-use 控制器。
- 不新增真实点击、表单填写、页面读取或后台浏览器控制能力。
- 不绕过 `CLAW_ALLOW_BROWSER_CONTROL`、`CLAW_BROWSER_APP_ALLOWLIST` 或 `CLAW_BROWSER_HOST_ALLOWLIST`。
- 不把 raw URL、search query、HTML/page text、form fields、candidate labels、stdout/stderr、browser app name、host、allowlist 值、token、header、cookie、secret、`toolArguments` 或 `file://` payload 写进 metadata 或 UI。
- 不改变 envelope schema、action kind、artifact kind 或审批闸门。

## 当前架构依据

- `Tools/claw-gateway-server.mjs` 的 `browserControlReviewMetadata()` 已集中生成 Browser Control metadata。
- `browserControlArtifact()` 已覆盖 not-requested、dry-run、unavailable、app policy blocked、host blocked、opened 和 failed 路径。
- `ClawGatewayBrowserControlReviewSummary` 已从 artifact metadata 派生 metadata-only 复核摘要，并对 mode/policy/result/safety flags 做 allowlist。
- `ClawGatewayBrowserControlReviewRow` 已展示 policy、control、allowlist 和 safety chips。
- Gateway direct/WebSocket smoke 已有 `assertBrowserControlReviewMetadata()` helper，可扩展固定 allowed keys 和断言。

## 实现步骤

1. 扩展 `browserControlReviewMetadata()`，新增 `policyDiagnostic`、`retryableReason`、`openAttempted`、`hostPolicyChecked`、`appPolicyChecked`，全部从当前路径和布尔状态派生。
2. 在 `browserControlArtifact()` 各返回路径传入固定诊断：
   - not requested: `not-requested` / `none`
   - dry-run: `dry-run` / `enable-browser-control`
   - non-macOS unavailable: `platform-unavailable` / `requires-macos`
   - app blocked: `app-blocked` / `allow-browser-app`
   - host blocked: `host-blocked` / `allow-browser-host`
   - opened: `opened` / `none`
   - failed: `automation-failed` / `automation-failed`
3. `ClawGatewayBrowserControlReviewSummary` 新增对应字段和 allowlist，`compactStatus` 加入 `diagnostic ...`。
4. `ClawGatewayBrowserControlReviewRow` 增加诊断 chips；失败/阻断用橙色，成功用绿色，dry-run/未请求用次级或蓝色。
5. 更新 Gateway direct/WebSocket smoke 的 metadata allowed keys、expected 字段和泄漏断言。
6. 更新 XCTest 和 Swift logic smoke，覆盖新字段解析、fallback、敏感 metadata 拒绝。
7. 更新 README、`Docs/claw-mobile-gateway-protocol.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、`update_log.md`。

## 验收标准

- `controlBrowser` 的 browser trace 和 browser-control artifact metadata 都带固定策略诊断字段。
- 手机/iPad 端只展示 allowlist 后的 fixed enum 和布尔状态，不显示 URL/search/app/host/allowlist/raw output。
- Gateway smoke、XCTest 和 Swift logic smoke 在云端覆盖新字段和敏感字符串不外显。
- 最新 `origin/main` run 的 CI artifact 经 Agent C 复判通过。

## 风险和禁止项

- 风险：diagnostic 字段如果从 raw error/stdout/stderr 派生会泄漏内容；必须只用固定枚举。
- 风险：`retryableReason` 不能被 UI 表达成自动修复或自动放权，只能作为人工配置提示。
- 禁止删除旧 smoke/XCTest 断言来制造通过。
- 禁止本地编译或用本地 smoke 替代云端验收。
