# v0.49 / Mac Accessibility Signal Quality 观察信号质量

提交信息：`v0.49: Mac Accessibility Signal Quality 观察信号质量`

## 版本分配依据

v0.48 已补齐 Browser Control policy diagnostics。下一步应增强 macOS Gateway 的观察面：当前 Accessibility review 只显示 mode、policy、节点数和候选控件数，人工仍需要自己判断这是 dry-run、窗口 metadata、真实只读 Accessibility 摘要、权限失败还是平台不可用。本轮补齐 metadata-only 的信号质量，不扩大 Accessibility 权限。

## 背景

Claw 是 OpenClaw 式电脑接管智能体控制台。iPhone/iPad 负责计划、风险展示、审批和复核；真实电脑观察与控制只发生在用户授权的桌面 Gateway。`observeScreen` 已能生成 `accessibilityTree` artifact，并在授权 macOS Gateway 上采集只读前台 App/窗口和有限候选控件摘要；默认 dry-run 或 window metadata。需要把这种“证据质量”作为固定 metadata 交给手机端复核。

## 目标

- Gateway `accessibilityTree` artifact metadata 增加固定信号质量字段：
  - `signalQuality`
  - `evidenceTier`
  - `controlCoverage`
  - `valuesOmitted`
  - `passwordFieldsOmitted`
  - `rawTextOmitted`
  - `actionExecutionSupported`
- Swift `ClawGatewayAccessibilityReviewSummary` 解析并 allowlist 新字段，`requiresSignalReview` 用 metadata 缺失、降级/失败质量和非观察-only 安全信号派生。
- Accessibility review row 展示 signal/evidence/coverage/omission chips；Mission Run 复核优先队列用信号质量文案提示。
- XCTest、Swift logic smoke 和 Gateway direct/WebSocket smoke 覆盖新字段、固定键和敏感字符串不外显。
- 同步 README、协议、flow/flowchart、测试说明和 update_log。

## 非目标

- 不实现完整 Accessibility bridge、点击、输入、选择器、表单提交、视觉定位或自动操作。
- 不新增 action kind、artifact kind、event kind 或 envelope schema 字段。
- 不绕过 `CLAW_ALLOW_ACCESSIBILITY_OBSERVE`、window metadata、screen capture、desktop app allowlist 或用户审批。
- 不读取或展示 Gateway `file://` payload、raw app/window title、raw control label、raw UI text、password/field value、screenshot content、token/header、cookie、secret、workspace path 或 `toolArguments`。

## 当前架构依据

- `collectAccessibilityTreeSummary()` 已覆盖 not-requested、dry-run/window-metadata、platform unavailable、accessibility failed 和 accessibility-summary 分支。
- `accessibilityTreeMetadata()` 集中生成 Accessibility metadata。
- `ClawGatewayAccessibilityReviewSummary` 已从 `accessibilityTree` artifact metadata 派生手机端 review，不读取 payload。
- `ClawGatewayAccessibilityReviewRow` 已展示 mode/policy/count/platform/redaction/safety chips。
- direct smoke 已有 `assertAccessibilityTreeArtifact()`；Swift XCTest 和 LogicSmoke 已有 Accessibility review 断言。

## 实现步骤

1. Gateway 增加 `accessibilitySignalMetadata(tree)` 或等价 helper，从 `tree.mode` 和计数派生固定字段：
   - `signalQuality`: `not-requested` / `dry-run` / `window-metadata` / `accessibility-summary` / `permission-missing` / `platform-unavailable`
   - `evidenceTier`: `missing` / `degraded` / `satisfied`
   - `controlCoverage`: `none` / `window-only` / `candidate-controls`
   - omission/action booleans 全部来自 `tree.safety` 固定状态。
2. `accessibilityTreeMetadata()` 附加新字段，不加入 raw title/app/label/error/note。
3. Swift summary 新增字段、allowlist 和 `requiresSignalReview`。
4. SwiftUI row 加 signal quality chips，继续用 `Label`、`PhoneAgentTag` 和 Dynamic Type，不新增 icon-only button。
5. Simulator `accessibilityTreeMetadata()` 补 dry-run/degraded/candidate-controls fixture。
6. XCTest/LogicSmoke 补 happy path、fallback/redaction path 和 unsafe metadata 拒绝。
7. Gateway direct/WebSocket smoke 断言新字段、metadata key 和不泄漏敏感内容。
8. 更新 README、协议、flow/flowchart、md/test/test.md 和 update_log。

## 验收标准

- Accessibility artifact metadata 带固定 signal quality 字段。
- 手机/iPad 端只展示 allowlist 后的固定枚举和布尔状态。
- Gateway direct/WebSocket smoke、XCTest、Swift logic smoke 和云端 CI artifact 覆盖新字段。
- 最新 `origin/main` run 的 CI artifact 经 Agent C 复判通过。

## 风险和禁止项

- 风险：如果把 osascript error、App 名、窗口标题或控件 label 放进 metadata，会泄漏桌面内容；必须只用固定枚举/布尔/计数。
- 风险：`evidenceTier=satisfied` 只能表示只读摘要可用于复核，不代表 UI 可自动操作。
- 禁止删除断言制造通过。
- 禁止本地编译或用本地 smoke 替代云端验收。
