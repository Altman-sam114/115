# v0.50 / Desktop App Policy Diagnostics 桌面 App 策略诊断

提交信息：`v0.50: Desktop App Policy Diagnostics 桌面策略诊断`

## 版本分配依据

v0.48 已补齐 Browser Control policy diagnostics，v0.49 已补齐 Accessibility signal quality。下一步应增强 macOS 桌面 App 操作的可解释性：当前 Delivery Safety 只展示最终提交闸门、正文/paste 省略和按键计数，人工还不能一眼区分 dry-run、平台不可用、App allowlist 阻断、按键策略阻断、已尝试自动化或自动化失败。本轮补齐 metadata-only 的桌面策略诊断，不扩大桌面控制权限。

## 背景

Claw 是 OpenClaw 式电脑接管智能体控制台。iPhone/iPad 负责计划、风险展示、审批和复核；真实桌面操作只发生在用户授权的桌面 Gateway。`operateDesktopApp` 已有 allowlist、最终提交闸门和 Delivery Safety metadata。需要把桌面 App 策略判断压缩成固定枚举，让手机端能安全判断下一步是授权、改目标、补 allowlist、等待人工还是查看失败。

## 目标

- Gateway `operateDesktopApp` 相关 artifact metadata 增加固定桌面策略诊断字段：
  - `desktopPolicyDiagnostic`
  - `desktopRetryableReason`
  - `automationAttempted`
  - `appPolicyChecked`
  - `keyPolicyChecked`
- Swift `ClawGatewayDeliverySafetyReviewSummary` 解析并 allowlist 新字段，`requiresDesktopPolicyReview` 用 metadata 缺失、阻断、失败、自动化尝试和最终提交闸门派生。
- Delivery Safety review row 展示 policy diagnostic、retry reason、automation attempted、app/key policy checked chips；Mission Run 复核优先队列用桌面策略诊断文案提示。
- Gateway direct/WebSocket smoke、XCTest 和 Swift logic smoke 覆盖新字段、固定键、敏感字符串不外显和 legacy fallback。
- 同步 README、协议、flow/flowchart、测试说明和 update_log。

## 非目标

- 不新增 action kind、artifact kind、event kind 或 envelope schema 字段。
- 不实现新的桌面 App 点击、输入、视觉定位、控件级 Accessibility 动作或自动提交。
- 不绕过 `CLAW_ALLOW_DESKTOP_CONTROL`、`CLAW_DESKTOP_APP_ALLOWLIST`、`CLAW_DESKTOP_KEY_ALLOWLIST`、final-submit gate 或用户审批。
- 不读取或展示 Gateway `file://` payload、target app 原文、app allowlist 值、key sequence、paste text、草稿正文、stdout/stderr、osascript error 原文、token/header、cookie、secret、workspace path 或 `toolArguments`。

## 当前架构依据

- `Tools/claw-gateway-server.mjs` 的 `desktopAppAction()` 已覆盖 dry-run、非 macOS、缺少 target、App allowlist 阻断、自动化执行成功/暂停/失败。
- `deliverySafetyMetadata()` 集中生成 `messageDraft` 和 `operateDesktopApp` 的 metadata。
- `ClawGatewayDeliverySafetyReviewSummary` 已从 delivery metadata 派生手机端 review，不读取 payload。
- `ClawGatewayDeliverySafetyReviewRow` 已展示 mode/action/target、omission 和 key count chips。
- direct/WebSocket smoke 已有 `assertDeliverySafetyMetadata()`；Swift XCTest 和 LogicSmoke 已有 Delivery Safety 断言。

## 实现步骤

1. Gateway 增加 `desktopPolicyDiagnostic(action, details)` 或等价 helper，从 `details.mode`/执行结果和 policy 分支派生固定字段：
   - `not-requested`：非 `operateDesktopApp` 的 message draft。
   - `dry-run`：`CLAW_ALLOW_DESKTOP_CONTROL` 未启用。
   - `platform-unavailable`：非 macOS。
   - `missing-target`：缺少 target app。
   - `app-blocked`：App allowlist 阻断。
   - `key-blocked`：key allowlist 或 final submit gate 阻断。
   - `automation-attempted`：已尝试 osascript/System Events。
   - `automation-failed`：自动化尝试失败。
2. 增加 `desktopRetryableReason` 固定枚举：`none`、`enable-desktop-control`、`requires-macos`、`provide-target-app`、`allow-desktop-app`、`allow-desktop-key`、`automation-failed`、`user-final-submit`。
3. 在 `deliverySafetyMetadata()` 只为 `operateDesktopApp` metadata 输出这些字段；非桌面草稿输出 `not-requested/none/false/false/false`，保持固定键。
4. Swift summary 新增字段、allowlist 和 `requiresDesktopPolicyReview`。
5. SwiftUI row 加 desktop policy chips，继续使用 `Label`/`PhoneAgentTag`、Dynamic Type 和非纯颜色区分。
6. Simulator Delivery Safety metadata 补 dry-run 桌面策略 fixture。
7. XCTest/LogicSmoke 补 happy path、desktop result path、sensitive metadata 拒绝和 compact status。
8. Gateway direct/WebSocket smoke `assertDeliverySafetyMetadata()` 断言新字段、固定键和敏感字符串不外显；desktop policy blocked path 也必须断言 metadata。
9. 更新 README、`Docs/claw-mobile-gateway-protocol.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和 `update_log.md`。

## 验收标准

- Desktop App delivery metadata 带固定 policy diagnostic 字段。
- 手机/iPad 端只展示 allowlist 后的固定枚举和布尔状态。
- Gateway direct/WebSocket smoke、XCTest、Swift logic smoke 和云端 CI artifact 覆盖新字段。
- 最新 `origin/main` run 的 CI artifact 经 Agent C 复判通过。

## 风险和禁止项

- 风险：如果把 target app、allowlist、key sequence、paste text、stdout/stderr 或 osascript error 放进 metadata，会泄漏桌面内容；必须只用固定枚举/布尔/计数。
- 风险：`automationAttempted=true` 只能表示 Gateway 尝试过受控自动化，不代表最终提交已发生。
- 禁止删除断言制造通过。
- 禁止本地编译或用本地 smoke 替代云端验收。
