# v0.51 / Shell Command Policy Diagnostics Shell 命令策略诊断

提交信息：`v0.51: Shell Command Policy Diagnostics Shell 策略诊断`

## 版本分配依据

v0.48 已补齐 Browser Control policy diagnostics，v0.50 已补齐 Desktop App policy diagnostics。Shell 侧目前只有 mode/policy/allowlist/execution presence，人工还不能一眼区分 missing structured command、parse failed、dry-run、allowlist blocked、execution attempted 或 execution failed。本轮补齐 metadata-only 的 Shell 策略诊断，不扩大 Shell 权限。

## 背景

Claw 是 OpenClaw 式电脑接管智能体控制台。iPhone/iPad 负责计划、风险展示、审批和复核；真实 Shell 执行只发生在用户授权的桌面 Gateway。`runShellCommand` 已有 structured command、allowlist、dry-run 和 Shell Command Safety metadata。需要把 Shell 策略判断压缩成固定枚举，让手机端能安全判断下一步是补结构化命令、启用 Shell、配置 allowlist、允许 binary，还是复核执行失败。

## 目标

- Gateway `runShellCommand` 相关 `commandOutput` artifact metadata 增加固定字段：
  - `shellPolicyDiagnostic`
  - `shellRetryableReason`
  - `policyChecked`
  - `binaryAllowlistChecked`
  - `structuredCommandChecked`
- Swift `ClawGatewayShellCommandSafetyReviewSummary` 解析并 allowlist 新字段，`requiresShellPolicyReview` 用 metadata 缺失、阻断、失败、执行尝试派生。
- Shell Safety review row 展示 policy diagnostic、retry reason、policy/binary/structured checked chips；Mission Run 复核优先队列用 Shell 策略诊断文案提示。
- Gateway direct/WebSocket smoke、XCTest 和 Swift logic smoke 覆盖新字段、固定键、敏感字符串不外显和 legacy fallback。
- 同步 README、协议、flow/flowchart、测试说明和 update_log。

## 非目标

- 不新增 action kind、artifact kind、event kind 或 envelope schema 字段。
- 不实现自然语言直连 Shell，不绕过 `CLAW_ALLOW_SHELL` / `CLAW_SHELL_ALLOWLIST`。
- 不读取或展示 Gateway `file://` payload、raw command、binary/args、cwd、stdout/stderr、token/header、cookie、secret、自然语言 instruction 或 `toolArguments`。
- 不做完整 artifact payload viewer、真实 multi-turn agent loop、Accessibility bridge 或 Playwright 控制器。

## 当前架构依据

- `Tools/claw-gateway-server.mjs` 的 `runShellAction()` 已覆盖 missing structured command、parse failed、policy blocked、allowlist execution。
- `shellCommandSafetyMetadata()` 集中生成 Shell safety metadata。
- `ClawGatewayShellCommandSafetyReviewSummary` 已从 shell metadata 派生手机端 review。
- `ClawGatewayShellCommandSafetyReviewRow` 已展示 mode/policy/execution/output/omission chips。
- direct/WebSocket smoke 已有 `assertShellCommandSafetyMetadata()`；Swift XCTest 和 LogicSmoke 已有 Shell Safety 断言。

## 固定枚举

### shellPolicyDiagnostic

- `not-requested`
- `missing-structured-command`
- `command-parse-failed`
- `dry-run`
- `allowlist-blocked`
- `execution-attempted`
- `execution-failed`

### shellRetryableReason

- `none`
- `provide-structured-command`
- `fix-command-parse`
- `enable-shell`
- `configure-shell-allowlist`
- `allow-shell-binary`
- `automation-failed`
- `review-execution`

### 派生规则（Gateway）

- missing structured command -> diagnostic `missing-structured-command`, retry `provide-structured-command`, structuredCommandChecked=true, binaryAllowlistChecked=false, policyChecked=true
- parse failed -> diagnostic `command-parse-failed`, retry `fix-command-parse`, structuredCommandChecked=true, binaryAllowlistChecked=false, policyChecked=true
- policy blocked + dry-run / !allowShell -> diagnostic `dry-run`, retry `enable-shell`, binaryAllowlistChecked=true, structuredCommandChecked=true, policyChecked=true
- policy blocked + allowShell + allowlist miss -> diagnostic `allowlist-blocked`, retry `configure-shell-allowlist`（allowlist empty）或 `allow-shell-binary`（allowlist configured）, binaryAllowlistChecked=true
- executed + exit zero + not timed out -> diagnostic `execution-attempted`, retry `none`
- executed + failed/timeout -> diagnostic `execution-failed`, retry `automation-failed` 或 `review-execution`

## 实现步骤

1. Gateway `shellCommandSafetyMetadata()` 调用 `shellPolicyDiagnostics(details)`，写入固定 5 键；`runShellAction` 各分支传入足够 details。
2. direct/WebSocket smoke 的 allowedKeys 与 expected 覆盖新字段；缺命令、policy blocked、allowlist 真执行路径都断言诊断。
3. Swift 模型增加字段、enum allowlist、`requiresShellPolicyReview`、compactStatus。
4. 全局 metadata key allowlist 增加新键。
5. Simulator `shellCommandSafetyMetadata()` 补齐固定诊断（dry-run / allowlist-blocked 场景）。
6. Mission Run priority queue 用 `requiresShellPolicyReview`，action hint 改为“确认 Shell 策略、allowlist 和执行状态”。
7. Shell Safety UI row 增加 policy diagnostic chips。
8. XCTest / LogicSmoke 覆盖固定键、requires review、敏感字符串拒绝。
9. 更新 README、Docs protocol、flow/flowchart、test、update_log。

## 验收标准

- metadata 固定包含 5 个新键，enum 外值被 Swift 丢弃为 nil。
- UI 不展示 raw command/stdout/stderr/cwd/token/toolArguments。
- dry-run、missing structured command、allowlist 真执行路径在 smoke 中断言诊断。
- requiresShellPolicyReview 在 metadata 缺失/阻断/失败时为 true。
- 文档同步，commit 格式 `v0.51: ...`。

## 风险和禁止项

- 禁止把诊断当成执行授权。
- 禁止自然语言 Shell。
- 禁止扩大 allowlist 默认权限。
- 禁止无关重构或回滚用户改动。
- 禁止本地编译替代云端验证。
