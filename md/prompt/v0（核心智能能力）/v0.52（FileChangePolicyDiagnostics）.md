# v0.52 / File Change Policy Diagnostics 文件变更策略诊断

提交信息：`v0.52: File Change Policy Diagnostics 文件变更策略诊断`

## 版本分配依据

v0.50/v0.51 已补齐 Desktop/Shell policy diagnostics。文件变更侧目前只有 mode/workspace/pathEscape/write presence，人工还不能一眼区分 path escape、write failed 与 write succeeded。本轮补齐 metadata-only 的文件变更策略诊断，不扩大 workspace 写权限。

## 目标

- Gateway `manageFiles` artifact metadata 增加固定：
  - `filePolicyDiagnostic`
  - `fileRetryableReason`
  - `policyChecked`
  - `workspacePolicyChecked`
  - `pathPolicyChecked`
- Swift `ClawGatewayFileChangeSafetyReviewSummary` 解析 allowlist，并派生 `requiresFilePolicyReview`。
- File Change row / 优先队列展示诊断 chips。
- direct/WebSocket smoke、XCTest、LogicSmoke 覆盖固定键与敏感值拒绝。
- 同步 README、协议、flow/flowchart、test、update_log。

## 非目标

- 不新增 schema/event/action/artifact kind。
- 不读取 raw path/content/diff/`toolArguments`。
- 不绕过 session workspace 限制。

## 固定枚举

### filePolicyDiagnostic
- `not-requested`
- `path-escape-blocked`
- `workspace-write-failed`
- `write-attempted`
- `write-succeeded`

### fileRetryableReason
- `none`
- `provide-workspace-path`
- `fix-workspace-scope`
- `retry-write`
- `review-write-failure`

## 验收标准

- metadata 固定包含 5 新键；enum 外值 Swift 丢弃为 nil。
- path escape / write failed / write succeeded 路径 smoke 断言诊断。
- 文档同步；commit `v0.52: ...`；云端 CI artifact 验收。
