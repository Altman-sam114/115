# v0.53 / Extraction Source Completeness Diagnostics 提取来源完整性诊断

提交信息：`v0.53: Extraction Source Completeness Diagnostics 提取来源完整性诊断`

## 版本分配依据

v0.50–v0.52 已补齐 Desktop / Shell / File Change policy diagnostics。提取完整性目前只有 mode、rowCount、completenessStatus 与 source counts，人工还不能一眼区分 dry-run、empty、partial 与 complete。本轮补齐 metadata-only 的提取来源完整性诊断，不读取 row 内容。

## 目标

- Gateway `extractData` 输出 artifact metadata 增加固定：
  - `extractionPolicyDiagnostic`
  - `extractionRetryableReason`
  - `policyChecked`
  - `sourceCoverageChecked`
  - `completenessChecked`
- Swift `ClawGatewayExtractionCompletenessReviewSummary` 解析 allowlist，并派生 `requiresExtractionPolicyReview`。
- Extraction review row / 优先队列展示诊断 chips。
- Gateway smoke、XCTest、LogicSmoke 覆盖固定键与敏感值拒绝。
- 同步 README、协议、flow/flowchart、test、update_log。

## 非目标

- 不新增 schema/event/action/artifact kind。
- 不读取 row content、URL/path、`toolArguments` 或 file:// payload。
- 不改变提取算法或自动补全来源。

## 固定枚举

### extractionPolicyDiagnostic
- `not-requested`
- `dry-run`
- `empty`
- `partial`
- `complete`

### extractionRetryableReason
- `none`
- `provide-source-artifacts`
- `review-empty-extraction`
- `review-partial-extraction`

## 派生规则

- mode `dry-run-extraction` -> diagnostic `dry-run`, retry `provide-source-artifacts`
- completeness `empty` -> diagnostic `empty`, retry `review-empty-extraction`
- completeness `partial` -> diagnostic `partial`, retry `review-partial-extraction`
- completeness `complete` -> diagnostic `complete`, retry `none`
- sourceCoverageChecked/completenessChecked/policyChecked 固定 true（有 extraction review 时）

## 验收标准

- metadata 固定包含 5 新键；enum 外值 Swift 丢弃为 nil。
- `"none"` retry reason 不能被 cleanValue 丢弃。
- smoke / XCTest / LogicSmoke 覆盖；文档同步；云端 CI artifact 验收。
