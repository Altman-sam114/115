# v0.54 / Mission Run Policy Diagnostics Board 策略诊断看板

提交信息：`v0.54: Mission Run Policy Diagnostics Board 策略诊断看板`

## 版本分配依据

v0.48–v0.53 已分别补齐 Browser / Desktop / Shell / File / Extraction 的 metadata-only 策略诊断。用户仍需跨多个 detail row 才能判断“现在哪一类策略在阻断、可重试或只缺 metadata”。本轮做 presentation-layer 看板，只汇总已有诊断，不扩 Gateway 权限。

## 目标

- 新增 `ClawMissionRunPolicyDiagnosticsBoard` / Item，固定 5 行：
  - delivery-safety（Desktop App）
  - shell-safety
  - file-change-safety
  - extraction-completeness
  - browser-control
- 从既有 review 的 diagnostic / retry / requires* 派生 status、tone、是否阻断、是否 metadata 缺口、是否可聚焦。
- compact Mission Run 与 iPad Dock 在 Readiness Board 后展示。
- XCTest / LogicSmoke 覆盖 idle、有诊断、聚焦和敏感字符串不外显。
- 同步 README、flow、test、update_log。

## 非目标

- 不新增 schema/event/action/artifact kind。
- 不改变 Gateway JS 执行权限。
- 不读取 file:// payload、toolArguments、raw command/path/URL/row content。
- 不自动审批、发送、重试或继续 Loop。

## 验收标准

- 看板只从已有 review 派生。
- row 按钮只改变本地聚焦。
- idle 不虚构阻断。
- 云端 CI artifact 验收通过。
