# v0.56 / Control Snapshot Live Health Signal 控制态势连接健康信号

提交信息：`v0.56: Control Snapshot Live Health Signal 控制态势连接健康信号`

## 版本分配依据

v0.55 已把 Live Gateway health 做成 Mission Run 健康条，但 Control Snapshot 顶部态势仍不显示 live/fallback/ping 状态。本轮把既有 Health Strip 信号并入 Control Snapshot presentation-layer，不改 transport。

## 目标

- `ClawMissionRunControlSnapshotSummary` 增加 live health 只读信号字段。
- `controlSnapshot(focusedOn:liveHealth:)` 可选并入 `ClawMissionRunLiveGatewayHealthStrip`。
- Control Snapshot UI 增加 live/fallback/error/ping chips。
- Mission Run / Dock 调用时传入 store 的 health strip。
- XCTest / LogicSmoke 覆盖 fallback 信号与脱敏。
- 同步 README、flow、test、update_log。

## 非目标

- 不新增 schema/event/action/artifact kind。
- 不改 WebSocket 重连/心跳协议。
- 不自动连接、审批或继续 Loop。

## 验收标准

- 无 live 证据时字段为安全默认值，不虚构阻断。
- 有 fallback/error 时 Control Snapshot 展示对应 chip/status 提示。
- 云端 CI artifact 通过。
