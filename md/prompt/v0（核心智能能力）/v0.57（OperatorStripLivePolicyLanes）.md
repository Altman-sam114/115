# v0.57 / Operator Strip Live/Policy Lanes 操作条连接与策略格

提交信息：`v0.57: Operator Strip Live/Policy Lanes 操作条连接与策略格`

## 版本分配依据

v0.55/v0.56 已把 Live Gateway health 和策略诊断做成独立面板，但 Operator Strip 仍只有 Gateway/证据/复核/下一步 4 格。本轮在 Operator Strip 增加 Live 与 Policy 两格，复用既有 Health Strip 与 Policy Diagnostics Board，不扩 Gateway 权限。

## 目标

- `operatorStrip(focusedOn:liveHealth:policyBoard:)` 派生 6 格：gateway、live、policy、evidence、review、next。
- Live 格从 Health Strip 取状态；Policy 格从 Policy Diagnostics Board 取阻断/人工/metadata 摘要。
- Mission Run 调用传入 store health strip 与 summary policy board。
- XCTest / LogicSmoke 更新 lane id 期望。
- 同步 README、flow、test、update_log。

## 非目标

- 不新增 schema/event/action/artifact kind。
- 不执行连接、审批、重试或自动继续。
- 不读取 payload 或敏感原文。

## 验收标准

- idle 仍稳定 6 格，不虚构 live 阻断。
- fallback/error 时 live 格 tone/status 正确。
- policy 格可聚焦 board 主复核 kind。
- 云端 CI artifact 通过。
