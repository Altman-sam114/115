# v0.6 / 增强 AgentTrace 证据策略

## 版本分配依据

当前最新记录为 v0.5。本轮只增强 macOS/Desktop Gateway prototype 的 `runAgentLoop` artifact 内容和 smoke 断言，不新增 schema、action kind 或真实执行权限，因此分配为 v0.6。

## 背景

Claw 当前已有 `runAgentLoop`/`agentTrace`，能基于 browser/file/shell/screen artifact 生成 observe-plan-act-verify 轨迹。但 trace 仍偏模板化，缺少明确的证据充分性、缺口、风险标签、停止原因和下一步选择依据。用户总目标是持续优化 macOS Gateway 和 iPad/iPhone 控制台，让 Claw 更像 OpenClaw 式电脑智能体。本轮先做 Gateway 端可审计智能化小步。

## 目标

- 增强 `Tools/claw-gateway-server.mjs` 的 `agentLoopAction` trace JSON。
- 在不改 envelope schema 的前提下，为 `agentTrace` 增加 artifact 内部字段：
  - `readiness`: 证据分数、已满足信号、缺失信号、是否可继续。
  - `decisionChecklist`: 每类输入证据的通过/缺失状态和原因。
  - `selectedNextAction`: 从 `nextActions` 中选出的当前推荐动作。
  - `riskTags`: 由候选动作、安全闸门和缺失证据派生的风险标签。
  - `stopReason`: destructive/external/final-submit/insufficient-evidence 等停止原因。
  - `handoffSummary`: 给手机端或人工复核的简短摘要。
- 更新 direct smoke 和 WebSocket smoke，断言上述结构存在且与已有 artifact 上下文一致。
- 同步 README、协议、flow、flowchart、update_log 中受影响描述。

## 非目标

- 不新增 `ClawMobileActionKind`、`ClawGatewayArtifactKind` 或 `ClawGatewayEventKind`。
- 不改 Swift schema、Mission Run UI、iPad 详细 trace 预览。
- 不启用真实 Accessibility 点击、Playwright/browser-use、真实网络抓取或真实桌面提交。
- 不把自然语言直接交给 Shell、AppleScript 或桌面自动化。
- 不引入第三方依赖。

## 当前架构依据

- `ClawMobileBridge` 已为 `runAgentLoop` 生成结构化 `toolArguments`：`inputSources`、`allowedNextActions`、`approvalRequiredFor`、`stopBeforeDestructiveAction`。
- Gateway `agentLoopAction` 已写出 `agentTrace` artifact，并已有 `sourceArtifacts`、`evidenceRows`、`observations`、`nextActions`、`safetyGates`、`iterations`。
- smoke 已断言 `agentTrace` 消费 browser/file/shell artifact 并提出 `composeMessage`。
- `agentTrace` 是 artifact 内容，扩展 JSON 字段不需要改变 `claw.computer.control.v1` envelope。

## 实现步骤

1. 先读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和本提示词。
2. 同步最新 `origin/main`，确认当前分支是 `main`。如果工作区仍有无关未提交改动，暂停并说明，不要回滚或混入本轮提交。
3. 在 `Tools/claw-gateway-server.mjs` 中新增小型 helper，基于 `summarizeArtifactContext`、`inputSources`、`proposedActions`、`approvalRequiredFor` 生成 readiness/checklist/risk/stop/handoff。
4. 保留现有 `agentTrace` 字段，新增字段只做向后兼容扩展。
5. 更新 `Tools/claw-gateway-direct-smoke.mjs` 和 `Tools/claw-gateway-smoke.mjs`：
   - 断言 `readiness.score` 是数字。
   - 断言 browser/file/command 信号被标记为 satisfied。
   - 断言 `selectedNextAction.kind` 来自 `nextActions`。
   - 断言 `riskTags` 或 `stopReason` 覆盖需要审批/停止的下一步。
6. 同步文档：`README.md`、`Docs/claw-mobile-gateway-protocol.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md`。
7. 只提交本轮相关文件，commit message 用 `v0.6: 增强 AgentTrace 证据策略`，push 到 `origin/main`。

## CI / main push / artifact 要求

- Agent B 必须在 `main` 上提交并 `git push origin main`。
- GitHub Actions 应生成未加密 `ci-results` artifact。
- Agent C 下载到 `/private/tmp/claw-c-review-<run_id>/`。
- Agent C 必须核对 `ci-artifact-manifest.json` 的 `branch=main`、`commitSha`、`runId`、`runAttempt`、`artifactName`，并检查 `gateway-direct-smoke.log`、`gateway-websocket-smoke.log`、`node-check.log`、`junit.xml`、`ci-failure-summary.md`。
- 失败时只允许在 `main` 上追加修复 commit，不回滚、不创建 PR。

## 文档更新要求

- README 和协议文档必须说明本轮只增强 `agentTrace` artifact 的可审计证据策略，不扩大 Gateway 执行权限。
- flow 和 flowchart 必须同步 Gateway `runAgentLoop` 输出从轨迹扩展为证据充分性、下一步选择、风险和停止原因。
- `update_log.md` 必须记录 v0.6 变更、关键文件、验证结果和遗留风险。

## 验收标准

- `agentTrace` 仍包含旧字段，且新增证据策略字段可由 smoke 读取断言。
- direct/WebSocket smoke 都能证明 agent loop 消费 artifact、判断缺口、选择下一步并输出审批/停止原因。
- 文档说明本轮是 artifact trace 智能化，不扩大 Gateway 权限。
- 本地轻量检查通过，云端最新 `origin/main` run 由 Agent C artifact 复判通过。

## 风险和禁止项

- 禁止扩大真实浏览器、Shell、AppleScript、桌面 App 执行权限。
- 禁止把自然语言指令直接执行。
- 禁止删除 smoke 断言来制造通过。
- 禁止把无关未提交改动混入 v0.6。
- 禁止把旧 CI run 或旧 artifact 冒充本轮结果。
