# v0.7 / 手机端 AgentTrace 复核体验

## 版本分配依据

当前最新版本为 v0.6，已增强 Gateway `agentTrace` JSON 的 readiness、decisionChecklist、selectedNextAction、riskTags、stopReason 和 handoffSummary。本轮承接 v0.6，把这些安全复核线索以小型、向后兼容的 artifact metadata 形式带到 iPhone/iPad 控制台展示，因此分配为 v0.7，仍放在 `v0（核心智能能力）` 阶段。

## 背景

Claw 当前手机端 Mission Run 只汇总 artifact 数量和 kind。Gateway v0.6 已能写出更有审计价值的 `agentTrace`，但控制台无法直接看见 trace 的证据分数、缺失信号、推荐下一步和停止原因。用户总目标是继续优化 Claw 的 macOS Gateway 与 iPhone/iPad 控制台，让它更像 OpenClaw 式电脑智能体，同时严格遵守安全边界和 A/B/C/X 流程。

## 目标

- 为 `ClawGatewayArtifact` 增加向后兼容的可选字符串 metadata，用于承载 `agentTrace` 的轻量复核摘要。
- Gateway 在 `agentTrace` artifact event reference 上附带安全 metadata，例如 readiness score、missing signals、selected next action、risk tags、stop reason、handoff summary。
- Swift simulator 为模拟 `agentTrace` 也附带同类 metadata，保证无 live Gateway 时 UI 和 smoke 可验证。
- 在 `ClawMissionRunSummary` 中派生 `agentTrace` 复核摘要，并在 Mission Run / Gateway 会话 UI 中显示紧凑复核状态。
- 补充 Swift logic smoke、XCTest、Gateway direct/WebSocket smoke 断言。
- 同步 README、协议、flow、flowchart、update_log 和本提示词文件。

## 非目标

- 不读取或解析 iOS 无权限访问的 Gateway `file://` artifact 文件内容。
- 不新增 action kind、artifact kind、event kind，不升级 `claw.computer.control.v1` schema。
- 不实现完整 JSON artifact viewer、历史 artifact 浏览器、live Gateway 心跳/重连。
- 不新增真实浏览器点击、Shell 执行、AppleScript、Accessibility 控件动作或桌面提交能力。
- 不把自然语言直接交给 Shell、AppleScript 或桌面自动化。
- 不引入第三方依赖。

## 当前架构依据

- `ClawGatewayArtifact` 当前只有 kind/title/reference/isRedacted，live/simulated event reducer 会把 artifact 元数据合并进 `ClawGatewayActionResult`。
- `ClawMissionRunSummary` 是手机端 presentation layer，只从 loop/task/session 派生，不进入 envelope。
- `ClawGatewaySimulator` 已为 `runAgentLoop` 生成 `.agentTrace` artifact。
- `Tools/claw-gateway-server.mjs` 的 `agentLoopAction` 已生成 v0.6 trace JSON 字段。
- CI 会覆盖 Swift logic smoke、Gateway direct smoke、Gateway WebSocket smoke 和 iOS xcodebuild。

## 实现步骤

1. 先读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md` 和本提示词。
2. 同步最新 `origin/main`，确认当前分支是 `main`，工作区无无关改动；如有冲突或不明改动，暂停说明，不要回滚。
3. 在 `Claw/Core/ClawModels.swift` 中给 `ClawGatewayArtifact` 增加可选 `metadata: [String: String]?`，initializer 默认 nil，legacy artifact JSON 缺少 metadata 时必须可正常 decode。
4. 新增或扩展手机端派生模型，例如 `ClawAgentTraceReviewSummary`，并让 `ClawMissionRunSummary` 可携带最近一次 agentTrace 复核摘要。建议字段包括 traceCount、latestTitle、readinessScore、missingSignals、selectedNextActionKind、riskTags、stopReason、handoffSummary、isRedacted。
5. 在 `Claw/Services/ClawStore.swift` 中从最新 session 的 `.agentTrace` artifacts 派生复核摘要；metadata 缺失时降级显示“已收到智能体轨迹，等待详细复核 metadata”，不能崩溃。
6. 更新 `ClawGatewaySimulator` 的 `.agentTrace` artifact，附带小而安全的 metadata。不要放入浏览器文本、命令输出、草稿正文、联系人、截图内容或 token。
7. 在 `Tools/claw-gateway-server.mjs` 中为 `agentTrace` artifact reference 附加 metadata。建议只复制/压缩 trace 的安全字段：
   - `readinessScore`
   - `readinessCanContinue`
   - `satisfiedSignals`
   - `missingSignals`
   - `selectedNextActionKind`
   - `selectedNextActionRequiresApproval`
   - `riskTags`
   - `stopReason`
   - `handoffSummary`
8. 更新 `Claw/Views/ContentView.swift`：Mission Run 面板和 Gateway session/result 区域显示紧凑 AgentTrace 复核条。使用现有 tag/row 视觉语言，不做嵌套大卡片，不加入说明型长文案。
9. 更新 `ClawTests/ClawTests.swift`：
   - legacy artifact 无 metadata 可 decode。
   - 模拟发送后 Mission Run 能派生 agentTrace review。
   - review 能显示 readiness / selected action / stop reason / redacted 状态的关键字段。
10. 更新 `Tools/LogicSmoke.swift`，覆盖 Mission Run agentTrace review 的主要断言。
11. 更新 `Tools/claw-gateway-direct-smoke.mjs` 和 `Tools/claw-gateway-smoke.mjs`，断言 event 中的 `agentTrace` artifact reference 带 metadata，且 metadata 与 trace JSON 的关键字段一致。
12. 同步文档：`README.md`、`Docs/claw-mobile-gateway-protocol.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md`。
13. 只提交本轮相关文件，commit message 用 `v0.7: 增强手机端 AgentTrace 复核体验`，push 到 `origin/main`。

## 验收标准

- 旧 artifact JSON 缺失 metadata 时仍能 decode，旧 simulated/live event 不崩溃。
- Gateway direct/WebSocket smoke 能证明 `agentTrace` artifact event 带安全 metadata。
- Swift logic smoke 和 XCTest 能证明 Mission Run 可派生并展示 agentTrace review。
- UI 只展示复核摘要、风险、缺口、停止原因和下一步建议，不声称 iOS 能直接控制电脑。
- 文档明确本轮是 artifact 复核体验，不新增执行权限。
- 最新 `origin/main` run 的 CI artifact 由 Agent C 下载复判通过。

## 风险和禁止项

- 禁止把 trace payload 中的敏感正文、截图内容、命令输出、草稿正文、联系人或 token 放进 metadata。
- 禁止让 metadata 成为可执行指令；它只能用于展示和复核。
- 禁止扩大真实浏览器、Shell、AppleScript、Accessibility、桌面 App 执行权限。
- 禁止删除或弱化 smoke / XCTest 断言来制造通过。
- 禁止混入无关重构或未授权分支流程。
- 禁止把旧 CI run、旧 artifact 或本地输出冒充本轮云端结果。
