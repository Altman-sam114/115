# v0.8（iPad多栏复核工作台）Agent B 实现提示词

## 版本号

- 版本：v0.8
- 阶段：v0（核心智能能力）
- 任务名：iPad多栏复核工作台
- 提交信息格式：`v0.8: iPad多栏复核工作台`

## 版本分配依据

v0.6 已增强 Gateway `agentTrace` 证据策略，v0.7 已把 `agentTrace` metadata 以安全摘要形式展示到手机端 Mission Run / Gateway 会话。`update_log.md` 的当前遗留明确包含“完整 artifact 内容复核体验和 iPad 多栏操作面板”。本轮 v0.8 接续 v0.7，不扩展协议或执行能力，只优化 iPad/宽屏上的复核与操作效率。

## 背景

Claw 当前是 SwiftUI iPhone 控制台原型：手机端负责输入任务、生成计划、审批高风险动作、发送 envelope、查看 Gateway 事件和 artifact 摘要；真实电脑操作只发生在用户授权的桌面/自托管 Gateway。

当前电脑接管页 `PhoneAgentView` 在 `Claw/Views/ContentView.swift` 中使用单个 `ScrollView` + `VStack` 依次展示：

1. `PhoneAgentCommandPanel`
2. `ClawMissionRunPanel`
3. `PhoneAgentPlanPanel`
4. `ClawMobileBridgePanel`
5. `ClawGatewaySessionPanel`
6. `PhoneAgentPermissionMatrix`
7. `PhoneAgentExecutionPanel`

这个布局适合 iPhone compact 宽度，但在 iPad 或横向 regular size class 上，命令输入、Mission Run、计划、Gateway 会话、事件时间线、envelope 和权限矩阵都挤在同一滚动流里，复核时需要频繁上下跳。v0.8 目标是把现有信息按工作台方式分区，让宽屏用户同时看到左侧命令/任务状态与右侧计划/会话/事件/envelope 复核。

## 目标

在不新增协议、action、artifact、Gateway 权限的前提下，优化电脑接管页的 iPad/宽屏体验：

- 在 `regular` horizontal size class 上使用多栏/分区工作台。
- 左侧工作区展示命令输入和 Mission Run，作为任务启动、审批、继续、重试的主操作区。
- 右侧复核区展示执行计划、Claw 电脑任务/Gateway 配置、Gateway 会话、事件时间线、envelope、权限矩阵和执行日志。
- 手机 `compact` 布局保持现有单栏滚动的信息顺序和行为，不删除现有入口。
- 所有按钮继续调用现有 `ClawStore` 方法：`generatePhoneAgentPlan()`、`simulatePhoneAgentExecution()`、`startAutonomousComputerTakeover()`、`approveAndContinueAutonomousLoop()`、`continueAutonomousLoopAfterReview()`、`queueClawMobileTaskFromCurrentPlan()`、`approveLatestClawMobileTask()`、`sendLatestClawMobileTask()`、`retryLatestGatewayFailures()`。
- 宽屏布局要适合复核：右侧可以分组或分段展示“计划 / Gateway 会话 / Envelope 与审计 / 权限与日志”，避免把所有长文本堆在同一个不可定位的长滚动中。
- 保持现有安全文案：手机审批，桌面 Gateway 执行；不得暗示 iPad/iPhone 可以静默控制电脑或读取其他 App 私有数据。

## 非目标

- 不新增或修改 `claw.computer.control.v1` schema。
- 不新增 action kind、artifact kind、Gateway event kind。
- 不修改 `Tools/claw-gateway-server.mjs`、Gateway handler、smoke 中的协议行为或权限策略，除非 CI 暴露与本 UI 改动直接相关的编译问题。
- 不让 iOS 端读取 Gateway `file://` artifact 内容；继续只展示事件和 metadata 中已有的安全摘要。
- 不改 `ClawMobileBridge` 的 allowlist、审批、token 指纹或 envelope 生成规则。
- 不引入第三方依赖，不引入新的设计系统。
- 不做无关重构，不重命名项目、不改变 Tab 结构、不删除现有面板能力。

## 当前架构依据

关键文件和现状：

- `Claw/Views/ContentView.swift`
  - `PhoneAgentView` 是电脑接管页入口。
  - 已有可复用面板：`PhoneAgentCommandPanel`、`ClawMissionRunPanel`、`PhoneAgentPlanPanel`、`ClawMobileBridgePanel`、`ClawGatewaySessionPanel`、`PhoneAgentPermissionMatrix`、`PhoneAgentExecutionPanel`。
  - 当前 `PhoneAgentView` 是单栏滚动布局；v0.8 应优先复用这些面板，新增少量布局容器，而不是重写业务 UI。
- `Claw/Core/ClawModels.swift`
  - `ClawMissionRunSummary`、`ClawMissionRunStage`、`ClawMissionRunPrimaryActionKind` 是 Mission Run 展示派生模型。
  - `ClawAgentTraceReviewSummary` 从 `agentTrace` artifact metadata 派生轻量复核摘要。
  - `ClawGatewayArtifact.metadata` 是可选字符串字典，不能被扩展成读取真实 artifact 内容的入口。
- `Claw/Services/ClawStore.swift`
  - `missionRunSummary` 从 `autonomousLoop`、最新 `ClawMobileTask`、最新 `ClawGatewaySession` 派生。
  - 发送、审批、重试、live fallback、event reducer 都已有实现。
  - v0.8 应只消费现有 store 状态，不新增 source of truth。
- `README.md`、`Docs/claw-mobile-gateway-protocol.md`、`md/flow/flow.md`、`md/flow/flowchart.md`
  - 已说明 Mission Run 是 presentation layer，不进入 envelope，不扩展 Gateway 权限。
  - 本轮 UI 行为变化后需要同步说明 iPad regular size class 的多栏复核工作台属于展示层优化。

## 实现步骤

1. 开始前同步和检查
   - 执行 `git fetch origin`。
   - 确认当前分支是 `main`，目标远端是 `origin/main`。
   - 检查 `git status --short`，如果存在无关改动，不要覆盖；需要时向人工说明。

2. 重构 `PhoneAgentView` 为自适应布局入口
   - 在 `PhoneAgentView` 中读取 `@Environment(\.horizontalSizeClass)`。
   - `compact` 或 nil 时保持现有单栏 `ScrollView` + `VStack` 顺序和间距。
   - `regular` 时走新的宽屏工作台布局。
   - 保留 `navigationTitle("Claw Agent")`、`.navigationBarTitleDisplayMode(.inline)` 和 `AppSurfaceBackground()`。

3. 新增宽屏工作台容器
   - 可在 `ContentView.swift` 内新增私有/普通 SwiftUI view，例如 `PhoneAgentWorkbenchLayout`、`PhoneAgentPrimaryColumn`、`PhoneAgentReviewColumn`、`PhoneAgentReviewSection`。
   - 左侧列建议宽度稳定，例如 `minWidth` 320、`idealWidth` 380、`maxWidth` 440；右侧列占剩余空间。
   - 外层可以使用 `GeometryReader` + `HStack`、`ViewThatFits` 或 `LazyVGrid`，但要保证 iPad 横屏和较宽窗口不会出现文字重叠或按钮挤压。
   - 左侧列放入 `PhoneAgentCommandPanel(examples:)`、`ClawMissionRunPanel()`，可追加轻量状态块，但不要复制业务逻辑。
   - 右侧列使用分区组织现有面板。建议分区：
     - “计划复核”：`PhoneAgentPlanPanel()`
     - “Gateway 任务”：`ClawMobileBridgePanel()`
     - “会话与事件”：`ClawGatewaySessionPanel()`
     - “权限与日志”：`PhoneAgentPermissionMatrix()`、`PhoneAgentExecutionPanel()`
   - 右侧可以使用 `ScrollView` + `VStack`，或 `TabView`/segmented picker 做复核分区；若新增 picker 状态，应只保存在 View 的 `@State`，不进入 `ClawStore`。

4. 调整现有面板以适配宽屏
   - 尽量复用现有 `.panelCard()`、`SectionHeader`、`PhoneAgentTag`、按钮样式和 8px 圆角风格。
   - 如果某些面板在宽列下过宽或过窄，可加 `.frame(maxWidth: .infinity, alignment: .leading)`、合理的 `GridItem` 或 `ViewThatFits`。
   - 不要把卡片再包进卡片。可以用无边框分区标题、`GroupBox` 风格要谨慎，优先保持现有 panelCard 只用于独立面板。
   - 保证按钮文本在窄列不截断到不可理解；必要时使用竖向按钮组或 `ViewThatFits`。
   - 保证 `lastClawMobileEnvelope`、`lastGatewayEvent`、执行日志等 monospaced 长文本在右侧区域可读，不与其他内容重叠。

5. 保持 compact 行为不变
   - compact 布局的面板顺序、按钮、示例 chips、Mission Run 主动作、Gateway 发送模式、会话复核、事件时间线和执行日志都应保持现有可达。
   - 不要为了宽屏移除 compact 的任何内容。

6. 测试补充
   - 如果仅修改 `ContentView.swift` 且没有新增业务模型，至少保证 Swift 编译路径覆盖该文件。优先运行无签名 iOS build：


   - 如果改动触碰 `Claw/Core/ClawModels.swift` 或 `Claw/Services/ClawStore.swift`，还必须运行 Swift logic smoke：

```sh

  -swift-version 6 \
```

## 验收标准

Agent C 应按以下标准验收：

- `PhoneAgentView` 在 compact horizontal size class 下仍显示原有单栏滚动内容，主要面板顺序和操作能力不丢失。
- `PhoneAgentView` 在 regular horizontal size class 下显示多栏/分区工作台：左侧为命令输入和 Mission Run，右侧为计划、Claw 电脑任务/Gateway 配置、Gateway 会话、事件/envelope 复核、权限矩阵和执行日志。

- UI 未新增协议字段、action kind、artifact kind、Gateway event kind 或 Gateway 权限。
- AgentTrace 复核仍只使用 artifact metadata 安全摘要，不读取 `file://` artifact 内容。

- 本地轻量检查按触发条件运行并记录结果。
- `origin/main` 最新 commit 对应 GitHub Actions run 通过，Agent C 下载并核对最新未加密 artifact manifest/JUnit/日志后通过。

## 风险

- SwiftUI size class 在部分 iPad 分屏窗口可能为 compact；实现必须让 compact 路径仍完整可用。
- 宽屏列宽过小可能导致按钮文本、tag 或 monospaced envelope 重叠；需要用稳定 frame、`minimumScaleFactor`、可滚动区域或垂直堆叠处理。
- 如果右侧使用分段 picker，隐藏分区可能降低一次性复核效率；应确保关键状态仍可快速定位。
- 过度拆分 view 可能引入无关重构；本轮应保持 `ContentView.swift` 内的局部布局调整。
- 文档更新遗漏会导致 Agent C 判定不通过，因为这是用户可见行为变化。
