# v0.9（Gateway能力快照artifact）Agent B 实现提示词

## 版本号

- 版本：v0.9
- 阶段：v0（核心智能能力）
- 任务名：Gateway能力快照artifact
- 提交信息格式：`v0.9: Gateway能力快照artifact`

## 版本分配依据

v0.6 已增强 Gateway `agentTrace` 的证据策略，v0.7 已把 `agentTrace` 安全 metadata 回传到手机端复核，v0.8 已把 iPad/regular size class 复核体验重排为多栏工作台。`update_log.md` 当前遗留仍包含真实 macOS Gateway 能力透明度、完整 artifact 复核体验、live Gateway 心跳和重连等方向。

本轮 v0.9 继续放在 `v0（核心智能能力）` 阶段，但不新增真实执行能力；只让 Gateway 在每个 session 开始后写出一个安全的 session 级 `auditLog` artifact，记录本次 Gateway 的能力和策略快照，方便手机端、Agent C 和后续人工审计理解当前 Gateway 到底是 real、dry-run 还是 disabled。

## 背景

Claw 当前是 SwiftUI iPhone/iPad 控制台 + 桌面 Claw Gateway 原型：

- 手机端负责输入任务、生成计划、展示风险、审批、发送 envelope 和查看事件/artifact 引用。
- 真实电脑动作只发生在用户授权的桌面/自托管 Gateway。
- Gateway 已能处理屏幕观察、浏览器、workspace 文件、Shell、结构化提取、桌面 App 和 `runAgentLoop`，并用 allowlist、token 指纹、workspace 限制和审批闸门控制风险。
- 目前 Gateway 启动日志会打印一部分策略，但事件流里没有 session-start 的完整能力快照 artifact。用户和 Agent C 只能从后续 action artifact 推断某些能力是 dry-run、real 或 blocked。

v0.9 要补齐这个审计缺口：每个 Gateway session 在 `gatewayConnected` 后尽早写入一个 `auditLog` artifact，标题建议 `gateway-capability-snapshot.json`，内容只包含安全配置摘要、短 token 指纹、envelope allowlist 和策略状态，不包含 raw token、自然语言敏感 payload、命令输出、网页正文、截图内容或草稿正文。

## 目标

- Gateway 在 session 开始后写入一个 session 级 `auditLog` artifact，标题使用 `gateway-capability-snapshot.json`。
- 该 artifact 记录安全的 Gateway 能力快照，建议字段包括：
  - `mode`: `gateway-capability-snapshot`
  - `createdAt`
  - `gateway`: workspace root、session workspace、platform、arch、Node 版本、host/port 可选摘要
  - `token`: `configured`、`required`、`fingerprint`，只能使用短 SHA-256 指纹，不能写 raw token
  - `envelope`: `schemaVersion`、`sourceApp`、gateway endpoint/device/security mode、`allowedActionKinds`、`auditEnabled`、`requiresApprovalForSensitiveData`、action count、action kind 列表
  - `policies`: shell、browser network、browser control、screen capture、window metadata、desktop control、workspace 等策略状态和 allowlist
  - `capabilities`: 每类能力的 `state`，取值建议统一为 `real`、`dry-run`、`disabled` 或 `unavailable`，并附带简短原因
  - `safety`: 明确 raw token omitted、自然语言不直连 Shell/AppleScript/桌面自动化、workspace-only、allowlist enforced、final submit gated
- 快照要复用既有 `auditLog` artifact kind、既有 `artifactStored` event kind 和既有 `writeArtifact` 落盘机制。
- 快照应在 `gatewayConnected` 事件之后、任何 action `actionStarted` 之前进入事件流，便于审计 session 初始状态。
- direct smoke 和 WebSocket smoke 都要读取并断言该 artifact 内容，尤其是 raw token 不存在、token 指纹正确、allowedActionKinds 正确、策略状态正确。
- 同步 README、协议、flow、flowchart 和 update_log。

## 非目标

- 不新增 `claw.computer.control.v1` schema 版本。
- 不新增 action kind、artifact kind、Gateway event kind。
- 不新增真实权限，不启用新的 Shell、AppleScript、浏览器、屏幕、窗口、桌面 App 或文件能力。
- 不改变手机端 planner、bridge、event reducer、Mission Run 或 iPad 工作台行为，除非 CI 暴露 session 级 `artifactStored` 事件兼容性问题。
- 不把 raw token、完整 Authorization header、用户自然语言命令、action instruction、`toolArguments`、命令输出、网页正文、截图内容、草稿正文、联系人或其他敏感 payload 写入能力快照。
- 不让能力快照成为执行计划来源；它只能用于审计和复核。
- 不引入第三方依赖，不下载模型、大数据或历史 artifact。
- 不做无关重构，不重命名项目，不改变 main 直推和 Agent C 云端 artifact 验收制度。

## 实现步骤

1. 开始前同步和检查
   - 执行 `git fetch origin`。
   - 确认当前分支是 `main`，目标远端是 `origin/main`。
   - 执行 `git status --short --branch`，如果有无关改动，不要覆盖或回滚；需要时暂停说明。

2. 在 Gateway 中新增能力快照构造函数
   - 在 `Tools/claw-gateway-server.mjs` 中新增纯函数，例如 `gatewayCapabilitySnapshot(envelope, config, sessionID, sessionWorkspace)`。
   - 该函数只从 `envelope.gateway`、`envelope.task.actions[].kind` 和 Gateway `config/options` 派生安全摘要。
   - 不要读取或复制 `action.instruction`、`inputPreview`、完整 `toolArguments`、命令输出、网页正文、截图内容或草稿正文。
   - token 字段只能包含：
     - `configured: Boolean(config.token)`
     - `required: Boolean(config.requireToken)`
     - `fingerprint: config.token ? tokenFingerprint(config.token) : ""`
   - envelope token 也只能保留 `envelope.gateway?.tokenFingerprint || ""`，不能存 raw token 或 Authorization header。
   - allowlist 必须转成稳定数组，例如 `[...config.shellAllowlist].sort()`，保证 smoke 断言稳定。

3. 统一能力状态命名
   - 建议新增 helper，例如 `capabilityState({ enabled, dryRunWhenDisabled, platformRequired })` 或更直接的 per-policy helper。
   - 建议语义：
     - Shell：`real` 仅当 `CLAW_ALLOW_SHELL=1` 且 allowlist 非空；否则 `dry-run`。
     - Browser network fetch：`real` 仅当 `CLAW_ALLOW_BROWSER_NETWORK=1` 且 host allowlist 非空；否则 `disabled`。
     - Browser desktop control：`real` 仅当 `CLAW_ALLOW_BROWSER_CONTROL=1`、browser app allowlist 非空、host allowlist 非空且 `process.platform === "darwin"`；策略启用但平台不支持时为 `unavailable`；否则 `dry-run`。
     - Screen capture：`real` 仅当 `CLAW_ALLOW_SCREEN_CAPTURE=1` 且 `process.platform === "darwin"`；启用但平台不支持时为 `unavailable`；否则 `dry-run`。
     - Window metadata：`real` 仅当 `CLAW_ALLOW_WINDOW_METADATA=1` 且 `process.platform === "darwin"`；否则 `dry-run` 或 `unavailable`，保持与现有 `collectWindowMetadata` 行为一致。
     - Desktop control：`real` 仅当 `CLAW_ALLOW_DESKTOP_CONTROL=1`、desktop app allowlist 非空且 `process.platform === "darwin"`；启用但平台不支持时为 `unavailable`；否则 `dry-run`。
     - Workspace：`real` 或 `workspace-only`，但必须明确路径限定在 `sessionWorkspace` 下。
   - 如果最终取值略有调整，必须在 direct/WebSocket smoke 和文档中保持一致。

4. 在 session 开始处写入 `auditLog` artifact
   - 在 `makeGatewayEvents` 创建 `sessionContext` 和 `events` 后，复用现有 `writeArtifact`：

```js
const sessionConfig = { ...config, sessionWorkspace, sessionContext };
const capabilitySnapshotArtifact = await writeArtifact(
  "auditLog",
  "gateway-capability-snapshot.json",
  gatewayCapabilitySnapshot(envelope, config, sessionID, sessionWorkspace),
  true,
  sessionConfig,
);
```

   - 在 `gatewayConnected` 后立即追加一个 `artifactStored` event：

```js
events.push(
  event({
    sessionID,
    taskID: task.id,
    sequence: sequence++,
    kind: "artifactStored",
    artifacts: [capabilitySnapshotArtifact],
    summary: "Stored Gateway capability snapshot audit artifact",
  }),
);
```

   - 后续 `runAction` 也应复用同一个 `sessionConfig`，避免重复展开 `{ ...config, sessionWorkspace, sessionContext }` 时遗漏字段。
   - 这个 `artifactStored` 是 session 级事件，可以没有 `actionID/actionKind`；不要为它新增伪 action。

5. 保持安全和兼容
   - `auditLog` artifact 必须 `isRedacted: true`。
   - 不新增 artifact kind；标题是文件名，不是 schema 或 kind。
   - 不新增 event kind；继续使用 `artifactStored`。
   - 不改变 `actionPolicy` 对 allowedActionKinds 的执行语义。
   - 不改变任何 handler 的 real/dry-run 执行条件。
   - 不改变 token 校验逻辑，只复用 `tokenFingerprint` 做快照。

6. 更新 direct smoke
   - 在 `Tools/claw-gateway-direct-smoke.mjs` 中新增 helper，例如：
     - `readCapabilitySnapshot(events)`
     - `assertCapabilitySnapshot(events, expected)`
     - `assertNoRawToken(snapshot, token)`
   - 对 `dryRunEvents` 至少断言：
     - 存在 `kind === "artifactStored"` 且 artifact `kind === "auditLog"`、`title === "gateway-capability-snapshot.json"`。
     - artifact 文件可读取，payload `mode === "gateway-capability-snapshot"`。
     - JSON 字符串不包含 `smoke-token`。
     - `token.configured === true`，`token.fingerprint === tokenFingerprint(token)`。
     - `envelope.allowedActionKinds` 包含 direct smoke 的 `observeScreen`、`controlBrowser`、`manageFiles`、`runShellCommand`、`extractData`、`operateDesktopApp`、`runAgentLoop`。
     - Shell、browser control、screen capture、window metadata、desktop control 默认为 dry-run 或 disabled，具体取值与实现一致。
     - `gateway.sessionWorkspace` 或等价字段包含当前 session workspace，且路径位于配置 workspace 下。
   - 对 `allowlistEvents` 至少断言 Shell 能力从 dry-run 变为 real 或 enabled-with-allowlist，allowlist 包含 `pwd`。
   - 对 `desktopPolicyEvents` 和 `browserPolicyEvents` 可补充断言 respective policy 已启用但因 allowlist/platform 可能是 `real`、`unavailable` 或 policy-blocked 相关状态；要保证断言与当前 macOS/Linux CI runner 行为兼容。
   - 更新最终 event count 预期时不要写死固定数字；当前脚本只打印计算值，新增 snapshot 后事件数会增加。

7. 更新 WebSocket smoke
   - 在 `Tools/claw-gateway-smoke.mjs` 中读取 `auditLog` capability snapshot。
   - 断言 live path 事件流在 `gatewayConnected` 后有 session-start `artifactStored` 快照。
   - 断言 JSON 不包含 `smoke-token`。
   - 断言 `allowedActionKinds` 正好覆盖 WebSocket envelope 的 `controlBrowser`、`manageFiles`、`runShellCommand`、`extractData`、`runAgentLoop`，不要求包含 direct smoke 才有的 `observeScreen`/`operateDesktopApp`。

## 风险

- session 级 `artifactStored` 事件没有 `actionID/actionKind`；现有 Swift reducer 应能处理可选 action 字段，但如果云端或本地暴露问题，只做最小兼容修复，不扩展协议。
- 快照如果包含 `toolArguments`、action instruction 或用户命令，容易泄露敏感数据；实现时必须只保留 action kind/count 和 Gateway policy 摘要。
- policy state 命名若不稳定，会导致 smoke 在 macOS/Linux runner 间不一致；需要把平台差异显式写成 `unavailable` 或稳定 reason。
- allowlist 的 Set 遍历顺序不稳定；写入和断言前要排序。
- WebSocket smoke 在本机沙箱可能无法监听端口；需要按测试规则申请升级权限重跑，不能把沙箱失败伪装成业务失败或通过。
- 文档遗漏会导致 Agent C 判定不通过，因为本轮改变了 Gateway artifact/event 审计行为。

## 禁止项

- 禁止把项目方向改成法律、法务、合同、诉讼等垂直产品。
- 禁止让 iOS/iPad 声称能静默控制电脑或读取其他 App 私有数据。
- 禁止自然语言直连 Shell、AppleScript、浏览器或桌面 App 自动化。
- 禁止绕过 allowlist、审批闸门、workspace 限制、token 指纹和 artifact 审计。
- 禁止新增 schema/action/event/artifact kind。
- 禁止新增真实 Shell、浏览器、屏幕、窗口、AppleScript、Accessibility 或桌面 App 权限。
- 禁止把 raw token、Authorization header、用户命令、action instruction、完整 `toolArguments`、命令输出、网页正文、截图内容、草稿正文或联系人写入 capability snapshot。
- 禁止让 capability snapshot 成为可执行指令来源。
- 禁止删除或弱化 smoke 断言来制造通过。
- 禁止混入无关重构、无关格式化或其他版本工作。
- 禁止未更新相关文档就提交 Gateway 行为变化。
- 禁止未 push、未获得最新 CI run 或未下载最新 CI artifact 时声称云端验证已完成。
