# v0.1（建立多Agent迭代文档）

## 版本号

v0.1

## 版本分配依据

人工未指定版本号。本轮目标是建立长期迭代文档和 Agent 协作系统，属于项目初始化阶段，因此使用 `v0.1`。

## 背景

当前项目是 Claw iOS Controller Prototype：SwiftUI iPhone App 作为电脑接管控制台，桌面 Claw Gateway 作为执行端。项目已经有 README、协议文档、Swift 核心逻辑、Gateway prototype 和 smoke，但缺少统一的 Agent 协作入口、版本日志、测试分层、核心流程文档和提示词目录。

## 目标

建立以下结构：

- `AGENTS.md`
- `update_log.md`
- `md/prompt/README.md`
- `md/prompt/v0（项目初始化）/v0.1（建立多Agent迭代文档）.md`
- `md/test/test.md`
- `md/flow/flow.md`
- `md/flow/flowchart.md`

文档必须适配当前 Claw 项目，不写空泛模板。

## 非目标

- 不开发新功能。
- 不改 Swift、Node Gateway 或 Xcode project。
- 不调整产品方向。
- 不运行完整业务测试，除非文档改动引发代码变更。

## 当前架构依据

- `README.md` 说明项目定位、Gateway 使用方式和 smoke 命令。
- `Docs/claw-mobile-gateway-protocol.md` 说明 `claw.computer.control.v1` envelope、事件流和安全边界。
- `Claw/Core/ClawModels.swift` 定义 action、artifact、event、状态对象。
- `Claw/Services/ClawStore.swift` 实现 planner、bridge、Gateway simulator、live client、event reducer。
- `Tools/claw-gateway-server.mjs` 实现桌面 Gateway prototype。
- `Tools/*smoke.mjs` 和 `Tools/LogicSmoke.swift` 提供测试入口。

## 实现步骤

1. 阅读 README、协议文档、现有 `AGENTS.md`、测试入口和关键源码。
2. 创建 `md/prompt`、`md/test`、`md/flow` 目录。
3. 将 `AGENTS.md` 收敛为入口规则和 Agent A/B/C 工作流。
4. 创建 `update_log.md`，记录当前状态、历史版本和本次文档维护。
5. 创建 `md/test/test.md`，按 Probe、Smoke、Stage Regression、Full 分层。
6. 创建 `md/flow/flow.md`，记录当前真实数据流、执行流、模块边界和铁律。
7. 创建 `md/flow/flowchart.md`，用 Mermaid 画核心逻辑、Gateway 安全边界和 Agent 迭代流。
8. 创建本提示词文件，作为 Agent A 产物样例。
9. 更新 README，把入口从旧 `agent.md` 改为 `AGENTS.md`，并说明新文档体系。
10. 运行 `git diff --check`。

## 文档更新要求

- README 必须指向 `AGENTS.md`，不能继续指向旧 `agent.md`。
- `update_log.md` 必须记录本轮文档体系建立。
- `md/flow/flow.md` 和 `md/flow/flowchart.md` 必须一致。
- `md/test/test.md` 必须记录当前真实测试命令。

## 验收标准

- 所有要求的文件存在。
- `AGENTS.md` 精简、清晰，不堆历史。
- `update_log.md` 有当前状态、历史记录、维护记录。
- `md/test/test.md` 可指导 Agent B/C 选择测试。
- `md/flow/flow.md` 记录真实 Claw 架构，不是通用模板。
- `md/flow/flowchart.md` 至少包含核心逻辑图和 Agent 迭代图，且有中文读图说明。
- `git diff --check` 通过。
