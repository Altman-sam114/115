# v0.2（任务回合化游戏体验）

## 版本号

v0.2

## 版本分配依据

人工未指定版本号。当前仓库已有 `v0.1` 项目初始化和多 Agent 协作文档；本轮目标是基于已有 Claw 手机控制台、自治循环、Gateway session 和 artifact 状态，继续向更像“任务回合/游戏关卡”的可操作体验推进。由于本轮不新增 Gateway action、不改 envelope schema、不引入真实执行器，只做原型体验和派生状态优化，因此沿用 `v0.x` 阶段并递增为 `v0.2`。

## 当前进展摘要

- 已完成 Claw iPhone 控制台原型主链路：自然语言任务 -> `PhoneAgentPlanner` -> `ClawMobileTask` -> `ClawMobileEnvelope` -> 模拟或 live Gateway -> `ClawGatewayEvent` -> `ClawGatewaySession` -> SwiftUI 展示。
- 已有安全边界：iOS 只负责输入、计划、审批、发送 envelope 和查看事件；真实电脑操作只发生在用户授权的桌面/自托管 Gateway。
- 已有 Gateway prototype 能力：屏幕观察 dry-run/截图策略、浏览器 trace 和打开/搜索计划、workspace 文件写入、Shell dry-run/allowlist、结构化提取、桌面 App 审批闸门、`runAgentLoop`/`agentTrace`。
- 已有 UI 基础：五个 Tab、电脑接管页面、自治循环面板、执行计划、Claw 电脑任务、Gateway 会话、权限矩阵、执行日志、能力和榜单。
- 已有测试基础：Swift logic smoke、XCTest、Gateway direct/WebSocket smoke、GitHub Actions `ci-results` 结果包规则。
- 当前主要体验缺口：电脑接管页仍是工程面板堆叠，用户很难一眼看出“当前任务回合到了哪一步、下一步该点什么、风险/证据/完成度如何变化”。这阻碍了继续向“游戏化任务推进”演进。

## 背景

用户要求总结目前进展、设计下一步优化目标，并“向游戏继续推进”。这里的“游戏”不要理解为脱离 Claw 方向的娱乐游戏，也不要引入虚假积分、云端账号或排行榜数据；应理解为把电脑接管流程做成可感知的任务回合：目标清晰、阶段明确、行动按钮聚焦、风险和证据可见、完成/受阻/重试都有反馈。

本轮应优先把现有状态整理成一个手机首屏可理解的“任务回合面板”，而不是扩展 Gateway 真执行能力。原因是当前模型、planner、bridge、simulator、session reducer 已足够支持体验层优化；新增真实工具层会扩大风险和测试面。

## 目标

1. 在电脑接管页加入一个高优先级的“任务回合 / Mission Run”面板，放在 `PhoneAgentCommandPanel` 之后或之前的首屏可见区域。
2. 面板必须从现有 `ClawStore` 状态派生，不新增持久化账户、真实 XP、外部排行榜或远端服务。
3. 面板至少展示：
   - 当前任务目标或默认引导。
   - 当前阶段：规划、待审批、发送中、观察网关、需处理、已完成、已阻断。
   - 回合进度：`iteration/maxIterations`、阶段轨道或进度条。
   - 下一步主动作：启动、审批并继续、复核后重试、查看阻断原因或等待事件，必须调用现有 store 方法。
   - 安全信息：风险分、审批点、阻断数、是否需要用户确认。
   - 证据/产物：最新 session 的 artifact 数量和 artifact kind 摘要，例如截图、浏览轨迹、文件变更、命令输出、智能体轨迹。
   - 结果摘要：成功/失败/可重试动作数量。
4. 保持 Claw 安全叙事：所有文案必须明确“手机审批，桌面 Gateway 执行”，不得暗示 iOS 可静默控制电脑或读取其他 App 私有数据。
5. 加入可测试的派生模型或计算逻辑，让 Swift logic smoke/XCTest 能覆盖任务回合摘要在 idle、waiting approval、needs attention、completed/blocked 等关键状态的输出。
6. 更新 README、`Docs/claw-mobile-gateway-protocol.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`update_log.md` 中受影响内容，说明本轮是 UI/派生状态优化，不改变 `claw.computer.control.v1` schema。

## 非目标

- 不新增 Gateway action kind、artifact kind、envelope 字段或协议版本。
- 不修改 `Tools/claw-gateway-server.mjs` 的真实执行策略，除非测试发现必须修补文档中已存在的行为不一致。
- 不引入 SpriteKit、SceneKit、Unity、游戏引擎、第三方依赖或网络服务。
- 不做虚假账号、真实积分、真钱奖励、云排行榜或跨设备成就系统。
- 不把 Claw 改成法律、法务、合同、诉讼等垂直产品。
- 不声称 iOS 可以静默读取短信、微信、钉钉、飞书等第三方 App 私有数据。
- 不做大规模 SwiftUI 文件拆分或设计系统重构；本轮只做与任务回合面板直接相关的小步调整。

## 实现步骤

1. 开始前按项目规则同步并检查：
   - `git fetch origin`
   - 确认当前分支是 `main`，目标远端是 `origin/main`。
   - `git status --short --branch` 确认无无关改动；如有无关改动，不要混入本轮。
2. 重新阅读 `AGENTS.md`、`update_log.md`、`md/flow/flow.md`、`md/flow/flowchart.md`、`md/test/test.md`、README、协议文档和本提示词。
3. 在 `Claw/Core/ClawModels.swift` 或 `Claw/Services/ClawStore.swift` 增加一个小型派生摘要模型。建议命名为 `ClawMissionRunSummary` 或类似名称，字段建议包括：
   - `command`
   - `phaseTitle`
   - `phaseIcon`
   - `progressCurrent`
   - `progressTotal`
   - `riskScore`
   - `approvalCount`
   - `blockedCount`
   - `succeededCount`
   - `failedCount`
   - `retryableCount`
   - `artifactCount`
   - `artifactKinds`
   - `primaryActionTitle`
   - `primaryActionIcon`
   - `requiresUserApproval`
   - `statusLine`
4. 在 `ClawStore` 增加只读 computed property，例如 `missionRunSummary`，从 `autonomousLoop`、`clawMobileTasks.first`、`clawGatewaySessions.first` 派生。不要把它做成新的 source of truth。
5. 在 `ContentView.swift` 的 `PhoneAgentView` 中加入新的任务回合面板。实现要求：
   - 首屏聚焦，信息密度高但不拥挤。
   - 使用 8px 或以下圆角，延续现有卡片风格，但避免卡片套卡片造成层级混乱。
   - 使用 `Label` 或带文字的 `Button`，不要用只有图标的可点击元素。
   - 支持小屏和 Dynamic Type，文字不能互相覆盖；长命令用 2 行以内并允许缩放或换行。
   - 避免全红单色；可在阶段中使用蓝/橙/绿/紫/红等语义色，但不要引入无意义渐变装饰。
6. 主按钮行为必须映射到现有方法：
   - idle/planning 前：`startAutonomousComputerTakeover()`
   - waitingForUserApproval：`approveAndContinueAutonomousLoop()`
   - needsAttention：`continueAutonomousLoopAfterReview()`
   - completed：可让用户重新启动当前命令，调用 `startAutonomousComputerTakeover()`
   - blocked：不要自动绕过，按钮文案应引导修改任务或白名单；可只禁用主按钮或调用生成计划，不能直接发送。
7. 加入阶段轨道，例如“计划 -> 审批 -> 发送 -> 观察 -> 交付”。轨道状态必须从 `ClawAutonomousLoopPhase` 和最新 session/task 状态派生。
8. 加入证据/产物区，展示 artifact kind 摘要和数量。没有 artifact 时显示空态，不要伪造截图或日志。
9. 补充测试：
   - XCTest：覆盖 idle 摘要、启动后待审批摘要、审批发送后需处理摘要、重试完成摘要、阻断摘要。
   - Logic smoke：至少断言 `missionRunSummary` 能反映 phase、progress、approval/risk、artifact/result 数量。
10. 更新文档：
   - README：完成情况和 UI 说明中加入任务回合面板。
   - `Docs/claw-mobile-gateway-protocol.md`：说明任务回合摘要是手机端 presentation layer，不改变 envelope/schema/event。
   - `md/flow/flow.md`：在 SwiftUI App、核心状态对象或未来扩展点中加入任务回合摘要。
   - `md/flow/flowchart.md`：更新 UI 节点，体现 session 被整理成任务回合面板。
   - `update_log.md`：新增 `v0.2 / 任务回合化游戏体验` 记录，写核心变更、关键文件、验证结果和遗留事项。

## 验收标准

- 主按钮只调用已有安全路径，不绕过审批、不直接发送被阻断任务。
- 任务回合摘要完全从现有 store 状态派生，不引入重复 source of truth。
- `ClawMobileEnvelope` 不出现原始 token，schema 仍是 `claw.computer.control.v1`。
- XCTest 和 Swift logic smoke 覆盖新增派生逻辑。
- README、协议文档、flow、flowchart、update_log 已同步。

## 风险和禁止项

- 禁止为了“游戏化”弱化高风险动作提示；Shell、桌面 App 最终提交、消息发送、文件写入仍必须突出审批和审计。
- 禁止伪造 artifact、进度、得分、排行榜或远端执行状态。
- 禁止新增未测试的 action/artifact/schema 字段。
- 禁止让 iOS 端声称能静默控制电脑、读取其他 App 私有数据或绕过 Gateway。
- 禁止把自然语言直接交给 Shell、AppleScript 或桌面自动化。
- 禁止删除测试断言来让测试通过。
- 禁止混入大规模 SwiftUI 重构、无关配色重做或无关文案改写。
- 禁止未配置权限时声称已经完成 GitHub Actions artifact 下载复判。
